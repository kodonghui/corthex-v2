## 1. 개요

이 도구는 LEET(법학적성시험) 수험생 커뮤니티에서 **Pain Point 관련 게시글**을 자동으로 수집하는 Python CLI 크롤러입니다.

### 해결하는 문제
2026 예비창업패키지 사업계획서에 "수험생들이 실제로 이 문제를 느끼고 있다"는 증거가 필요합니다. 이 도구는 수험생 커뮤니티에서 키워드 기반으로 관련 게시글을 대량 수집합니다.

### 지원 플랫폼

| 플랫폼 | 방식 | 로그인 필요 | 현재 상태 |
|--------|------|------------|----------|
| 디시인사이드 법전원갤러리 | requests + BS4 | X | 정상 작동 |
| 오르비 | requests + BS4 (SSR 파싱) | X | 정상 작동 |
| 네이버 블로그 | requests + BS4 | X | 정상 작동 |
| 네이버 카페 | Playwright (브라우저 자동화) | O | 구현됨 (CAPTCHA 이슈 가능) |
| 다음 카페 (서로연) | Playwright (카카오 로그인) | O | 구현됨 (CAPTCHA 이슈 가능) |

### 수집 대상 Pain Point 3가지

| # | Pain Point | 설명 | 키워드 수 |
|---|-----------|------|----------|
| 1 | 해설 불일치 | "뭐가 맞는 해설인지 모르겠다" | 34개 |
| 2 | 피드백 부재 | "왜 틀렸는지 모르겠다" | 35개 |
| 3 | 접근성 불평등 | "돈이 없으면 좋은 교육을 못 받는다" | 31개 |
| | **합계** | | **100개** |

## 3. 빠른 시작

### 가장 간단한 실행

```bash
# 디시인사이드에서 모든 Pain Point 검색 (기본 설정)
python leet_crawler.py -p dcinside -pp all
```

### 실제로 효과가 검증된 명령어 조합

```bash
# [추천] 디시인사이드 — 가장 솔직한 수험생 발언
python leet_crawler.py -p dcinside -pp all --max-posts 5 --delay 2 -o ./results_dc

# [추천] 오르비 — LEET 관련 정보글/후기
python leet_crawler.py -p orbi -pp all --max-posts 5 --delay 2 -o ./results_orbi

# 특정 Pain Point만 검색
python leet_crawler.py -p dcinside -pp 1    # 해설 불일치만
python leet_crawler.py -p dcinside -pp 2    # 피드백 부재만
python leet_crawler.py -p dcinside -pp 3    # 접근성 불평등만

# 특정 키워드 하나로 검색
python leet_crawler.py -p dcinside -k "LEET 해설 다르다"

# 모든 공개 플랫폼 동시 검색
python leet_crawler.py -p all -pp all -o ./results_all
```

### 실행 시 예상 소요 시간

| 설정 | 키워드 수 | 예상 시간 |
|------|----------|----------|
| 1개 플랫폼, 1개 PP | ~34개 | 5~10분 |
| 1개 플랫폼, all PP | 100개 | 15~25분 |
| all 플랫폼, all PP | 100개 x 3 | 45~75분 |
| 커스텀 키워드 1개 | 1개 | 30초~1분 |

> 소요 시간은 `--delay` 값과 `--max-posts` 값에 따라 달라집니다.

## 4. 전체 CLI 옵션 레퍼런스

```
python leet_crawler.py [옵션]
```

### 필수 옵션 (둘 중 하나 필수)

| 옵션 | 축약 | 값 | 설명 |
|------|------|-----|------|
| `--pain-point` | `-pp` | `1`, `2`, `3`, `all` | Pain Point 번호로 검색. config.py의 키워드 사용 |
| `--keyword` | `-k` | 문자열 | 직접 키워드 입력. `--pain-point` 대신 사용 |

### 플랫폼 설정

| 옵션 | 축약 | 값 | 기본값 | 설명 |
|------|------|-----|--------|------|
| `--platform` | `-p` | `dcinside`, `orbi`, `naver_blog`, `naver_cafe`, `daum_cafe`, `all` | `dcinside` | 크롤링 대상 플랫폼 |

> `all`은 공개 사이트(dcinside, orbi, naver_blog)만 포함. 카페는 `--login` 필요.

### 수집량 제어

| 옵션 | 축약 | 값 | 기본값 | 설명 |
|------|------|-----|--------|------|
| `--max-posts` | `-m` | 정수 | 20 | 키워드당 최대 수집 게시글 수 |
| `--delay` | `-d` | 실수(초) | 2.0 | 요청 간 딜레이. 서버 부하 방지용 |

> `--max-posts 5`이면 키워드당 최대 5개 게시글만 수집. 빠르게 테스트할 때 유용.
> `--delay`를 1 미만으로 설정하면 IP 차단 위험. 최소 1.5 이상 권장.

### 출력 설정

| 옵션 | 축약 | 값 | 기본값 | 설명 |
|------|------|-----|--------|------|
| `--output-dir` | `-o` | 경로 | 크롤링 폴더 | 결과 JSON/MD 저장 디렉토리 |
| `--json-only` | — | (플래그) | false | JSON만 출력, Markdown 생성 안 함 |

### 카페 로그인 옵션

| 옵션 | 값 | 설명 |
|------|-----|------|
| `--login` | (플래그) | 카페 로그인 모드 활성화 |
| `--naver-id` | 문자열 | 네이버 아이디 |
| `--naver-pw` | 문자열 | 네이버 비밀번호 |
| `--kakao-id` | 문자열 | 카카오 아이디 (다음 카페용) |
| `--kakao-pw` | 문자열 | 카카오 비밀번호 |

## 5. 플랫폼별 상세 사용법

### 5-1. 디시인사이드 법전원갤러리

**특징**: 가장 솔직하고 날것의 수험생 발언. 욕설/비속어 포함되지만 Pain Point가 가장 직접적으로 드러남.

**크롤링 방식**:
1. 먼저 **갤러리 내 검색** 시도 (`search_gallery`) — lawschool 갤러리 내에서만 검색
2. 결과 없으면 **통합검색**으로 폴백 (`search_keyword`) — lawschool URL 포함된 결과만 수집
3. 각 게시글을 **모바일 URL로 변환**하여 본문 수집 (모바일 페이지가 파싱 용이)
4. `is_leet_related()` 함수로 LEET 관련 글만 필터링 (게임/팬픽 등 노이즈 차단)

**Anti-bot 대응**:
- 모바일 Android User-Agent 사용
- Referer 헤더 설정
- 요청 간 딜레이 (기본 2초)
- IP 차단 시 약 10분 후 자동 해제

**실행 예시**:
```bash
# 기본 실행
python leet_crawler.py -p dcinside -pp all -o ./results_dc

# 빠른 테스트 (키워드당 3개만)
python leet_crawler.py -p dcinside -pp 1 -m 3 -d 1.5

# 딜레이 늘려서 안전하게
python leet_crawler.py -p dcinside -pp all -d 3 -o ./results_dc
```

**실제 수집 결과 (2026-03-15)**:
- 키워드 100개, max-posts 5, delay 2초
- 총 137건 수집 → 품질 필터링 후 **51건** (37% 통과율)
- PP1: 5건, PP2: 31건, PP3: 15건

### 5-2. 오르비

**특징**: 입시 커뮤니티라 LEET 관련 진지한 글이 많음. 다만 수능 관련 글도 많아 노이즈 발생.

**크롤링 방식**:
1. `https://orbi.kr/search?q=키워드` 검색
2. 게시글 링크 패턴 `/000XXXXXXXXXX/` 매칭으로 URL 추출
3. 각 게시글 접근 시 **og:title** 메타태그에서 제목, **body 텍스트**에서 본문 추출
   - 오르비는 React SPA이지만 SSR(Server-Side Rendering)으로 body에 텍스트 포함됨
   - CSS 셀렉터로는 본문 추출 불가 (div.content-container 등 없음)

**주의사항**:
- 오르비는 수능 커뮤니티이므로 "해설", "점수", "학원" 같은 키워드가 수능 글에도 걸림
- 반드시 후처리(품질 필터링)가 필요함
- 본문은 최대 3000자까지 수집 (body 전체 텍스트에서 추출하므로 길면 잘림)

**실행 예시**:
```bash
python leet_crawler.py -p orbi -pp all -m 5 -d 2 -o ./results_orbi
```

**실제 수집 결과 (2026-03-15)**:
- 키워드 100개, max-posts 5, delay 2초
- 총 850건 수집 → LEET 필터링 76건 → 품질 필터링 후 **12건** (1.4% 통과율)
- 오르비는 양보다 질이 중요. 수능 노이즈가 심해서 후처리 필수.

### 5-3. 네이버 블로그

**특징**: "LEET 후기", "로스쿨 준비 비용" 등 장문의 수험 후기가 많음. 정보성 글이 많아 Pain Point 증거보다는 배경 자료에 적합.

**크롤링 방식**:
1. 네이버 블로그 검색 (`search.naver.com?where=blog&query=키워드`)
2. 검색 결과에서 블로그 URL + 미리보기 텍스트 수집
3. 블로그 본문 접근 시 **iframe 구조** 처리 (mainFrame → PostView URL 변환)
4. 본문 접근 실패 시 미리보기 텍스트라도 저장

**실행 예시**:
```bash
python leet_crawler.py -p naver_blog -pp 3 -o ./results_blog
```

## 6. 키워드 설정 가이드

### 키워드 파일 위치
`크롤링/config.py`의 `KEYWORDS` 딕셔너리

### 현재 키워드 구성 (총 100개)

**Pain Point 1 — 해설 불일치 (34개)**
```
해설 불일치/신뢰성: "LEET 해설 다르다", "리트 해설 불일치", ...
가답안/오답 논란:   "리트 가답안 오류", "LEET 정답 바뀜", ...
해설 품질:         "리트 해설 이해 안됨", "LEET 기출 해설 부실", ...
강사 간 차이:      "리트 강사 풀이 다름", "법률저널 해설 오류", ...
```

**Pain Point 2 — 피드백 부재 (35개)**
```
점수 정체/안오름:   "LEET 왜 틀렸는지 모르겠다", "리트 110점 벽", ...
장수생/재수:       "리트 3수 포기", "LEET 멘탈 관리", ...
피드백 부재:       "LEET 피드백 받을 곳 없다", "리트 약점 파악 어렵다", ...
AI/새로운 시도:    "ChatGPT LEET 공부", "AI 리트 공부", ...
구조적 한계:       "리트 인강 한계", "LEET 문제 수 적다", ...
```

**Pain Point 3 — 접근성 불평등 (31개)**
```
비용 문제:         "LEET 학원비 비싸다", "로스쿨 대출", ...
금수저/불평등:     "로스쿨 금수저", "로스쿨 음서제", ...
지방 불리:         "지방 LEET 준비 불리", "지방 리트 정보 부족", ...
독학 vs 학원:      "리트 독학 vs 학원", "LEET 학원 안가면", ...
```

### 키워드 추가/수정 방법

`config.py`를 편집기로 열어서 KEYWORDS 딕셔너리에 추가:

```python
KEYWORDS = {
    "pain_point_1_해설불일치": [
        "LEET 해설 다르다",
        "새로운 키워드 추가",        # <-- 여기에 추가
    ],
    # 새로운 Pain Point 그룹도 추가 가능
    "pain_point_4_새주제": [
        "새 주제 키워드1",
        "새 주제 키워드2",
    ],
}
```

### 키워드 작성 팁
- **구체적일수록 좋음**: "LEET" 단독보다 "LEET 해설 다르다"가 관련 글을 잘 찾음
- **"리트"와 "LEET" 모두 사용**: 수험생들은 두 표기를 혼용
- **비속어/은어 포함**: "리트 ㅈ같다", "로스쿨 ㅅㅂ" 등도 실제 Pain Point 증거
- **너무 일반적인 키워드 주의**: "점수 안 오른다" 같은 건 수능/토익 등 다른 시험도 걸림

## 7. 결과 파일 구조

### 7-1. JSON 출력 (crawl_results.json)

```json
[
  {
    "platform": "디시인사이드 법전원갤러리",
    "url": "https://gall.dcinside.com/mgallery/board/view/?id=lawschool&no=253097",
    "title": "이재빈 언어이해 총평 올라옴",
    "content": "■총평[2024 LEET 언어이해 총평] By 해커스 로스쿨 언어이해 이재빈 강사...",
    "date": "2024-07-15",
    "comments": ["댓글1 텍스트", "댓글2 텍스트"],
    "collected_at": "2026-03-15T20:05:23.456789",
    "keyword": "LEET 해설 논란",
    "group": "pain_point_1_해설불일치"
  },
  ...
]
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `platform` | string | 출처 플랫폼명 |
| `url` | string | 원본 게시글 URL (증거 레퍼런스로 사용) |
| `title` | string | 게시글 제목 |
| `content` | string | 본문 전체 텍스트 (공백 정리됨) |
| `date` | string | 게시 날짜 (형식은 플랫폼마다 다름) |
| `comments` | array | 댓글 텍스트 배열 (수집 가능한 경우) |
| `collected_at` | string | 수집 시각 (ISO 8601) |
| `keyword` | string | 이 게시글을 찾은 검색 키워드 |
| `group` | string | Pain Point 그룹명 |

### 7-2. Markdown 출력 (crawl_results.md)

자동으로 증거 형식에 맞게 정리됩니다:

```markdown
## 수집 #1
- **출처**: 디시인사이드 법전원갤러리
- **URL**: https://gall.dcinside.com/...
- **날짜**: 2024-07-15
- **제목**: 이재빈 언어이해 총평 올라옴
- **내용 발췌**: "■총평[2024 LEET 언어이해 총평]..." (최대 500자)
- **댓글** (2건):
  - "댓글1..."
  - "댓글2..."
```

## 8. 결과 후처리 (품질 필터링)

크롤러가 수집한 원본 데이터에는 노이즈가 포함됩니다. 사업계획서에 쓰려면 **품질 필터링**이 필수입니다.

### 8-1. 자동 필터링 (LEET 관련성)

크롤러 내장 필터 `is_leet_related()` 함수가 자동으로:
- LEET/로스쿨 핵심 키워드가 포함된 글만 통과
- 게임/팬픽/번역/스포츠 등 명백한 노이즈 차단
- 조합 키워드 2개 이상 동시 출현 시 통과

### 8-2. 수동 필터링 (Python 스크립트)

오르비처럼 수능 노이즈가 많은 경우, 추가 필터링:

```python
import json

with open('results_orbi/crawl_results.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

leet_core = ['leet', '리트', '로스쿨', '법전원', '법학적성', '추리논증', '언어이해']
noise_block = ['수능', '수시', '정시', '내신', 'ebs', '수특', '대입', '6모', '9모']

filtered = []
for d in data:
    text = (d.get('title','') + ' ' + d.get('content','')).lower()
    has_leet = any(kw in text for kw in leet_core)
    has_noise = any(nk in text for nk in noise_block)
    if has_leet and not has_noise:
        filtered.append(d)

with open('results_orbi/crawl_results_filtered.json', 'w', encoding='utf-8') as f:
    json.dump(filtered, f, ensure_ascii=False, indent=2)

print(f'{len(data)}건 → {len(filtered)}건')
```

### 8-3. AI 기반 필터링 (Claude Code 에이전트)

가장 정확한 방법. Claude Code에서:

```
"results_dc/crawl_results.json 파일을 읽어서,
각 게시글을 하나하나 읽고 LEET Pain Point 증거로 쓸 수 있는 글만 남겨줘.
결과를 crawl_results_cleaned.json으로 저장해줘."
```

실제 결과: 디시 137건 → 51건, 오르비 76건 → 12건

### 9-3. CAPTCHA 우회 — Chrome 디버깅 모드 (가장 안정적)

네이버/카카오는 Selenium/Playwright 자동화를 감지하여 CAPTCHA를 띄웁니다. 이를 우회하는 가장 확실한 방법:

**Step 1**: Chrome을 디버깅 모드로 실행
```bash
# Windows
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:\ChromeDebug"

# 또는 Edge
"C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --remote-debugging-port=9222 --user-data-dir="C:\EdgeDebug"
```

**Step 2**: 열린 브라우저에서 네이버/카카오에 **수동으로 1회 로그인**

**Step 3**: 크롤러를 해당 브라우저에 연결 (향후 `--debug-port 9222` 옵션 추가 예정)

### 9-4. 주의사항
- **2단계 인증 해제** 필수 (네이버: 내 정보 > 보안 설정)
- 비공개 카페는 **해당 카페에 가입된 계정**이어야 접근 가능
- 카페 크롤링은 **headless=False** (브라우저 창이 열림)로 실행됨
- 과도한 크롤링은 계정 정지 위험. `--delay 3` 이상 권장

### 10-5. 설계 원칙

1. **공개 사이트는 requests, 로그인 사이트는 Playwright** — 불필요한 브라우저 실행 회피
2. **모바일 페이지 우선** — HTML 구조가 단순하여 파싱 안정적
3. **서버 부하 방지** — 요청 간 최소 2초 딜레이, 재시도 최대 3회
4. **중복 제거** — URL 기반 + 콘텐츠 해시 기반
5. **개인정보 미수집** — 닉네임/실명/IP는 수집하지 않음
6. **점진적 저장** — 크롤링 중 결과를 JSON으로 저장 (중단 시 복구 가능)

## 11. 트러블슈팅

### "요청 최종 실패" 오류
```
[ERR] 요청 최종 실패: https://gall.dcinside.com/... — ConnectionError
```
**원인**: IP 차단 또는 네트워크 오류
**해결**:
1. `--delay` 값을 높여보세요 (예: `--delay 5`)
2. 5~10분 후 다시 시도 (디시 IP 차단은 보통 10분 내 해제)
3. VPN 사용

### 디시인사이드 본문이 비어있음
**원인**: 디시가 모바일 페이지 구조를 변경했을 수 있음
**해결**:
1. 해당 URL을 브라우저에서 직접 열어 확인
2. `leet_crawler.py`의 `get_post()` 메서드에서 CSS 셀렉터 업데이트

### 오르비 수집이 적음 / 수능 글만 나옴
**원인**: 오르비는 수능 커뮤니티이므로 LEET 키워드가 수능 글에도 매칭됨
**해결**:
1. 후처리 필터링 적용 (섹션 8 참조)
2. 키워드를 더 구체적으로: "LEET 해설" 대신 "리트 추리논증 해설 불일치"

### 네이버 카페 로그인 실패 / CAPTCHA
**원인**: 네이버가 자동화 브라우저를 감지
**해결**:
1. Chrome 디버깅 모드 사용 (섹션 9-3 참조)
2. 2단계 인증 해제 확인
3. 기존 크롬 프로필에서 이미 로그인된 상태로 연결

### ImportError: No module named 'bs4'
```bash
pip install beautifulsoup4 lxml
```

### Playwright 관련 오류
```bash
pip install playwright
playwright install chromium
```

### Windows 인코딩 오류 (UnicodeDecodeError)
```bash
# 환경변수 설정 후 실행
set PYTHONIOENCODING=utf-8
python -X utf8 leet_crawler.py -p dcinside -pp all
```
