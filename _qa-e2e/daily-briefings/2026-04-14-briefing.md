# 클로드코드 데일리 브리핑 — 2026-04-14

## 1. Claude Code 업데이트

- **현재 최신 버전**: v2.1.101 (2026-04-10 릴리즈)
- **새 기능**:
  - `/team-onboarding` 커맨드 추가 — 신규 팀원 온보딩 가이드 자동 생성
  - Brief mode 개선 — 구조화된 응답 대신 plain text 반환 시 자동 재시도
  - Effort level 단순화 — low / medium / high + 새 심볼
  - Voice STT 지원 언어 20개로 확대 (러시아어, 폴란드어, 터키어 등 10개 추가)
  - CLAUDE_CODE_DISABLE_CRON 환경변수 추가 — 예약 크론 잡 비활성화
  - Prompt cache 버그 수정 — SDK query() 호출 시 캐시 무효화 문제, 입력 토큰 비용 최대 12배 절감
- **/dream 공식 상태**: **아직 미출시** (조용한 롤아웃 중)
  - v2.1.59+부터 바이너리 내 인프라 탑재 확인됨
  - 공식 문서/블로그/changelog 미게재 (2026-03 기준)
  - 수동 트리거 가능: Claude에게 "dream" 또는 "consolidate my memory files" 직접 지시

## 2. GitHub 핫토픽 (anthropics/claude-code)

- **Issue #46829** — Cache TTL 무음 회귀 (1h → 5min, 2026-03 초)
  캐시 생성 비용 20~32% 증가, 쿼터 소진 급증. 공식 발표 없이 변경되어 커뮤니티 강한 반발.
- **Issue #41930** — 전 유료 플랜 비정상 사용량 소진 (2026-03-23~)
  복수 원인 식별됐으나 공식 커뮤니케이션 없음. Max 플랜 5시간 한도가 1~2시간 만에 소진 됨.
- **Issue #43286** — Opus 4.6 품질 저하 (2026-04-03)
  thinking content redaction 롤아웃과 품질 회귀 상관관계 확인 (17,871개 thinking block 분석).
- **Issue #42796** — 2월 업데이트 이후 복잡한 엔지니어링 태스크에서 Claude Code 사용 불가 수준 성능 저하 보고.
- **Issue #15942** — Visual Studio 2026 네이티브 플러그인 요청 (현재 VS Code / JetBrains만 지원).

## 3. AI 코딩 뉴스

- **Cursor 3 출시 (2026-04-06)**: Agent-First 인터페이스로 전면 개편. 핵심은 "Agents Window" — 복잡한 멀티스텝 태스크를 병렬 에이전트로 오케스트레이션. Claude Code·Codex에 정면 도전.
- **도구 통합 흐름**: 2026-04 첫 주, Cursor가 병렬 에이전트 UI 재구축, OpenAI가 Claude Code 내부에서 작동하는 공식 플러그인 출시 → 세 도구(Claude Code + Codex + Cursor)를 함께 쓰는 사용자 증가.
- **시장 조사**: Pragmatic Engineer가 906명 개발자 대상 조사(2026-02) — Claude Code가 "가장 사랑받는 AI 코딩 도구" 1위 (46%). 7대 경쟁자: Claude Code, Google Antigravity, OpenAI Codex, Cursor, Kiro, Copilot, Windsurf.

## 4. 추천 액션

- **캐시 비용 확인**: Issue #46829 영향권 — `ANTHROPIC_CACHE_TTL` 환경변수 또는 SDK 설정 점검. 불필요한 토큰 낭비 발생 중일 수 있음.
- **Opus 4.6 품질 모니터링**: 복잡한 태스크에서 응답 품질 체감 저하 시 Issue #43286 구독 → Anthropic 공식 픽스 대기.
- **Claude Code 업데이트**: `npm i -g @anthropic-ai/claude-code` → v2.1.101로 올려 prompt cache 버그 픽스 적용.
- **/dream 실험**: 아직 공식 미출시지만 v2.1.59+ 사용 중이면 "dream" 또는 "consolidate my memory files" 지시로 수동 트리거 테스트 가능.
