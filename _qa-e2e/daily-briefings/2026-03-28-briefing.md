# 클로드코드 데일리 브리핑 — 2026-03-28

## 1. Claude Code 업데이트

- **현재 최신 버전**: v2.1.86 (2026-03-27 릴리즈)
- **새 기능**:
  - `X-Claude-Code-Session-Id` 헤더 추가 — 프록시가 세션별 요청 집계 가능
  - `.jj` / `.sl` VCS 디렉토리 제외 목록 추가 (Jujutsu, Sapling 메타데이터)
  - `/context` 명령어에 액션 제안 기능 추가 — 메모리 비대화, 용량 경고 포함
  - `autoMemoryDirectory` 설정 추가 — 커스텀 메모리 저장 경로 지정 가능
  - Opus 4.6 기본 최대 토큰 64k → 128k 상향
- **버그 수정**:
  - `--resume` 실패 수정 (v2.1.85 이전 세션 대상)
  - 프로젝트 루트 외부 파일(예: `~/.claude/CLAUDE.md`) Write/Edit/Read 실패 수정
  - 스킬 실행 때마다 불필요한 config 디스크 쓰기 수정 (Windows 성능/손상 문제)
- **/dream 공식 상태**: **아직 미공식 출시** — 서버사이드 feature flag로 단계적 롤아웃 중 (2026-03 기준). `/memory` 메뉴에는 노출되나 전체 사용자에게 열리지 않음. 수동 트리거는 Claude에게 "dream" / "consolidate my memory files" 라고 말하면 동작.
- **Auto Dream 구조**: 4단계 (스캔 → 추출 → 병합/정리 → MEMORY.md 200줄 이내 압축). UC Berkeley "Sleep-time Compute" 논문(arXiv:2504.13171) 기반.

## 4. 추천 액션

- **즉시**: `claude --version` 실행해서 v2.1.86으로 업데이트됐는지 확인. 미업데이트 시 `npm update -g @anthropic-ai/claude-code`.
- **Issue #38335 주의**: Claude Max 플랜 사용 중이라면 이번 주 세션 소진이 평소보다 빠를 수 있음. 중요 작업은 새 세션 시작 직후에 처리.
- **/dream 수동 활용**: Auto Dream 공식 미출시지만 "consolidate my memory files"로 수동 트리거 가능. MEMORY.md가 오래됐다면 지금 실행 권장.
- **Codex 동향 주시**: 토큰 효율 4배 주장 — 비용 민감 프로젝트에서 Claude Code API 대체재로 검토 가치 있음.
