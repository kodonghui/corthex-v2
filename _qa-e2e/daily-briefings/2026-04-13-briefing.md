# 클로드코드 데일리 브리핑 — 2026-04-13

## 1. Claude Code 업데이트

- **현재 최신 버전**: v2.1.89+ (March~April 2026 릴리즈 진행 중)
- **새 기능**:
  - Agent Teams 기능 출시 (Code Kit v5.0 포함)
  - 작업 의존성 추적 포함 Task Management 시스템 추가
  - `CLAUDE_CODE_DISABLE_CRON` 환경변수로 스케줄 크론 비활성화 가능
  - 자동 승인 allowlist 확장, 노력 레벨 low/medium/high 단순화
  - SDK `query()` 프롬프트 캐시 무효화 버그 수정 → 입력 토큰 비용 최대 12배 절감
  - 음성 전사(voice transcription) 정확도 향상 (regex, OAuth, JSON 등 dev 용어)
  - bash 명령 파싱: 네이티브 모듈 전환 (속도 향상 + 메모리 누수 제거)
- **/dream 공식 상태**: **아직 미출시** (조용한 롤아웃 중)
  - v2.1.59+부터 바이너리에 포함되어 있으나 공식 문서/블로그 없음
  - 수동 트리거: Claude에게 "dream" / "auto dream" / "consolidate my memory files" 라고 말하면 실행
  - 자동 실행 조건: 마지막 사이클로부터 24시간+ 경과 + 5회 이상 세션 후
- **주요 변경사항**: Homebrew 업데이트 프롬프트, 멀티라인 처리, 샌드박스, MCP 서버 연결, 세션 재개 버그 다수 수정

## 3. AI 코딩 뉴스

- **Cursor 3 출시 (4월 6일)**: 에이전트 퍼스트 인터페이스로 전면 재설계. 복수 에이전트 병렬 실행 가능 (리팩터링·테스트·문서화 동시 처리). Claude Code·Codex 정면 도전.
- **OpenAI Codex**: 독립 클라우드 에이전트 + 데스크탑 앱으로 출시. Anthropic Claude Code 내부에서 실행되는 공식 플러그인도 배포 — 두 도구 함께 쓰는 얼리어답터 증가.
- **시장 현황**: 2026년 기준 7대 경쟁자 — Claude Code, Google Antigravity, OpenAI Codex, Cursor, Kiro, GitHub Copilot, Windsurf. Pragmatic Engineer 설문(906명, 2월)에서 **Claude Code 1위, 46% "가장 애용"**.

## 4. 추천 액션

- **캐시 TTL 이슈 확인**: API 직접 사용 시 `cache_control` TTL이 5분으로 변경됐는지 모니터링. 비용 급증 시 Issue #46829 참고.
- **/dream 수동 테스트**: `CLAUDE.md`에 dream 설정이 이미 되어 있으므로, 다음 세션에서 "auto dream" 명령으로 메모리 통합이 정상 작동하는지 확인.
- **Cursor 3 관심**: 병렬 에이전트 기능이 Claude Code Agent Teams와 유사 — 두 도구 비교 후 워크플로우 최적화 검토.
- **사용량 모니터링**: Max 플랜 사용 중이라면 3월 23일 이후 소진 속도 이상 여부 점검.
