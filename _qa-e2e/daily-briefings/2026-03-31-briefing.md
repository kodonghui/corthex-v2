# 클로드코드 데일리 브리핑 — 2026-03-31

## 1. Claude Code 업데이트

- **최신 버전**: v2.1.59+ (정확한 버전은 `claude --version` 확인)
- **새 기능 (최근)**:
  - `TeammateIdle` / `TaskCompleted` 훅 — 멀티에이전트 워크플로우용
  - `PostCompact` 훅 — 컨텍스트 압축 완료 후 실행
  - `/effort` 슬래시 커맨드 — 모델 노력 수준 설정 (low/medium/high, `/effort auto`로 리셋)
  - `worktree.sparsePaths` 설정 — 대형 모노레포에서 필요한 디렉토리만 체크아웃
  - `-n / --name` CLI 플래그 — 세션 시작 시 표시 이름 설정
  - `ElicitationResult` 훅 — 응답 전 인터셉트/오버라이드
  - Agent Memory — `memory` 프론트매터 필드 (user/project/local 스코프)
  - Sub-agent 제한 — `Task(agent_type)` 문법으로 스폰 가능한 서브에이전트 제한
- **/dream 공식 상태**: **아직 미공식 출시** — 바이너리에 탑재 완료, 조용한 롤아웃 중. 공식 문서 없음. `/memory` 명령으로 "Auto-dream: on/off" 확인 가능. 수동 트리거는 "dream", "consolidate my memory files" 발화로 작동.
- **모델 업데이트**: Claude Sonnet 4.6 출시 (1M 토큰 컨텍스트 베타). Haiku 3 (`claude-3-haiku-20240307`) 2026-04-19 퇴역 예정 → Haiku 4.5 마이그레이션 권장.
- **Team 플랜**: Claude Code가 Team 플랜 Standard 시트에 기본 포함됨.

## 2. GitHub 핫토픽

- **Issue #38335**: *Max 플랜 세션 한도가 3월 23일 이후 비정상 빠르게 소진* — 1~2시간 만에 5시간 윈도우 100% 도달, Max 20x 플랜도 단일 프롬프트로 100% 급등 보고. 현재 진행 중인 버그.
- **Issue #39649**: *Max 5x 플랜 — 유휴 상태에서도 사용량 카운터 증가* — 세션 종료, API 키 폐기, 브라우저 로그아웃 후에도 카운터 상승. 3월 23일 이후 4가지 이상 동시 이상.
- **Issue #33415**: *Windows 11 KB5079473 (3월 패치) → WSL2에서 힙 고갈* — 업데이트 직후 매 세션 4.6GB 메모리 상한 도달.
- **Issue #35981**: *Opus 4.6 반복 장애 (3월 17~18일)* — 단순 프롬프트에 10~15분 행 걸림. Opus 장애 시 Sonnet 자동 폴백 요청 다수.
- **Issue #21431**: *품질 대규모 퇴행 (1월 26일~)* — 생각을 덜 하고 반복 실패 시도 증가 다수 보고.

## 4. 추천 액션

- **즉시**: `claude --version` 확인 → 최신 버전 아니면 `npm update -g @anthropic-ai/claude-code` 업데이트
- **/dream 상태 확인**: Claude Code 세션에서 `/memory` 입력 → "Auto-dream: on/off" 확인. off면 "consolidate my memory files" 발화로 수동 실행 가능.
