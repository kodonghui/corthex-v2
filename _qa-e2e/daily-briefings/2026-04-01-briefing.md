# 클로드코드 데일리 브리핑 — 2026-04-01

## 1. Claude Code 업데이트

- **현재 최신 버전**: v2.1.76+
- **새 기능**:
  - `PermissionDenied` 훅 추가 — auto mode 차단 시 `{retry: true}` 반환으로 재시도 가능
  - `CLAUDE_CODE_NO_FLICKER=1` 환경변수 — 플리커 없는 alt-screen 렌더링 옵트인
  - Named subagent가 `@` mention typeahead에 등장
  - MCP Elicitation 지원 — 에이전트 실행 중 구조화된 입력 요청 가능
  - Effort 레벨 단순화: `low/medium/high` (max 제거), `/effort auto`로 리셋
  - ECONNRESET 재시도 시 fresh TCP 연결 사용 (연결 오류 크게 감소)
  - 프롬프트 캐시 fix — SDK `query()` input 토큰 비용 최대 **12배 절감**
  - `X-Claude-Code-Session-Id` 헤더 추가 — 프록시에서 세션별 집계 가능
- **`/dream` 공식 상태**: **미출시 (quiet rollout 중)**
  - v2.1.59+부터 바이너리에 포함되어 있으나 모든 사용자에게 활성화되지 않음
  - `/memory` 메뉴에서 "Auto-dream: off" 토글 확인 가능 (아직 켜기 불가한 경우 많음)
  - 수동 트리거: Claude에게 직접 "dream" 또는 "consolidate my memory files" 라고 말하면 동작
- **플랫폼 변경**:
  - **Claude Sonnet 4.6** 출시 — 향상된 에이전틱 성능, 더 적은 토큰 소비, 1M 컨텍스트 (beta)
  - Claude Opus 4 / 4.1 모델 선택기 및 Claude Code에서 **deprecated**
  - 1M 컨텍스트 beta (`context-1m-2025-08-07`) → **2026-04-30 종료** 예정 (Sonnet 4.5/4 대상)
  - Claude Code 이제 **Team plan Standard 시트**에서 접근 가능

## 2. GitHub 핫토픽

- **Issue #41242**: ECONNRESET 80% 실패율 (2026-03-30, 보스턴 지역) — CDN 엣지 레이어 문제로 하루 종일 CLI 사용 불가. `api.anthropic.com` + Datadog 텔레메트리 동시 장애.
- **Issue #38335**: Claude Max plan 세션 한도 비정상 소진 (2026-03-23~) — 5시간 윈도우가 1~2시간 만에 소진. Max 20x도 단일 프롬프트에서 21% → 100% 점프 보고.
- **Issue #39649**: Max 5x plan 세션 종료 후에도 사용량 카운터 자동 증가 — 모든 세션/API 키 비활성화 상태에서도 claude.ai 카운터가 올라가는 현상.
- **Issue #33415**: Windows 11 KB5079473 (2026-03) → WSL2에서 heap exhaustion (4.6GB) — Hyper-V 동적 메모리 할당 레이어 변경이 원인으로 추정.
- **Issue #21431**: 2026-01-26~부터 품질 저하 보고 — 문제를 생각하기보다 즉시 여러 번 깨진 시도를 반복하는 행동 패턴.
