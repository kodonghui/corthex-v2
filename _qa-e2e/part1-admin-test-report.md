## 발견된 버그

### BUG-001: 임시 비밀번호 모달에서 비밀번호 미표시 (Critical)

- **페이지**: /admin/employees → 열쇠 아이콘 → 초기화
- **증상**: 비밀번호 초기화 후 모달에 비밀번호가 빈칸으로 표시됨
- **원인**: API 응답 필드명 불일치
  - API 반환: `{ data: { tempPassword: "..." } }`
  - 프론트 코드: `res.data.newPassword` (존재하지 않는 필드)
- **파일**: `packages/admin/src/pages/employees.tsx` (line 44, 182)
- **수정**: `newPassword` → `tempPassword` (타입 + 사용처)
- **커밋**: `500495bb` — 배포 완료
- **심각도**: Critical — 관리자가 직원에게 임시 비밀번호를 전달할 수 없음

## 문제점 및 개선 필요 사항

### P1. 임시 비밀번호 표시 버그 (수정됨)
- 비밀번호 초기화 모달에서 `<code>` 태그가 빈칸
- API 필드명(`tempPassword`) vs 프론트 코드(`newPassword`) 불일치
- **수정 완료**: 커밋 500495bb

### P2. 비밀번호 초기화 후 재확인 불가
- 초기화 직후 모달에서 비밀번호를 한 번 보여주고, 이후 열쇠 아이콘 재클릭 시 "이 직원의 임시비밀번호가 없습니다" 표시
- 관리자가 비밀번호를 놓치면 다시 초기화해야 함
- **제안**: 임시 비밀번호를 DB에 저장해두고 직원이 첫 로그인할 때까지 재확인 가능하도록 변경

### P3. 부서 미배정 직원의 접근 범위 미검증
- UNASSIGNED 직원이 전체 에이전트에 접근 가능한지, 아무것도 못 보는지 테스트 필요
- noname01 테스트 계정 생성 후 확인 예정

### P4. 직원 수정/삭제/비활성화 미테스트
- 직원 편집(이름/이메일/부서 변경) 아직 안 함
- 직원 비활성화 후 로그인 차단 확인 안 함
- 직원 삭제(하드 딜리트) 안 함

### P5. 1-1~1-6 실제 인터랙션 미완
- 대시보드, 회사, 사용자, 부서, 에이전트 페이지는 로딩만 확인
- 각 페이지의 생성/수정/삭제/필터/검색 등 CRUD 테스트 필요

### P6. 1-8~1-15 전체 미시작
- 도구, 비용, 자격증명, 보고라인, 소울템플릿, 모니터링, NEXUS, 설정 전부 미테스트
