프론트엔드 코딩 컨벤션 v2.1

이 문서는 CORTHEX v2 프로젝트의 코드 품질 유지와 협업 효율성을 높이기 위해 작성되었습니다. 모든 프론트엔드 개발자는 이 규칙을 준수해야 합니다.

1. 기술 스택

  • Framework: Next.js 14 (App Router)
  • Styling: Tailwind CSS v3
  • Icons: Phosphor Icons
  • State Management: Zustand

2. 명명 규칙 (Naming Convention)

변수 및 함수명은 카멜 케이스(camelCase)를 사용하며, 컴포넌트 파일은 파스칼 케이스(PascalCase)를 사용합니다.

// 예시
const handleSearch = () => { ... }
export function DocumentCard() { ... }

3. 컴포넌트 구조

각 컴포넌트는 단일 책임 원칙을 따르며, 재사용 가능한 단위로 분리되어야 합니다. 로직은 커스텀 훅으로 추출하는 것을 권장합니다.
