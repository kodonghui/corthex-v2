1. 개요 (Overview)

CORTHEX 플랫폼은 대규모 지식 처리 및 AI 기반 문서 분석을 위해 설계된 분산 마이크로서비스 아키텍처를 채택하고 있습니다. 본 가이드는 핵심 서비스 컴포넌트와 데이터 흐름을 설명합니다.

주요 설계 원칙:

  • 고가용성 (High Availability)
  • 수평적 확장성 (Horizontal Scalability)
  • 보안 우선 (Security First) 설계

2. 핵심 컴포넌트 (Core Components)

API Gateway

클라이언트 요청 라우팅, 인증, 속도 제한 담당

Embedding Engine

문서 텍스트를 벡터로 변환하는 AI 서비스

Vector DB

고차원 벡터 데이터 저장 및 유사도 검색

RAG Controller

검색 증강 생성 로직 및 LLM 오케스트레이션

3. 데이터 파이프라인 예시 (Data Pipeline Example)

새로운 문서가 업로드될 때의 처리 파이프라인 구성 예시입니다.

name: document_ingestion
steps:
  - extract:
      type: pdf_parser
      ocr_enabled: true
  - chunking:
      strategy: semantic
      max_tokens: 512
      overlap: 50
  - embed:
      model: korthex-embed-v2
      dimension: 1536
  - store:
      destination: milvus_cluster_primary
