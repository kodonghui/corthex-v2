# 01. 팀 에이전트 개요 & 서브에이전트와의 차이

## 팀 에이전트란?

Claude Code의 **Agent Teams**는 여러 개의 Claude Code 인스턴스가 **각자 독립적으로** 돌아가면서, 공유 태스크 리스트와 메시지 시스템을 통해 **협업**하는 기능입니다.

쉽게 말하면:
- **서브에이전트** = 비서한테 심부름 시키고 결과만 받는 것
- **팀 에이전트** = 팀원들이 각자 책상에서 일하면서 슬랙으로 대화하는 것

## 핵심 구조

```
┌─────────────────────────────────────────────────┐
│  Team Lead (팀장 = 메인 Claude Code 세션)          │
│  - 작업 분해 및 위임                                │
│  - 결과 종합                                       │
│  - 팀원 생성/종료 관리                              │
├─────────────────────────────────────────────────┤
│  공유 인프라                                       │
│  ├── Task List: ~/.claude/tasks/{team-name}/     │
│  │   └── {task-id}.json (상태, 의존성, 담당자)      │
│  ├── Mailbox: ~/.claude/teams/{team-name}/inboxes/│
│  │   └── {agent-name}.json (수신함)               │
│  └── Config: ~/.claude/teams/{team-name}/config.json│
├─────────────────────────────────────────────────┤
│  Teammate A        Teammate B        Teammate C  │
│  (독립 프로세스)    (독립 프로세스)    (독립 프로세스) │
│  - 자기 컨텍스트    - 자기 컨텍스트    - 자기 컨텍스트 │
│  - 자기 도구들      - 자기 도구들      - 자기 도구들   │
│  - P2P 메시지 가능  - P2P 메시지 가능  - P2P 메시지 가능│
└─────────────────────────────────────────────────┘
```

## 서브에이전트 vs 팀 에이전트 비교

| 구분 | 서브에이전트 (Agent tool) | 팀 에이전트 (TeamCreate) |
|------|--------------------------|--------------------------|
| **프로세스** | 메인 세션 안에서 실행 | 각각 **별도 OS 프로세스** |
| **컨텍스트** | 부모와 공유 (결과만 반환) | 각자 **독립 컨텍스트 윈도우** (1M 토큰) |
| **통신** | 부모에게만 결과 반환 | **팀원끼리 직접 대화** 가능 (P2P) |
| **관찰** | 결과만 볼 수 있음 | tmux에서 **실시간으로** 각 팀원 작업 관찰 |
| **개입** | 끝날 때까지 기다려야 함 | **작업 중간에 메시지** 보내서 방향 변경 가능 |
| **생존** | 부모 죽으면 같이 죽음 | tmux 모드에서 **팀장 죽어도 생존** |
| **중첩** | 서브에이전트가 서브에이전트 불가 | 팀원이 팀 생성은 불가 (1단계만) |
| **토큰 비용** | 낮음 (결과 요약됨) | **높음** (각자 풀 세션 소비) |
| **적합한 일** | 단일 집중 작업 | 병렬 탐색, 토론, 크로스-레이어 |
| **파일 충돌** | 없음 (순차 실행) | **가능** (같은 파일 수정 시 충돌) |

## 언제 팀을 쓰고, 언제 서브에이전트를 쓰나?

### 팀 에이전트가 좋은 경우

- **"이거 A 방법이 나을까 B 방법이 나을까?"** → 2명이 각각 시도해서 비교
- **코드 리뷰를 여러 관점에서** → 보안 전문가, 성능 전문가, 테스트 전문가 각각 리뷰
- **프론트/백엔드 동시 개발** → 각자 다른 파일에서 병렬 작업
- **리서치 + 구현 병렬** → 한 명은 조사, 한 명은 구현
- **BMAD 파티 모드** → 여러 에이전트가 진짜로 토론

### 서브에이전트가 좋은 경우

- **"이 파일 분석해줘"** → 단순 분석 후 결과 반환
- **순차적 작업** → A 끝나야 B 시작하는 경우
- **같은 파일 수정** → 충돌 위험 없이 안전하게
- **빠른 단일 작업** → 오버헤드 없이 신속하게
