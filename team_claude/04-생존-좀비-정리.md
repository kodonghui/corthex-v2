# 04. 팀장 사망 시 팀원 생존 & 좀비 프로세스 처리

## 핵심 질문: "팀장이 죽으면 팀원도 죽나?"

**답: 모드에 따라 다릅니다.**

## In-Process 모드 (같은 터미널)

```
┌─────────────────────────────┐
│  하나의 프로세스              │
│  ├── Lead (PID 1234)        │
│  ├── Teammate A             │  ← 같은 프로세스 안
│  └── Teammate B             │
└─────────────────────────────┘

팀장 죽음 → 전원 사망 ☠️
```

- 모두 **같은 프로세스**이므로 팀장 프로세스가 죽으면 전부 종료
- 서브에이전트와 동일한 동작
- `/resume`이나 `/rewind`로도 팀원 복구 불가

## Split-Pane 모드 (tmux) ⭐

```
┌──────────────┬──────────────┬──────────────┐
│ tmux pane %0 │ tmux pane %1 │ tmux pane %2 │
│ Lead         │ Teammate A   │ Teammate B   │
│ PID 1234     │ PID 5678     │ PID 9012     │
│              │              │              │
│  각각 독립    │  각각 독립    │  각각 독립    │
│  OS 프로세스  │  OS 프로세스  │  OS 프로세스  │
└──────────────┴──────────────┴──────────────┘

팀장 죽음 → 팀원들 살아남음! 🧟 (하지만 좀비가 됨)
```

- 각 팀원이 **별도 OS 프로세스** (별도 PID)
- 팀장 프로세스가 죽어도 tmux pane 안의 팀원들은 **계속 실행**
- **문제**: 이 팀원들은 이제 아무도 관리 안 하는 **고아 프로세스**가 됨
- 리소스(CPU, 메모리, 토큰!)를 계속 소비
- 작업 결과가 어디로 가는지 아무도 모름

## 좀비 프로세스 식별 및 정리

### 1. tmux 세션 확인

```bash
# 모든 tmux 세션 목록
tmux ls

# 출력 예시:
# my-team-lead: 3 windows (created Tue Mar 11 10:00:00 2026)
# my-team-frontend: 1 windows (created Tue Mar 11 10:00:05 2026)  ← 좀비!
# my-team-backend: 1 windows (created Tue Mar 11 10:00:06 2026)   ← 좀비!
```

### 2. 특정 좀비 세션 종료

```bash
# 개별 종료
tmux kill-session -t my-team-frontend
tmux kill-session -t my-team-backend

# 또는 전부 한번에 (주의: 다른 tmux 세션도 날아감!)
tmux kill-server
```

### 3. Claude Code 프로세스 직접 확인

```bash
# 실행 중인 Claude Code 프로세스 확인
ps aux | grep claude

# 고아 프로세스 종료
kill <PID>
```

## 예방 전략

### 1. 정상 종료 순서 준수

```
1. 팀장: 모든 태스크 완료 확인 (TaskList)
2. 팀장: 각 팀원에게 shutdown_request 전송
3. 팀원: shutdown_response(approve: true) 반환
4. 팀원 프로세스 종료됨
5. 팀장: 최종 정리 후 종료
```

### 2. 팀장 비정상 종료 시 복구

```bash
# 1. 남아있는 tmux 세션 확인
tmux ls

# 2. 좀비 팀원 세션에 접속해서 상태 확인
tmux attach -t my-team-frontend

# 3. 팀원이 작업 중이었다면 결과 확인 후 종료
# 4. 필요없으면 kill
tmux kill-session -t my-team-frontend
```

### 3. 자동 정리 훅 (고급)

```json
// .claude/settings.json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "tmux kill-session -t $(tmux display-message -p '#S') 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

## 서브에이전트와의 차이 (생존 관점)

| 상황 | 서브에이전트 | 팀 (in-process) | 팀 (tmux) |
|------|-------------|-----------------|-----------|
| 팀장 정상 종료 | 함께 종료 | 함께 종료 | 종료 요청 후 종료 |
| 팀장 비정상 종료 | 함께 사망 | 함께 사망 | **살아남음 (좀비)** |
| 팀장 세션 재개 | 복구 불가 | 복구 불가 | 별도 접속 가능 |
| 리소스 회수 | 자동 | 자동 | **수동 정리 필요** |

## 실전 팁

1. **tmux 모드에서는 항상 정상 종료** (shutdown_request) 사용
2. 작업 끝나면 `tmux ls`로 고아 세션 확인하는 습관
3. 긴 작업이면 중간중간 태스크 상태 체크 (팀원이 멈춘 건 아닌지)
4. 팀원이 "시체 남기는" 이유: 대부분 팀장이 정상 종료 안 시키고 그냥 꺼서
