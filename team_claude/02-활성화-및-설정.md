# 02. 활성화 방법 및 설정

## 활성화 (필수!)

팀 에이전트는 **실험적 기능**이라 기본 꺼져 있습니다.

`~/.claude/settings.json`에 추가:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

또는 프로젝트별로 `.claude/settings.local.json`에:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

## 디스플레이 모드 3가지

### 1. In-Process (기본, tmux 없을 때)

```
┌──────────────────────────────────────┐
│  하나의 터미널 안에서 전부 실행         │
│                                      │
│  [Lead] ← Shift+Down → [Teammate A] │
│          ← Shift+Down → [Teammate B] │
│                                      │
│  키보드로 팀원 사이를 전환해서 관찰     │
└──────────────────────────────────────┘
```

- **장점**: tmux 설치 필요 없음, 어디서든 동작
- **단점**: 3명 이상이면 출력이 뒤섞여서 혼란
- **조작**: `Shift+Down` / `Shift+Up`으로 팀원 전환

### 2. Split-Pane (tmux)

```
┌──────────────┬──────────────┬──────────────┐
│   Team Lead  │  Teammate A  │  Teammate B  │
│              │              │              │
│  작업 분배    │  프론트 작업   │  백엔드 작업  │
│  결과 종합    │  실시간 관찰!  │  실시간 관찰! │
└──────────────┴──────────────┴──────────────┘
```

- **장점**: 각 팀원이 뭘 하는지 **실시간으로 관찰 가능**
- **단점**: tmux 필요, Windows에서는 WSL 필요
- **추천**: 3명 이상일 때 강력 추천

### 3. Auto (기본값)

- tmux 세션 안이면 → Split-Pane
- 아니면 → In-Process
- 대부분의 경우 이걸 쓰면 됨

## 사용 가능한 도구들 (팀 활성화 시)

| 도구 | 역할 |
|------|------|
| `TeamCreate` | 팀 생성 (이름, 설명) |
| `TaskCreate` | 태스크 생성 (제목, 설명, 의존성) |
| `TaskUpdate` | 태스크 상태/담당자 변경 |
| `TaskList` | 전체 태스크 목록 조회 |
| `TaskGet` | 개별 태스크 상세 조회 |
| `SendMessage` | 팀원에게 메시지 (DM, 브로드캐스트, 종료 요청) |

## 팀 구성 파일 구조

```
~/.claude/
├── teams/
│   └── my-team/
│       ├── config.json          # 팀 설정 (멤버 목록, 리더 ID)
│       └── inboxes/
│           ├── team-lead.json   # 팀장 수신함
│           ├── frontend.json    # 팀원 A 수신함
│           └── backend.json     # 팀원 B 수신함
└── tasks/
    └── my-team/
        ├── task-001.json        # 태스크 1 (상태, 담당자, 의존성)
        ├── task-002.json        # 태스크 2
        └── task-003.json        # 태스크 3
```

### config.json 예시

```json
{
  "leadAgentId": "lead@my-team",
  "leadSessionId": "session-abc123",
  "createdAt": "2026-03-11T10:00:00Z",
  "description": "Frontend/Backend parallel development",
  "members": [
    {
      "agentId": "frontend@my-team",
      "name": "frontend",

      "cwd": "/project",

    },
    {
      "agentId": "backend@my-team",
      "name": "backend",
      "agentType": "general-purpose",

      "cwd": "/project",
      "tmuxPaneId": "%2"
    }
  ]
}
```

## 주의사항

- **한 세션에 한 팀만** 가능 (동시에 여러 팀 불가)
- **팀원이 팀을 만들 수 없음** (팀장만 가능)
- **세션 재개 시** in-process 팀원은 복원 안 됨
- **VS Code 내장 터미널**에서는 split-pane 불가 (in-process만)
- **Windows**: WSL 안에서 tmux 사용 가능
