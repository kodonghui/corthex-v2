# 07. 비용 & 최적화

## 토큰 비용 구조

### 핵심 원리: **선형 증가**

```
팀원 1명 = Claude Code 세션 1개 = 컨텍스트 윈도우 1개

팀원 N명 = N × (개별 세션 토큰 소비)
```

### 대략적 비용 감각

| 상황 | 예상 토큰 소비 |
|------|---------------|
| 혼자 코드 분석 | ~100K 토큰 |
| 같은 일을 3명 팀 | ~300-400K 토큰 (협업 오버헤드 포함) |
| 같은 일을 5명 팀 | ~500-750K 토큰 |
| Anthropic C 컴파일러 (16명) | $20,000 (약 2,000 세션) |

### CLI 토큰 기준 (우리 프로젝트)

CORTHEX v2는 API 과금이 아닌 **CLI 토큰** 방식:
- Claude Max 구독 = 무제한은 아니지만 API 비용보다 저렴
- 팀원마다 별도 세션이므로 **동시 사용량 제한**에 걸릴 수 있음
- 토큰보다 **동시 실행 제한(rate limit)**이 더 제약이 될 가능성

## 최적 팀 크기

### 권장: 3~5명

```
   효과
    ↑
    │         ★ 최적점 (3~5명)
    │        ╱╲
    │       ╱  ╲
    │      ╱    ╲─────── 효과 정체
    │     ╱
    │    ╱
    │   ╱
    │──╱────────────────────→ 팀 크기
    1  2  3  4  5  6  7  8

    2명: 조정 비용 낮음, 병렬성 제한
    3명: 병렬성↑, 조정 비용 적당
    5명: 최대 병렬성, 조정 비용 증가 시작
    8명+: 조정 비용 > 병렬 이득
```

### 팀원당 최적 태스크 수: 5~6개

- **너무 적으면** (1~2개): 팀 스폰 오버헤드가 이득보다 큼
- **적당하면** (5~6개): 팀원이 계속 바쁘게 일할 수 있음
- **너무 많으면** (10+): 컨텍스트 윈도우 부족, 품질 저하

## 비용 절감 전략

### 1. 모델 혼합 사용

```
팀장: Opus (종합 판단력 필요)
리서치 팀원: Haiku (빠르고 저렴, 검색/분석)
구현 팀원: Sonnet (코드 작성 능력, 합리적 비용)
리뷰 팀원: Sonnet (충분한 판단력)
```

### 2. 태스크 독립성 극대화

```
좋은 태스크 설계:
Task 1: "auth/ 폴더의 모든 파일 리팩토링"        → 팀원 A
Task 2: "dashboard/ 폴더의 모든 파일 리팩토링"    → 팀원 B
Task 3: "api/ 폴더의 모든 파일 리팩토링"          → 팀원 C

나쁜 태스크 설계:
Task 1: "auth.ts 파일의 1-50줄 수정"             → 충돌!
Task 2: "auth.ts 파일의 51-100줄 수정"           → 충돌!
```

### 3. 불필요한 브로드캐스트 회피

```
나쁨: broadcast("작업 상태 어때?")   ← 5명이면 5배 비용
좋음: message("frontend", "진행 상황 알려줘")   ← 1명만 비용
```

### 4. 적시 종료

```
태스크 완료되면 바로 shutdown_request 보내기
idle 상태의 팀원 = 리소스 낭비
모든 작업 끝나면 즉시 팀 해산
```

### 5. 팀이 필요 없는 경우 판별

| 상황 | 추천 |
|------|------|
| "이 파일 하나 고쳐줘" | 서브에이전트 또는 직접 |
| "3개 파일 순차 수정" | 서브에이전트 |
| "5개 독립 모듈 동시 개발" | **팀 에이전트** ✓ |
| "코드 리뷰를 3가지 관점으로" | **팀 에이전트** ✓ |
| "방법 2개 비교 토론" | **팀 에이전트** ✓ |
| "단순 검색/분석" | 서브에이전트 |

## 비용 모니터링

```bash
# tmux 세션 수 = 팀원 수
tmux ls | wc -l

# 각 팀원의 프로세스 상태
ps aux | grep claude | grep -v grep
```

### 사후 분석

- Claude Code는 세션별 토큰 사용량을 표시
- 팀장 세션에서 전체 팀 비용 확인 가능
- Hook의 `Stop` 이벤트에서 `cost-tracker` 활용 가능
