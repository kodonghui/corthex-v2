# Party Mode Round 1 — Collaborative Lens
## Spec: 05-agora (Agora 토론 엔진)

---

### Sally (UX)
토론 선택 → 타임라인 확인까지 2클릭으로 자연스럽습니다. 다만 모바일에서 InfoPanel 정보가 완전히 숨겨지는데, 참여자 목록이나 토론 유형 같은 핵심 정보를 타임라인 헤더에 요약해주면 좋겠습니다. 빈 상태 디자인이 "토론을 선택하거나 새 토론을 시작하세요"로 충분하지만, 새 토론 버튼으로의 CTA가 빈 상태 영역에도 있으면 더 직관적입니다.

### Winston (Architect)
컴포넌트 목록에 `DebateResultCard`가 빠져 있습니다. 이 컴포넌트는 채팅 페이지에서 토론 결과를 보여주는 카드인데, 실제 코드에 존재하며 UXUI 리팩토링 대상입니다. 나머지 5개 컴포넌트 구조는 합리적이고, 3컬럼 레이아웃이 재사용 가능한 패턴입니다.

> **Sally → Winston**: DebateResultCard는 agora 페이지가 아니라 채팅 페이지에서 쓰이는 거 아닌가요?
> **Winston → Sally**: 맞습니다. 하지만 agora 컴포넌트 폴더(`components/agora/`)에 있고, 디자인 톤이 agora와 일관되어야 하므로 이 스펙에서 언급해야 합니다. 최소한 "관련 컴포넌트"로 기록해둬야 합니다.

### Amelia (Dev)
data-testid 목록은 대부분 충분합니다. 하지만 토론 생성 모달의 **제출 버튼** testid가 빠져 있습니다. `debate-create-submit` 같은 ID가 필요합니다. 또한 `debate-result-badge` (합의/비합의 뱃지)도 테스트에서 검증할 중요한 요소입니다. `fromChat` 상태일 때 나타나는 "채팅으로 돌아가기" 버튼의 testid도 없습니다.

### Quinn (QA)
로딩 상태가 스펙에 명시되어 있지 않습니다. 섹션 7에 색상은 정의했지만, 토론 목록 로딩 중이나 타임라인 로딩 중일 때 어떤 UI가 보이는지 구체적으로 정의해야 합니다. 에러 상태도 Banana2 프롬프트에는 "Loading state, error state"로 언급되지만 스펙 본문에는 구체적 정의가 없습니다. 토론 생성 실패 시 에러 처리도 빠져 있습니다.

### John (PM)
v1-feature-spec에 있는 **"Diff 뷰 + 책 형식 정리"**가 섹션 4.3에서 "Diff 뷰: 라운드 간 의견 변화 시각화"로만 언급되고, 컴포넌트 목록이나 data-testid에는 반영되지 않았습니다. 이건 v1의 실제 기능인데, 현재 UI에 노출이 안 되는 문제를 스펙 3번에서 인식했으면서도 구체적 해결책이 없습니다. "사령관실로 돌아가기" 텍스트 변경도 문제점으로 지적했지만 변경 후 텍스트가 명시되지 않았습니다.

> **John → Quinn**: Diff 뷰가 기존 기능인데, 이걸 "UI에 노출 안 됨"이라고만 적고 해결 안 하면 v1 기능 누락 아닌가요?
> **Quinn → John**: 동의합니다. 최소한 data-testid에 `debate-diff-view` 같은 걸 추가하고, 어디에 배치할지 (타임라인 내? 별도 탭?) 정의해야 합니다. 테스트 항목에도 Diff 뷰 확인이 필요합니다.

### Bob (SM)
범위는 적절합니다. 5개 컴포넌트의 UI 스타일 변경 + 레이아웃 조정이 현실적입니다. 다만 Diff 뷰를 새로 UI에 노출하는 건 기능 로직 변경 없이 가능한지 확인 필요합니다 — API(`/debates/:id/timeline`)가 이미 라운드별 데이터를 반환한다면 UI-only로 가능합니다.

### Mary (BA)
비즈니스 가치는 명확합니다: AI 에이전트 간 토론을 더 읽기 쉽게 만들어 CEO의 의사결정 지원을 강화합니다. 라운드 진행도 표시와 발언자별 색상 구분은 사용성을 크게 높일 것입니다.
