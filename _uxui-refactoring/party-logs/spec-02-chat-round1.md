# Party Mode Round 1 — Collaborative Lens
## Spec: 02-chat (1:1 에이전트 채팅)
> Date: 2026-03-09

---

### Sally (UX)
대화 흐름 자체는 자연스럽습니다. 세션 목록 -> 세션 선택 -> 메시지 입력 -> 전송, 3클릭 이내로 핵심 동작이 가능합니다. 그러나 **파일 첨부 기능**이 스펙에서 완전히 빠져 있습니다. 실제 코드에는 파일 업로드/첨부/미리보기가 구현되어 있는데, 이걸 Banana2가 모르면 입력 영역 디자인이 불완전해집니다. 또한 **스트리밍 중지 버튼**도 현재 동작하는 기능인데 스펙에 언급이 없습니다.

### Winston (Architect)
컴포넌트 목록(섹션 5)에 ChatPage, SessionPanel, ChatArea, AgentListModal 4개만 나열되어 있는데, 실제로는 **ToolCallCard**, **ReportView**, **ReportDetailModal** 등 하위 컴포넌트가 더 있습니다. 리팩토링 범위를 명확히 하려면 이 하위 컴포넌트들도 나열해야 합니다. 또한 `ConfirmDialog`가 삭제 시 사용되는데, 이것도 scope에 포함시켜야 할지 명시가 필요합니다.

> **Sally -> Winston 크로스토크**: 맞아요, ToolCallCard가 메시지 말풍선 안에 들어가는데, 이걸 Banana2가 모르면 메시지 레이아웃을 잘못 디자인할 수 있어요. 도구 호출 카드의 시각적 처리가 Banana2 프롬프트에 반드시 들어가야 합니다.

### Amelia (Dev)
data-testid 목록이 상당히 잘 정리되어 있지만, 누락된 것들이 있습니다:
- `chat-file-attach-btn` (파일 첨부 버튼)
- `chat-stop-btn` (스트리밍 중지 버튼)
- `chat-retry-btn` (에러 시 재시도 버튼)
- `chat-delegation-panel` (위임 내역 패널)
- `chat-tool-call-card` (도구 호출 카드)
- `session-menu-btn` (세션 ... 메뉴 버튼)
- `session-confirm-delete` (삭제 확인 다이얼로그)
- `session-group-header` (날짜 그룹 헤더)

이것들이 없으면 Playwright 테스트에서 해당 요소를 잡을 수 없습니다.

### Quinn (QA)
**스펙의 가장 큰 빈 곳은 edge case 정의 부족**입니다. 섹션 3에서 문제점으로 "로딩/에러 상태 미정의"를 지적했지만, 정작 개선 방향(섹션 4)에서 구체적 정의가 없습니다. 실제 코드에는 이미 구현되어 있는 상태들:
- 연결 끊김 배너 (WebSocket disconnect)
- 재연결 성공 배너
- 스트리밍 대기 상태 (타이핑 인디케이터)
- 에러 + 재시도 버튼
- 이전 메시지 로드 중 스피너

이것들이 스펙에 명시되어야 합니다.

### John (PM)
**심각한 누락**: 실제 구현된 기능 중 스펙에 빠진 것들이 많습니다:
1. **파일 첨부** (최대 5개, 이미지 미리보기, 다운로드)
2. **토론 명령어** (/토론, /심층토론 → AGORA 연동)
3. **위임 내역 패널** (비서 에이전트의 부서별 위임 현황)
4. **도구 호출 카드** (에이전트가 사용한 도구 표시)
5. **무한 스크롤** (이전 메시지 더 보기)
6. **세션 날짜 그룹핑** (오늘/어제/이번 주/이전)
7. **세션 그룹 접기/펼치기**

특히 파일 첨부와 위임 패널은 핵심 사용 시나리오에 직접 영향을 줍니다.

> **John -> Quinn 크로스토크**: Quinn님이 말씀하신 edge case 외에도, "세션이 0개일 때 빈 상태"는 코드에 이미 있는데 ("대화 내역이 없습니다") 스펙에는 빈 상태가 ChatArea 쪽만 언급되어 있네요. SessionPanel 빈 상태도 추가해야 합니다.

### Bob (SM)
스코프가 UI-only로 제한되어 있어 현실적입니다. 다만, 파일 첨부나 토론 명령어 같은 **기존 기능의 UI 표현**이 빠져 있으면, Banana2가 이것들 없이 디자인하고 나서 나중에 끼워넣으려면 레이아웃이 깨질 수 있습니다. 처음부터 모든 기존 기능을 Banana2에게 알려줘야 재작업을 줄일 수 있습니다.

### Mary (BA)
비즈니스 가치는 명확합니다 — 에이전트와의 1:1 커뮤니케이션은 제품의 핵심 가치입니다. 그러나 비서 에이전트의 위임 기능이 이 페이지에서 시각적으로 어떻게 표현되는지가 누락되면, 사용자가 "지시했는데 어떻게 처리되는지 모르겠다"는 핵심 pain point가 해결되지 않습니다. 위임 체인/병렬 위임 상태 표시는 반드시 스펙에 포함되어야 합니다.
