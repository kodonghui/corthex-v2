## Expert Debate

### 📋 John (PM)
v1-feature-spec 6번 항목의 6가지 핵심 기능(관심종목, 실시간 가격, 차트 모달, AI 채팅, 비교 모드, 포트폴리오)이 섹션 9에서 모두 체크되어 있어 빠진 기능은 없습니다. 핵심 시나리오도 명확하게 4개로 정리되어 있어요. 다만 **종목 검색 후 관심종목에 추가하는 플로우**가 빠져 있습니다. 현재 검색은 있지만 "검색 결과에서 관심종목 추가" 동작이 명시되지 않아 핵심 시나리오가 불완전해 보입니다.

### 🏗️ Winston (Architect)
컴포넌트 5개, 모두 기존 파일 수정이라 범위가 적절합니다. 3패널 그리드 레이아웃은 기존 구조를 유지하면서 스타일만 개선하는 방향이 맞아요. 다만 **태블릿(768px~1439px) 대응에서 "채팅은 오버레이"라는 정의가 모호합니다**. 오버레이가 모달인지, 슬라이드인 패널인지, 어떤 트리거로 열고 닫는지 구체적이지 않아서 구현 시 해석이 분분할 수 있어요. 컴포넌트 수준에서 ChatPanel의 오버레이 모드 variant가 필요한 건지도 불명확합니다.

### 🎨 Sally (UX)
3패널 워크스테이션 구조에서 사용자 플로우가 자연스럽습니다. 종목 선택 → 차트 확인이 1클릭, AI 채팅은 이미 오른쪽에 보이니 0클릭, 비교 모드 토글도 1클릭이에요. **그런데 에러 상태 UI가 전혀 정의되지 않았습니다.** 섹션 3에서 "로딩/에러 상태 미정의"를 문제로 지적했으면서, 개선 방향(섹션 4)에서 에러 상태 해결책이 없어요. 시세 API 실패 시, 채팅 연결 실패 시의 표시가 필요합니다.

### 💻 Amelia (Dev)
data-testid 14개로 주요 인터랙션 요소는 커버되어 있습니다. 하지만 **`trading-error-state` (에러 상태), `stock-add` (관심종목 추가 버튼), `chart-period-selector` (차트 기간 선택)가 빠져 있습니다**. 특히 차트 기간 선택은 Banana2 프롬프트에서 "time period selector"로 명시했는데 testid가 없어요. 또한 `portfolio-summary` (포트폴리오 요약) testid도 있어야 포트폴리오 데이터 표시 여부를 테스트할 수 있습니다.

### 🧪 Quinn (QA)
테스트 항목 8개는 happy path 위주입니다. **에러 상태 테스트가 아예 없어요**: (1) 시세 데이터 로딩 실패 시 에러 UI 표시, (2) 종목 0개일 때 빈 상태 표시, (3) 비교 모드에서 종목 1개만 선택 시 동작. 또한 드래그 정렬 테스트도 빠져 있고, 포트폴리오 표시 확인 테스트도 없습니다. 현재 8개로는 v1 기능 6개를 모두 검증하기 어렵습니다.

### 🏃 Bob (SM)
범위: 5개 컴포넌트, 모두 기존 파일 수정, "절대 건드리면 안 되는 것" 리스트가 명확해서 UI-only 경계가 잘 잡혀 있습니다. 비교 모드 토글 버튼 추가는 새 UI 요소이지만 기존 URL searchParams 로직을 활용하므로 범위 안이에요. **태블릿 오버레이 채팅은 구현 복잡도가 좀 있지만**, 기존 ChatPanel 컴포넌트를 래핑하는 정도라면 괜찮겠습니다.

### 📊 Mary (BA)
트레이딩 워크스테이션은 CORTHEX의 "AI가 투자도 해준다"는 차별화 포인트의 핵심 페이지입니다. 실시간 가격 + AI 분석 채팅의 조합이 비즈니스 가치를 잘 전달해요. Banana2 프롬프트에서 "TradingView meets AI chat"이라는 톤이 제품 포지셔닝에 딱 맞습니다. 설명서가 이 가치를 보존하면서 비주얼만 개선하는 방향이라 안심됩니다.
