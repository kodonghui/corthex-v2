## 리뷰 결과

### John (PM)
v1-feature-spec 21번 항목의 에이전트별/모델별 비용 추적, 일일 비용 추이, 도넛 차트, 예산 사용률, 기간 선택이 모두 스펙에 반영되어 있다. 섹션 9의 체크리스트가 v1 기능을 빠짐없이 커버한다. 다만 v1에서 "부서별 도넛 차트"였던 것이 v2에서 "프로바이더별 도넛 차트"로 변경되었는데, bySource 데이터가 프로바이더 매핑에 사용되고 있으므로 일관성이 있다.

### Winston (Architect)
컴포넌트가 모두 pages/costs.tsx 인라인으로 되어 있는데, 도넛 차트나 막대 차트 같은 시각화 컴포넌트는 재사용 가능성이 있으므로 별도 파일 분리를 검토할 만하다. 데이터 바인딩에서 4개 API 엔드포인트가 명확히 정의되어 있고, "UI 변경 시 절대 건드리면 안 되는 것"이 구체적이어서 안전한 리팩토링이 가능하다.

### Sally (UX)
비용 확인 흐름이 "/costs 접속 -> 기간 선택 -> 차트 확인"으로 2클릭 이내에 가능하다. 문제점 #4에서 뒤로가기 버튼 동선 문제를 지적하고, 컴포넌트 목록에서 "뒤로가기 제거(사이드바로 충분)"로 해결했는데, Playwright 테스트 #17에 "뒤로가기 -> /dashboard로 이동" 테스트가 여전히 남아 있어 모순이다. 또한 모바일에서 커스텀 기간 date input이 3줄째로 이동한다는 설명만 있고 구체적인 터치 UX가 부족하다.

### Amelia (Dev)
data-testid가 27개로 풍부하게 정의되어 있다. 모델별 비용 영역(`costs-model-breakdown`, `costs-model-item`)과 에이전트 링크(`costs-agent-link`)도 포함되어 있어 개선사항에 맞는 테스트가 가능하다. 다만 에러 상태 testid(`costs-error-state`)는 있지만 Playwright 테스트 항목에 에러 상태 테스트(API 실패 시나리오)가 누락되어 있다.

### Quinn (QA)
빈 상태/에러 상태/로딩 상태 모두 testid가 정의되어 있어 좋다. 그러나 Playwright 테스트 #17 "뒤로가기 -> /dashboard로 이동"은 컴포넌트 목록에서 뒤로가기를 제거한다고 했으므로 삭제하거나 수정해야 한다. 또한 예산 미설정 시의 엣지케이스가 문제점 #5에서 언급되었지만 테스트 항목에 없다.

### Bob (SM)
UI만 변경 범위가 명확하다. 4개 API 호출 로직, PROVIDER_COLORS 상수, microToUsd/formatNumber 헬퍼, 프로바이더 판별 로직을 보호한다. 모델별 브레이크다운 추가는 기존 byModel 데이터를 활용하는 것이므로 API 변경 없이 가능하여 리스크가 낮다.

### Mary (BA)
비용 분석 대시보드는 CEO가 AI 운영 비용을 파악하는 핵심 경영 도구다. 프로바이더별 비용 비율 시각화는 멀티 프로바이더 전략의 비용 효율성을 판단하는 데 직접 기여한다. 에이전트별 비용 순위는 비용 최적화 대상을 식별하는 데 실질적 가치가 있다.
