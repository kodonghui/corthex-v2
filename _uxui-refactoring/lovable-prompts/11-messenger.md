# 11. 사내 메신저 페이지

> Route: `/messenger` — CEO App

---

## What This Page Is For

회사 내 인간 직원과 AI 에이전트가 함께 소통하는 실시간 채널 기반 메신저입니다. 채널을 만들어 멤버를 초대하고 메시지를 주고받습니다. AI 에이전트를 @멘션하면 해당 에이전트가 채널 컨텍스트를 읽고 자동으로 응답합니다. 스레드(답글), 리액션(이모지), 파일 첨부, 메시지 검색을 지원합니다.

---

## Data Displayed

### 채널 목록 (사이드바)
- 내가 참여한 채널들: **채널명**, **마지막 메시지** (내용, 보낸 사람 이름, 시간)
- **채널별 미읽음 메시지 수** (unread count — lastReadAt 이후 새 메시지 수)
- 채널 선택 시 상세 정보: **설명**, **멤버 수**

### 메시지 영역 (선택된 채널)
- 각 메시지: **보낸 사람 이름** (userName), **내용** (content), **보낸 시간** (createdAt)
- **답글 수** (replyCount) — 0이 아니면 스레드 열기 가능
- **리액션** — 이모지별 개수와 누가 눌렀는지(userIds)
- **첨부파일** — 파일명, MIME 타입, 크기 (attachments 배열)
- 최대 50~100건씩 로드 (페이지네이션 또는 무한 스크롤)

### 스레드 패널
- 원본 메시지 + 하위 답글 목록
- 각 답글도 리액션, 첨부파일 포함

### 온라인 상태
- 같은 회사의 현재 온라인 유저 ID 목록 (WebSocket 연결 기준)

### 채널 멤버 목록
- 멤버별: **이름**, **역할** (admin/user), **참여 시간** (joinedAt)

### 회사 유저 목록 (멤버 추가용)
- 같은 회사의 활성 유저 목록: **이름**, **역할** (멤버 추가 시 자동완성/검색 데이터 소스)

## User Actions

### 채널 관리
1. **채널 생성** — 채널명(필수), 설명(선택), 초기 멤버 선택(선택). 생성자는 자동으로 멤버에 추가됨
2. **채널 수정** — 채널명, 설명 변경 (멤버만 가능)
3. **채널 삭제** — 생성자만 가능. 관련 리액션, 메시지, 멤버 모두 함께 삭제됨
4. **멤버 추가** — 같은 회사의 유저 검색 후 채널에 추가
5. **멤버 제거** — 특정 멤버를 채널에서 내보내기
6. **채널 나가기** — 본인이 채널에서 탈퇴

### 메시지
7. **메시지 보내기** — 텍스트 입력(최대 4000자), 파일 첨부(최대 5개), 답글로 보내기(parentMessageId)
8. **@멘션** — `@에이전트이름`으로 AI 에이전트 호출 → 에이전트가 최근 10건 대화 컨텍스트로 자동 응답. `@유저이름`으로 유저 멘션 → 해당 유저에게 알림 생성
9. **리액션 추가** — 메시지에 이모지 리액션 (같은 이모지 중복 불가)
10. **리액션 제거** — 본인이 추가한 리액션 취소
11. **스레드 열기** — 메시지의 답글 목록 조회 및 답글 작성

### 검색
12. **메시지 검색** — 키워드로 내가 참여한 채널의 메시지 검색 (최소 2글자). 결과에 채널명, 보낸 사람, 내용, 시간 표시

### 읽음 처리
13. **채널 읽음 처리** — 채널 진입 시 자동으로 lastReadAt 업데이트
14. **미읽음 확인** — 각 채널의 미읽음 메시지 수 실시간 표시

## UX Considerations

- **실시간 업데이트 필수**: WebSocket `messenger` 채널을 통해 새 메시지, 타이핑 인디케이터, 리액션 변경이 즉시 반영되어야 합니다.
- AI 에이전트가 응답을 생성하는 동안 **타이핑 인디케이터**가 표시되어야 합니다 (WebSocket typing 이벤트).
- 채널 목록은 항상 접근 가능해야 하고, 미읽음 배지가 눈에 띄어야 합니다.
- 스레드(답글)는 메인 대화 흐름을 방해하지 않으면서도 쉽게 접근할 수 있어야 합니다.
- @멘션 입력 시 자동 완성(에이전트 + 유저 이름)이 필요합니다.
- 파일 첨부 시 MIME 타입에 따라 이미지는 인라인 미리보기, 그 외는 파일 아이콘+이름으로 표시하세요.
- 오프라인 유저에게는 Web Push 알림이 전송되므로, 온라인/오프라인 상태 표시가 유용합니다.
- URL 파라미터 `?channelId=xxx`로 특정 채널 직접 진입을 지원해야 합니다 (알림 클릭 시 사용됨).
- 메시지 로딩은 최신순으로 가져온 후 시간순으로 뒤집어서 표시합니다 (오래된 것이 위).
- 빈 상태: 채널이 없을 때, 메시지가 없을 때, 검색 결과가 없을 때 각각 안내 필요.

## What NOT to Include

- 색상, 폰트, 크기, px/rem 등 시각적 스타일 지정
- 특정 컴포넌트 라이브러리나 UI 프레임워크 언급
- 레이아웃 배치 지정
- Web Push 구독 로직 (이미 서버에서 처리)
- AI 응답 생성 내부 로직 (서버에서 자동 처리)
