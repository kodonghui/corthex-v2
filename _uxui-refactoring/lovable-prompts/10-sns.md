# 10. SNS 퍼블리싱 페이지

> Route: `/sns` — CEO App

---

## What This Page Is For

AI 에이전트가 생성하거나 사용자가 직접 작성한 SNS 콘텐츠를 관리하는 페이지입니다. 콘텐츠 초안 작성부터 승인, 예약 발행, 실제 발행까지의 전체 워크플로우를 한 곳에서 처리합니다. 6개 플랫폼(인스타그램, 티스토리, 다음 카페, 트위터/X, 페이스북, 네이버 블로그)을 지원하며, A/B 테스트 변형 생성과 카드뉴스 시리즈도 관리합니다.

---

## Data Displayed

### 콘텐츠 목록
- 각 콘텐츠의 **제목**, **플랫폼** (instagram/tistory/daum_cafe/twitter/facebook/naver_blog), **상태** (draft/pending/approved/scheduled/rejected/published/failed)
- **연결된 SNS 계정명** (snsAccountId → accountName)
- **작성자 이름** (createdBy → users.name)
- **발행 URL** (published 상태일 때 publishedUrl)
- **예약 시간** (scheduledAt, 있을 경우)
- **A/B 변형 여부** (variantOf 필드 — null이면 원본, 값이 있으면 변형)
- **생성일/수정일**

### 통계 섹션
- **총 콘텐츠 수** (기간 내)
- **상태별 분포** (draft/pending/approved/scheduled/rejected/published/failed 각 몇 건)
- **플랫폼별 분포** (총 건수 + 발행 완료 건수)
- **일별 생성 추이** (날짜별 콘텐츠 생성 건수 차트)
- 조회 기간 선택 가능 (기본 30일, 1~365일)

### 콘텐츠 상세 (개별 항목 선택 시)
- **제목, 본문 전체, 해시태그, 이미지** (imageUrl)
- **상태 이력** — 현재 상태 + 반려 사유(rejectReason, rejected일 때)
- **카드뉴스 정보** (isCardNews=true일 때: cardSeriesId, cardIndex)
- **A/B 변형 목록** (variantOf로 연결된 콘텐츠들)
- **메타데이터** (metadata) — 추가 정보 (있을 경우)

### SNS 계정 관리 (서브 섹션 또는 설정 영역)
- 등록된 SNS 계정 목록: **플랫폼, 계정명, 계정 ID, 활성 여부**
- 연결된 콘텐츠 수 (삭제 시 연결 콘텐츠 있으면 차단됨)

## User Actions

### 콘텐츠 관리
1. **수동 콘텐츠 작성** — 플랫폼 선택, 제목, 본문, 해시태그, 이미지 URL 입력, SNS 계정 연결(선택), 예약 시간 설정(선택)
2. **AI 콘텐츠 생성** — 플랫폼 + 에이전트 + 주제를 입력하면 AI가 제목/본문/해시태그를 자동 생성
3. **AI 콘텐츠+이미지 동시 생성** — 위와 동일 + 이미지 프롬프트(선택)로 텍스트와 이미지를 함께 생성
4. **콘텐츠 수정** — 제목, 본문, 해시태그, 이미지 URL, 예약 시간 변경
5. **콘텐츠 삭제**

### 승인 워크플로우
6. **승인 요청** (draft → pending) — CEO 승인 대기로 전환
7. **승인** (pending → approved) — CEO만 가능
8. **반려** (pending → rejected) — 사유 입력 필수, CEO만 가능
9. **발행** (approved → published/failed) — 실제 플랫폼에 게시 실행
10. **발행 재시도** (failed 상태) — 실패한 발행을 재시도
11. **발행 결과 확인** — 특정 콘텐츠의 발행 상태/URL/에러 조회

### A/B 테스트
12. **수동 변형 생성** — 원본 콘텐츠의 제목/본문/해시태그/이미지를 변경한 변형 추가
13. **AI 변형 자동 생성** — 에이전트가 전략(tone/length/hashtag/headline/mixed)에 따라 2~5개 변형 생성
14. **A/B 결과 비교** — 원본+변형의 성과 지표(views, likes, shares, clicks) 비교, 우승 콘텐츠 확인

### 성과 지표
15. **수동 지표 입력** — 조회수, 좋아요, 공유, 클릭 수 입력

### 이미지
16. **이미지 생성** — 기존 콘텐츠에 이미지 프롬프트로 이미지 추가 생성

### 계정 관리
17. **SNS 계정 등록** — 플랫폼(instagram/tistory/daum_cafe), 계정명, 계정 ID, 인증 정보(선택)
18. **SNS 계정 수정** — 계정명, 계정 ID, 인증 정보, 활성 상태 변경
19. **SNS 계정 삭제** — 연결된 콘텐츠가 없을 때만 가능

### 필터링
20. **플랫폼별 필터**, **상태별 필터**, **계정별 필터**, **원본/변형 필터** (variantOf=root이면 원본만)

## UX Considerations

- 콘텐츠 상태 전환(draft → pending → approved → published)이 직관적인 워크플로우로 보여야 합니다. 현재 상태에서 가능한 다음 액션만 제공하세요.
- 6개 플랫폼을 시각적으로 구분해야 합니다 (아이콘 또는 레이블).
- AI 생성은 시간이 걸리므로 로딩 인디케이터가 필요합니다.
- 반려된 콘텐츠는 반려 사유가 바로 보여야 합니다.
- A/B 변형은 원본과 묶여서 표시되어야 합니다 (별도 목록이 아닌 그룹).
- 통계 섹션은 접었다 펼칠 수 있어야 합니다.
- 발행 실패 시 에러 내용과 재시도 버튼이 눈에 띄어야 합니다.
- 카드뉴스 시리즈는 순서(cardIndex)가 보여야 합니다.
- 계정 관리는 콘텐츠 관리와 분리된 탭이나 섹션으로 배치하세요.

## What NOT to Include

- 색상, 폰트, 크기, px/rem 등 시각적 스타일 지정
- 특정 컴포넌트 라이브러리나 UI 프레임워크 언급
- 레이아웃 배치 (좌/우, 그리드 등) 지정
- 실제 Selenium 자동화 로직이나 발행 엔진 내부 구현
- 인증 정보(credentials)의 실제 값 표시 — 암호화되어 있으므로 표시 불가
