# 13. 보고서 페이지

> Route: `/reports` — CEO App

---

## What This Page Is For

인간 직원(Employee)이 CEO에게 보고서를 작성하여 제출하고, CEO가 이를 검토하며 코멘트를 주고받는 페이지입니다. 보고서는 초안(draft) → 제출(submitted) → 검토완료(reviewed) 워크플로우를 따릅니다. 양방향 코멘트로 피드백 교환이 가능하고, 마크다운 형식의 보고서 다운로드도 지원합니다.

---

## Data Displayed

### 보고서 목록
- **제목** (title, 최대 200자)
- **상태** (status) — draft(초안), submitted(제출됨), reviewed(검토 완료)
- **작성자 이름** (authorName — users.name JOIN)
- **보고 대상** (submittedTo — CEO의 userId, 제출 후에만)
- **제출 시간** (submittedAt)
- **생성일** (createdAt), **수정일** (updatedAt)
- 목록은 수정일 내림차순으로 정렬됨
- 본인이 작성한 보고서 + 자신에게 제출된 보고서 모두 표시됨

### 보고서 상세 (개별 항목 선택 시)
- **제목**, **본문 전체** (content — 텍스트, 마크다운 가능)
- **상태**, **작성자**, **제출 대상**, **제출 시간**
- **생성일**, **수정일**

### 코멘트 섹션 (제출된 보고서에만)
- **코멘트 목록**: 작성자 이름(authorName), 내용(content), 작성 시간(createdAt)
- **총 코멘트 수** (total)
- 시간순 정렬 (오래된 것이 위)
- 커서 기반 페이지네이션 (before 파라미터)

---

## User Actions

### 보고서 작성 (Employee 관점)
1. **새 보고서 작성** — 제목(필수), 내용(선택)으로 초안 생성
2. **보고서 수정** — 초안(draft) 상태에서만 제목, 내용 수정 가능
3. **보고서 제출** — 초안을 CEO에게 제출 (draft → submitted). 같은 회사의 admin 역할 유저에게 자동 할당됨. 제출 시 CEO에게 알림 전송
4. **보고서 삭제** — 초안 상태에서만 본인 작성 보고서 삭제 가능 (코멘트도 함께 삭제)

### 보고서 검토 (CEO 관점)
5. **검토 완료 마킹** — 제출된 보고서를 검토 완료로 표시 (submitted → reviewed). 보고 대상(submittedTo)만 가능. 작성자에게 검토 완료 알림 전송

### 코멘트
6. **코멘트 작성** — 제출된 보고서(submitted 또는 reviewed)에만 코멘트 가능. 작성자 또는 보고 대상만 코멘트 가능. 상대방에게 알림 전송
7. **코멘트 목록 조회** — 페이지네이션으로 코멘트 로드 (최대 100건씩)

### 다운로드
8. **보고서 다운로드** — 제목+내용을 마크다운(.md) 파일로 다운로드. 작성자 또는 보고 대상만 다운로드 가능

## UX Considerations

- 보고서 상태에 따라 가능한 액션이 다릅니다:
  - **draft**: 수정, 삭제, 제출 가능 / 코멘트 불가
  - **submitted**: 검토 완료 가능(CEO만), 코멘트 가능 / 수정·삭제 불가
  - **reviewed**: 코멘트 가능 / 수정·삭제·검토 불가
- CEO와 Employee가 같은 페이지를 사용하므로, 역할에 따라 보이는 보고서가 다릅니다:
  - Employee: 자기가 쓴 보고서 + 자기에게 온 보고서
  - CEO: 자기가 쓴 보고서 + 자기에게 제출된 보고서
- 코멘트는 실시간이 아닌 새로고침 방식입니다 (WebSocket 미사용).
- "내가 쓴 보고서"와 "나에게 온 보고서"를 구분할 수 있어야 합니다 (탭이나 필터).
- 본문 내용이 마크다운일 수 있으므로 마크다운 렌더링을 고려하세요.
- 빈 상태: 보고서가 없을 때, 코멘트가 없을 때 안내 필요.

## What NOT to Include

- 색상, 폰트, 크기, px/rem 등 시각적 스타일 지정
- 특정 컴포넌트 라이브러리나 UI 프레임워크 언급
- 레이아웃 배치 지정
- 보고서 서식 템플릿 (현재 v2에 없음)
- 보고 대상 선택 기능 (현재 자동으로 같은 회사 admin에게 할당)
