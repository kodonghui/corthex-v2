# 14. 작업 관리 (Jobs) 페이지

> Route: `/jobs` — CEO App

---

## What This Page Is For

AI 에이전트에게 지시하는 비동기 작업(야간 작업)을 관리하는 페이지입니다. 단건 작업, 체인 작업(2~5단계 순차 실행), 반복 스케줄(cron), 이벤트 트리거(가격 변동, 시장 개장/폐장)를 한 곳에서 관리합니다. 작업 결과를 아침에 확인할 수 있는 알림 기능도 포함됩니다.

---

## Data Displayed

### 작업 목록 (단건 + 체인)
- **에이전트 이름** (agentName)
- **지시 내용** (instruction)
- **상태** (status) — queued(대기), processing(처리 중), completed(완료), failed(실패), blocked(체인 대기)
- **결과** (result) — 완료 시 텍스트 결과
- **구조화된 결과** (resultData) — JSON 형식의 상세 실행 결과
- **에러** (error) — 실패 시 에러 메시지
- **재시도 횟수** (retryCount) / **최대 재시도** (maxRetries, 기본 3)
- **예약 시간** (scheduledFor)
- **시작 시간** (startedAt), **완료 시간** (completedAt)
- **읽음 여부** (isRead)
- **체인 정보** — parentJobId(이전 단계), chainId(같은 체인 그룹)
- **생성일** (createdAt)
- 최신 50건, 생성일 내림차순
- scheduleId/triggerId가 null인 것만 (직접 등록한 작업만)

### 아침 알림 섹션
- **읽지 않은 완료/실패 작업** 수
- 각 항목: 에이전트 이름, 지시 내용, 상태, 결과/에러, 구조화된 결과, 완료 시간

### 반복 스케줄 목록
- **스케줄 이름** (name)
- **에이전트 이름** (agentName)
- **지시 내용** (instruction)
- **cron 표현식** (cronExpression) + **사람이 읽을 수 있는 설명** (description, 예: "매일 09:00")
- **다음 실행 시간** (nextRunAt), **마지막 실행 시간** (lastRunAt)
- **활성 상태** (isActive)
- **생성일/수정일**

### 스케줄 실행 기록
- 특정 스케줄의 실행 이력: **상태** (running/success/failed), **시작 시간**, **종료 시간**, **결과**
- 페이지네이션 (page, limit, total, totalPages)
- 상태별 필터 가능

### 이벤트 트리거 목록
- **트리거 이름** (name, 선택적 — null일 수 있음)
- **에이전트 이름** (agentName)
- **지시 내용** (instruction)
- **트리거 유형** (triggerType) — price-above(가격 상회), price-below(가격 하회), market-open(시장 개장), market-close(시장 폐장)
- **조건** (condition) — 가격 트리거 시 종목코드(stockCode)+목표가격(targetPrice)
- **활성 상태** (isActive)
- **마지막 발동 시간** (lastTriggeredAt)
- **생성일**

## User Actions

### 단건 작업
1. **작업 등록** — 에이전트 선택, 지시 내용 입력, 예약 시간(선택). 에이전트는 같은 회사 소속이어야 함
2. **작업 취소** — queued 상태인 작업만 삭제 가능
3. **읽음 처리** — 개별 작업을 읽음 표시
4. **전체 읽음 처리** — 모든 미읽음 작업을 한번에 읽음 처리

### 체인 작업
5. **체인 작업 등록** — 2~5단계의 순차 실행 작업 일괄 등록. 각 단계별 에이전트+지시 입력. 첫 단계만 queued, 나머지는 blocked 상태
6. **체인 전체 취소** — 처리 중(processing)인 단계가 없으면 체인 내 queued/blocked 작업 전체 삭제

### 반복 스케줄
7. **스케줄 생성** — 이름, 에이전트, 지시 내용, cron 표현식(직접 입력) 또는 주기(daily/weekdays/custom)+시간+요일 입력
8. **스케줄 수정** — 이름, 지시, 에이전트, cron 표현식 또는 주기/시간/요일 변경
9. **스케줄 활성/비활성 토글** — isActive on/off. 활성화 시 nextRunAt 자동 계산, 비활성화 시 nextRunAt null
10. **스케줄 삭제**
11. **실행 기록 조회** — 특정 스케줄의 과거 실행 이력 페이지네이션 조회 + 상태 필터
12. **단일 실행 상세** — 특정 실행 기록의 상세 정보

### 이벤트 트리거
13. **트리거 생성** — 에이전트 선택, 지시 내용, 트리거 유형 선택, 조건 입력(가격 트리거 시 종목코드+목표가격 필수)
14. **트리거 수정** — 지시 내용, 트리거 유형, 조건 변경
15. **트리거 활성/비활성 토글**
16. **트리거 삭제**

## UX Considerations

- 이 페이지는 3가지 관련 기능(단건/체인 작업, 반복 스케줄, 이벤트 트리거)을 통합합니다. 탭이나 구분된 섹션으로 나누어야 합니다.
- **아침 알림**은 페이지 진입 시 눈에 띄게 표시되어야 합니다 (밤사이 완료된 작업 확인 용도).
- 체인 작업은 단계별 연결 관계(1→2→3)가 시각적으로 보여야 합니다. blocked 상태는 "이전 단계 대기 중"임을 표시하세요.
- 작업 상태별 시각적 구분이 필요합니다 (queued/processing/completed/failed/blocked).
- cron 표현식은 비개발자에게 이해하기 어려우므로, 사람이 읽을 수 있는 설명(description)을 함께 표시하세요. 입력 시에도 주기(매일/평일/커스텀)+시간 방식의 간편 UI를 제공하세요.
- 이벤트 트리거의 가격 조건은 종목코드와 목표가격을 명확하게 입력받아야 합니다.
- WebSocket `night-job` 채널을 통해 작업 완료/실패가 실시간으로 반영됩니다.
- 빈 상태: 작업 없을 때, 스케줄 없을 때, 트리거 없을 때, 알림 없을 때 각각 안내 필요.
- resultData(구조화된 결과)가 있으면 원문(result)보다 보기 좋게 정리해서 보여줄 수 있습니다.

## What NOT to Include

- 색상, 폰트, 크기, px/rem 등 시각적 스타일 지정
- 특정 컴포넌트 라이브러리나 UI 프레임워크 언급
- 레이아웃 배치 지정
- 작업 실행 엔진 내부 로직 (job-queue, cron-runner 등)
- ARGOS 시스템의 상세 모니터링 (별도 페이지)
