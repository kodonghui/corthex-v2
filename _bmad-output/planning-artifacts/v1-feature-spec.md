# CORTHEX v1 기능 명세서 (v2 이식 기준 문서)

> 이 문서는 v1에서 **실제로 동작했던** 기능을 정리한 것이다.
> v2의 모든 기획/설계/구현 단계에서 이 문서를 참조하여,
> "v1에서 되던 건 v2에서도 반드시 된다"를 보장해야 한다.
> CRUD 페이지가 아니라 **실제 동작하는 기능**이 핵심이다.

---

## 1. 사령관실 (Command Center)

v1에서 CEO가 AI에게 명령하는 **유일한 진입점**.

### 1.1 명령 입력 방식
- **일반 텍스트**: "삼성전자 분석해줘" → 비서실장이 자동 라우팅
- **@멘션**: "@투자분석팀장 삼성전자 분석" → 특정 팀장 직접 지정
- **슬래시 명령어** (8종):
  - `/전체` — 6명 팀장에게 동시 지시
  - `/순차` — 에이전트 릴레이 모드
  - `/도구점검` — 전체 도구 상태 확인
  - `/배치실행` — 대기 중인 AI 요청 일괄 전송
  - `/배치상태` — 배치 작업 진행 확인
  - `/명령어` — 전체 명령어 목록
  - `/토론` — 6명 팀장 임원 토론 (2라운드)
  - `/심층토론` — 6명 팀장 심층 토론 (3라운드)
- **프리셋**: CEO가 자주 쓰는 명령을 단축어로 저장 (예: "시황" → "현재 주식시장 시황을 분석해줘")

### 1.2 명령 처리 흐름 (핵심 — 이것이 v2에 없는 것)
```
CEO 명령 입력
  → Orchestrator.process_command()
    → 비서실장(Chief of Staff)에게 TaskRequest 전송
      → 비서실장 AI가 명령 분류 (어느 부서?)
        → 해당 팀장(Manager)에게 위임
          → 팀장이 부하 전문가들에게 병렬 배분
            → 전문가들이 도구 사용하며 자율 작업
          → 팀장이 결과 종합
        → 비서실장이 최종 정리 (CEO 눈높이로)
      → CEO에게 보고서 반환
```

### 1.3 실시간 상태 표시
- 에이전트 작업 중: 이름 + 경과 시간 표시
- 위임 체인: "비서실장 → CMO → 콘텐츠 전문가" 실시간 추적
- SSE(Server-Sent Events)로 스트리밍

## 2. 에이전트 조직 시스템

### 2.1 조직 구조 (29명)
```
CEO (사용자)
├── 비서실장 (Chief of Staff) + 3명 보좌관
└── 2개 본부:
    ├── LEET MASTER 본부
    │   ├── CTO (5명): 프론트엔드, 백엔드, 인프라, AI모델
    │   ├── CSO (3명): 시장조사, 사업계획서, 재무모델링
    │   ├── CLO (2명): 저작권, 특허/약관
    │   └── CMO (3명): 설문/리서치, 콘텐츠, 커뮤니티
    ├── 투자분석 본부
    │   └── CIO (5명): 시황분석, 종목분석, 기술적분석, 리스크관리
    └── 출판 기록 본부
        └── CPO (3명)
```

### 2.2 에이전트 3계급
| 계급 | 역할 | AI 모델 | 비용 |
|------|------|--------|------|
| Manager (처장) | 업무 분해 + 배분 + 종합 | Claude Sonnet 4.6 | 중 |
| Specialist (전문가) | 실제 작업 수행 + 도구 사용 | Claude Haiku 4.5 | 저 |
| Worker (워커) | 단순 반복 (요약, 일정) | Claude Haiku 4.5 | 최저 |

### 2.3 에이전트 성격 (Soul) 시스템
- 각 에이전트는 **마크다운 파일**로 성격 정의
- 포함 내용: 역할, 전문분야, 판단 원칙, 보고서 형식, 사용 도구 목록
- **웹 UI에서 CEO가 직접 편집 가능** (서버 재시작 불필요)
- 예: 투자분석처장(CIO) 소울 = 16KB 상세 문서 (MPT, Kelly Criterion 등)

### 2.4 자율 딥워크
에이전트가 1번 답하는 게 아니라 자율적으로 다단계 작업:
1. 계획 수립
2. 데이터 수집 (도구 사용)
3. 분석
4. 초안 작성
5. 최종 보고서 생성

## 3. 도구 시스템 (125개+)

### 3.1 도구 카테고리
| 카테고리 | 주요 도구 | 설명 |
|---------|---------|------|
| 금융 | kr_stock, dart_api, sec_edgar, backtest_engine | 한/미 주식, 공시, 백테스트 |
| 법무 | law_search, contract_reviewer, trademark_similarity | 법률 검색, 계약 검토 |
| 마케팅 | sns_manager, seo_analyzer, hashtag_recommender | SNS, SEO, 감정 분석 |
| 기술 | uptime_monitor, security_scanner, code_quality | 모니터링, 보안 |
| 공통 | real_web_search, spreadsheet_tool, chart_generator, email_sender | 검색, 차트, 이메일 |

### 3.2 도구 권한 제어
- 모든 에이전트가 모든 도구를 쓸 수 있는 게 아님
- `agents.yaml`에서 `allowed_tools` 필드로 에이전트별 허용 도구 지정
- 예: CIO만 kr_stock 사용 가능, CMO만 sns_manager 사용 가능

### 3.3 도구 실행 방식
- 에이전트가 AI 응답 중 도구 호출 → ToolPool이 실행 → 결과를 AI에 반환
- 결과 최대 4,000자 (초과 시 요약)
- 도구 호출 로그 전부 기록 (UI에서 확인 가능)

## 7. 스케치바이브 (NEXUS SketchVibe)

### 7.1 핵심 개념
**"그림 그려서 AI랑 같이 보면서 대화하는 것"**

### 7.2 캔버스 기능
- Cytoscape.js 엔진 (Mermaid에서 전환)
- 8종 노드: agent, system, api, decide, db, start, end, note
- 각 노드 고유 시각 스타일 (색상, 모양)
- 드래그 이동, 더블클릭 이름 편집, Delete 삭제
- 연결 모드: Space바 토글 + Ctrl+클릭 멀티선택
- edgehandles: 드래그로 화살표 생성
- compound parent: subgraph 그룹핑

### 7.3 Mermaid ↔ Cytoscape 양방향 변환
- AI가 Mermaid로 그려주면 → 캔버스에 자동 표시
- 캔버스 수정하면 → Mermaid 코드로 자동 변환 (AI가 읽을 수 있게)
- classDef, style 지시어로 커스텀 색상
- subgraph → compound parent 자동 변환

### 7.4 MCP SSE 연동
- AI가 실시간으로 캔버스에 노드 추가/수정
- `_connectSketchVibeSSE()` → SSE 구독
- read_canvas() → AI가 현재 캔버스 상태를 Mermaid로 읽기

### 7.5 저장/불러오기
- 확인된 다이어그램 저장 (`/api/sketchvibe/confirmed`)
- 지식 베이스에서 flowchart 불러오기
