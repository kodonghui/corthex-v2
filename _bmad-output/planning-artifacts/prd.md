## 1. Project Discovery

### Project Classification

- **Project Type:** SaaS B2B Platform (Multi-tenant web application with real-time AI agent orchestration)
- **Domain:** Fintech + AI Agent Platform Hybrid (투자 자동매매 + AI 에이전트 조직 운영 + 125+ 도구 통합)
- **Complexity:** High (동적 조직 관리, 멀티테넌시 데이터 격리, 실시간 WebSocket 스트리밍, 다중 LLM 프로바이더 라우팅, KIS 증권 API 연동, Selenium 자동화)

## Executive Summary

CORTHEX v2는 **AI 에이전트 조직을 설계하고 운영하는 SaaS 플랫폼**이다. CEO(사용자)가 자연어 명령 한 줄을 입력하면, AI 비서실장이 명령을 자동 분류하고, 적합한 부서에 위임하며, 전문가 에이전트들이 병렬로 작업한 뒤, 결과를 종합하여 CEO 눈높이의 보고서를 반환한다.

**타겟 사용자**: 1인~중소기업의 대표/경영진/관리자. 개발 지식 없이 AI를 "조직"으로 운영하여 업무 자동화, 투자 분석, 콘텐츠 발행, 정보 수집을 수행하려는 비개발자 리더. 3명의 핵심 페르소나 -- 1인 사업가(김대표, 투자+마케팅+법무+전략 조직 직접 설계), 중소기업 관리자(박과장, 팀 표준화+비용통제+권한관리), 개인 투자자(이사장, 전문 분석+자동매매)가 각각 다른 조직 구성으로 같은 플랫폼을 사용한다.

**핵심 문제**: "여러 AI 에이전트가 조직 구조 안에서 자율적으로 협업하고, 비개발자가 그 조직을 자유롭게 설계/관리하는" 플랫폼이 시장에 존재하지 않는다. 1:1 대화형 AI(ChatGPT/Claude)는 다부서 병렬 작업 불가. 자동화 플랫폼(Zapier/Make)은 AI 자율 판단 불가. 멀티에이전트 프레임워크(AutoGPT/CrewAI)는 프로덕션 UI 부재. v1 CORTHEX는 29명 고정 조직으로 유연성 부족.

**v1 검증 기반**: v1은 22개 기능 영역(오케스트레이션, 125+ 도구 시스템, LLM 3사 라우팅, AGORA 토론, KIS 자동매매, SketchVibe 캔버스, SNS 자동 발행, 품질 게이트, 작전현황 대시보드, 통신로그, 작전일지, 기밀문서, 전력분석, 자동화 워크플로우 등)을 실제 운영 환경에서 동작 검증했다.

**v2 핵심 진화 -- 동적 조직 관리**: 29명 고정 조직에서 탈피하여 관리자가 웹 UI에서 부서/AI직원/Human직원을 자유롭게 CRUD한다. 시스템 에이전트(비서실장 등 오케스트레이션 필수 역할)는 보호, 사용자 에이전트는 자유 CRUD. 부서 삭제 시 cascade 처리(진행 중 작업 대기, 메모리 아카이브, 비용 기록 영구 보존, 하위 에이전트 미배속 전환). 조직 템플릿으로 빠른 시작. 멀티테넌시(companyId 기반 데이터 격리), 다중 사용자 인증, 사내 메신저를 추가하여 팀 단위 운영 지원.

**핵심 가치**: "나만의 AI 회사를 만들고, 운영하고, 진화시킨다."

### What Makes This Special

1. **동적 조직 관리**: 유일하게 비개발자가 AI 에이전트 조직을 자유롭게 설계/변경할 수 있는 플랫폼. 부서 생성, 에이전트 배치, 계급 지정, 도구 권한 설정, 조직 해체까지 코드 한 줄 없이 웹 UI에서 수행. 시스템 에이전트 보호 + cascade 안전 처리로 운영 안정성 보장.

2. **실전 검증된 22개 기능 영역**: 이론적 프레임워크가 아닌, v1에서 실제 운영된 125+ 도구 호출, 3사 LLM 라우팅, KIS 증권 자동매매, AGORA 다중 토론, SketchVibe 실시간 캔버스, SNS 멀티 플랫폼 발행. 모두 프로덕션 수준으로 동작 완료.

3. **3계급 오케스트레이션 + 비서실장 자동 라우팅**: Manager(Sonnet) -> Specialist(Haiku) -> Worker(Haiku) 계급 구조. 비서실장이 CEO 명령을 자동 분류하고 적합한 부서/팀장에게 위임. 병렬 작업 배분 + 결과 종합 + 품질 게이트(P0: 5항목 간이 검수, P1: quality_rules.yaml 자동 검수 + 환각 탐지) + 자동 재작업.

4. **아키텍처 원칙 -- 메모리 금지**: 모든 정보는 명시적 파일(작전일지, 기밀문서, Soul 문서, 학습 기록)로 관리. 숨겨진 저장소나 암묵적 상태 금지. 투명성과 감사 가능성 확보 (CEO 아이디어 #005).

5. **멀티테넌시 + Human 직원 워크스페이스**: 회사별 데이터 완전 격리(DB/파일/API키). Human 직원이 같은 AI 조직에 접속하여 사내 메신저로 소통하고, 각자의 워크스페이스에서 AI 에이전트에게 업무 위임. 1인용 도구에서 팀 도구로 진화.

6. **비용 투명성**: 에이전트별/모델별/부서별 비용 실시간 추적. 3계급 모델 배정(Manager=고급 모델, Worker=경량 모델)으로 자동 최적화. Batch API 50% 할인 활용. 일일/월 예산 한도 설정.

## Success Criteria

### User Success

**핵심 경험 기준**: 사용자가 "AI 조직이 실제로 일한다"고 느끼는 순간이 성공이다.

| 기준 | 측정 가능한 목표 | 성공 판정 기준 |
|------|----------------|--------------|
| 명령 응답 속도 | 단순 작업 < 60초, 복합 작업 < 5분 | 서버 타임스탬프(명령 입력 ~ 최종 응답) 기준 90%ile |
| 첫 명령 성공까지 소요 시간 | 회원가입 후 10분 이내 첫 보고서 수신 | 온보딩 타임스탬프 추적 |
| CEO 만족도 | 보고서 thumbs up 비율 80%+ | 피드백 버튼 집계, 월간 리포트 |
| 주간 활성 명령 수 | 사용자당 주 50+ (일 10+) | 사령관실 명령 로그 |
| 오케스트레이션 완주율 | 95%+ (명령 -> 최종 결과 정상 반환) | 에러/타임아웃 없이 완료된 비율 |
| 프리셋/크론 활용 | 사용자당 프리셋 5+, 크론 3+ | DB 집계 |

**Aha! 순간 (성공 감정 트리거):**
1. 첫 번째: "삼성전자 분석" -> 4명 에이전트 병렬 작업 -> 종합 보고서 -> "진짜 팀이 일하는 것 같다!"
2. 두 번째: 관리자 콘솔에서 "법무부" 생성 -> 에이전트 배치 -> 즉시 명령 가능 -> "조직을 내가 만들 수 있다니!"
3. 세 번째: 품질 게이트가 보고서 반려 -> 자동 재작업 -> 개선된 결과 -> "AI가 품질도 관리해주네"
4. 네 번째: 비용 대시보드에서 모델 변경 -> 비용 40% 절감 -> "이게 기업 운영이지"

**동적 조직 관리 성공 기준 (v2 핵심):**

| 기준 | 측정 가능한 목표 | 연결 페르소나 |
|------|----------------|-------------|
| 부서 생성/수정 빈도 | 활성 사용자 70%가 최소 1회 조직 변경 | 박과장 |

## User Journeys

### Journey 1: 김대표의 첫 날 -- "AI 회사를 만들다" (Primary User - Success Path)

**Opening Scene**: 김대표는 IT 서비스 회사를 혼자 운영하는 40대 사업가다. 오늘도 ChatGPT에 "삼성전자 분석해줘"라고 입력하고, 결과를 복사해서 다른 대화창에 붙여넣으며 마케팅 전략에 연결한다. 투자, 마케팅, 법무, 전략 -- 4개 영역의 의사결정을 혼자 내리면서 "AI가 팀처럼 일해주면 좋겠다"고 늘 생각한다. 월말 AI 사용 청구서를 보고 놀라지만, 어디에 얼마를 썼는지 추적할 방법이 없다.

**Rising Action**:
1. CORTHEX 회원가입 -> 조직 템플릿 선택 화면. "투자분석 조직" 선택
2. 비서실장(시스템 에이전트) + CIO(Manager) + 시황분석가/종목분석가/기술분석가/리스크관리자(Specialist) 4명이 자동 생성
3. 조직도 트리 뷰에서 계층 구조 확인 -- 각 에이전트의 계급, 모델, 허용 도구가 표시됨
4. 사령관실 진입. 온보딩 가이드: "아래 명령을 입력해보세요: 삼성전자 분석해줘"
5. 명령 입력 -> 화면 하단에 위임 체인 실시간 표시: "비서실장이 명령 분류 중..." -> "CIO에게 위임" -> "종목분석가/기술분석가/리스크관리자에게 병렬 배분" -> "CIO 독자 분석 + 종합 중..." -> "비서실장 품질 검수 중..."

**Climax**: 3분 후. 종합 보고서가 도착한다. 재무 분석, 기술 차트 패턴, 리스크 평가, CIO 종합 의견이 하나의 보고서에 정리되어 있다. 품질 게이트 통과 마크가 보인다. "이건 진짜 4명이 동시에 일한 거잖아..." 김대표의 첫 번째 Aha! 순간.

**Resolution**: 김대표는 관리자 콘솔에서 "마케팅부"를 새로 만들고, 콘텐츠 전문가와 SEO 분석가를 배치한다. 즉시 사령관실에서 "@마케팅팀장 이번 주 SNS 콘텐츠 기획해줘"라고 명령한다. 두 번째 Aha!: "조직을 내가 만들 수 있다니!" 비용 대시보드에서 에이전트별/부서별 비용을 확인하고, Worker 에이전트의 모델을 Haiku로 변경하여 비용을 40% 줄인다. "이게 기업 운영이지."

다음 날, 김대표는 신사업 진출을 고민하며 "/토론 신사업 A안 vs B안" 명령을 입력한다. 6명 팀장이 각자 부서 관점에서 A안과 B안을 토론하고, AGORA 엔진이 라운드별로 의견을 정리하여 합의/비합의 결과를 보여준다(Phase 2). 또한 SketchVibe 캔버스에서 신사업 구조도를 그리며 AI에게 "이 플로우에서 병목이 어디야?"라고 질문하면, AI가 캔버스 노드를 실시간으로 추가/수정하며 답변한다(Phase 2).

**Journey가 드러내는 기능 요구사항**: 조직 템플릿, 조직도 시각화, 사령관실 텍스트 명령, @멘션, 실시간 위임 체인 표시, 비서실장 자동 라우팅, 병렬 작업 배분, 품질 게이트, 부서/에이전트 CRUD, 비용 대시보드, 모델 변경, AGORA 토론(/토론 명령, 라운드 기반, Phase 2), SketchVibe 캔버스(Mermaid↔Cytoscape, MCP SSE 실시간 AI 편집, Phase 2)

### Journey 2: 이사장의 투자 루틴 -- "4명의 애널리스트" (Primary User - Domain-Specific)

**Opening Scene**: 이사장은 한국/미국 시장 양쪽에서 주식 투자를 하는 55세 전업 투자자다. 매일 아침 5시에 일어나 미국 장 마감 결과를 확인하고, 한국 장 개장 전에 전략을 세운다. 증권사 HTS 차트, 뉴스 사이트, AI 대화 -- 도구가 분산되어 있다. "삼성전자 매수할까?"라고 AI에게 물으면 일반론만 나온다. 실제 재무제표와 기술 지표를 종합한 전문 분석이 나오지 않는다.

**Rising Action**:
1. CORTHEX 조직 템플릿에서 "투자분석 조직" 선택 -> CIO + 4명 전문가 자동 생성
2. 전략실 진입 -> 포트폴리오 대시보드에서 보유 종목 현황 확인 -> 실시간 시세(60초 갱신)
3. 사령관실: "/전체 내일 시장 전망 및 포트폴리오 리밸런싱 제안" 입력
4. 시황분석가가 글로벌 매크로 분석 + 종목분석가가 보유종목 재무 체크 + 기술분석가가 차트 패턴 + 리스크관리자가 위험도 평가 -- 4명 병렬 동시 작업
5. CIO가 자체 분석 후 4명 결과 종합: "삼성전자 비중 5% 축소, TSLA 3% 추가 매수" 제안 + 신뢰도/비중 산출

**Climax**: 자동매매 승인 대기 알림. "자율 실행" 또는 "승인 후 실행" 중 선택. 이사장은 "승인 후 실행"을 선택하고, CIO 분석 근거를 확인한 뒤 "실행" 버튼. VECTOR가 KIS API로 주문 전송 -> 체결 확인. 모의거래로 새 전략을 먼저 테스트할 수도 있다.

**Resolution**: 이사장은 ARGOS에 "삼성전자 공시 발표", "TSLA 10% 이상 변동" 트리거를 설정한다. 조건 만족 시 자동으로 CIO에게 긴급 분석 요청. 크론 스케줄러로 매일 아침 6시 "오늘 시황 브리핑"을 자동 실행하고, 텔레그램으로 결과를 수신한다. "4명의 애널리스트를 고용한 효과"를 월 몇 만원에 얻는다.

**Journey가 드러내는 기능 요구사항**: 전략실(포트폴리오 대시보드, 실시간 시세, 모의거래), KIS 자동매매(자율/승인 선택), CIO+VECTOR 분리(#001), 투자 성향별 리스크 제어, ARGOS 트리거, 크론 스케줄러, 텔레그램 명령

### Journey 3: 박과장의 팀 구축 -- "AI를 팀 인프라로" (Primary User - Team Management)

**Opening Scene**: 박과장은 30명 규모 회사의 기획팀장이다. 경영진이 "AI로 생산성 올려"라고 지시했다. 팀원 8명이 각자 ChatGPT, Claude, Gemini를 쓰는데 결과물 품질이 제각각이다. 누가 얼마나 AI에 쓰는지 추적할 방법이 없다. "회사 표준 AI"를 만들고 싶지만 어디서 시작할지 모른다.

**Rising Action**:
1. CORTHEX 관리자 콘솔 접속 -> 회사 정보 설정 -> 빈 조직으로 시작
2. "마케팅부" 생성 -> AI 에이전트 3명 배치: 콘텐츠 전문가(Specialist), SEO 분석가(Specialist), SNS 관리자(Worker)
3. 각 에이전트의 Soul(성격 문서) 편집: "우리 회사 톤앤매너에 맞게 작성", "SEO 키워드 필수 포함" 등 부서 표준 설정
4. 에이전트별 allowed_tools 권한 설정: SNS 관리자에게만 소셜미디어 발행 도구 허용
5. 팀원 3명에게 워크스페이스 부여 + 접근 권한: 마케팅부 에이전트만 사용 가능
6. 비용 한도 설정: 팀원당 일일 $5, 부서당 월 $200

**Climax**: 팀원들이 사령관실에서 명령. 결과물이 부서 표준 Soul에 따라 일관된 톤과 품질로 나온다. 통신로그에서 팀원별 위임 기록(from -> to, 비용, 토큰)을 확인하고, QA 탭에서 품질 검수 통과율을 모니터링한다. "표준화가 이렇게 쉬울 줄이야."

**Resolution**: 박과장은 "법무부"를 추가 생성하고 계약서 검토 전문가를 배치한다. 관리자 콘솔에서 부서별 비용 대시보드를 확인: 마케팅부 $150/월, 법무부 $80/월. 경영진에게 "AI 도입 ROI 보고서"를 비용 데이터 기반으로 작성한다. 팀원들은 사내 메신저(Phase 3)로 AI 분석 결과를 공유하며 협업한다.

**Journey가 드러내는 기능 요구사항**: 관리자 콘솔(부서/에이전트/직원 CRUD), Soul 편집, 부서 표준 템플릿(#011), 에이전트별 도구 권한, Human 직원 워크스페이스, 부서별 접근 권한, 비용 한도, 통신로그(활동/통신/QA/도구), 팀원별 비용 추적

### Journey 4: 김대표의 위기 대응 -- "부서 해체와 재편" (Primary User - Edge Case / Error Recovery)

**Opening Scene**: 김대표가 6개월간 운영해온 "마케팅부"를 해체하고 "사업개발부"로 재편하려 한다. 마케팅부에는 진행 중인 SNS 콘텐츠 작성 작업이 3건, 에이전트 메모리(학습 기록)가 축적되어 있고, 이번 달 비용이 $120 기록되어 있다.

**Rising Action**:
1. 관리자 콘솔에서 "마케팅부" 삭제 시도
2. cascade 경고 팝업: "진행 중 작업 3건이 있습니다. 완료 대기 / 강제 종료 중 선택하세요"
3. 김대표는 "완료 대기" 선택 -> 3건 작업이 완료될 때까지 부서 삭제 보류
4. 작업 완료 후 재시도 -> "에이전트 메모리/학습 기록은 아카이브로 보존됩니다. 비용 기록 $120은 영구 보존됩니다(회계 추적). 하위 에이전트 3명은 미배속 상태로 전환됩니다."
5. 확인 -> 마케팅부 삭제 완료
6. "사업개발부" 신설 -> 미배속 에이전트 중 콘텐츠 전문가를 "사업개발부"로 재배치 -> Soul 편집으로 전문분야를 "마케팅 콘텐츠" -> "사업 제안서 작성"으로 변경

**Climax**: 사령관실에서 "@사업개발팀장 신규 파트너십 제안서 작성해줘" -> 재배치된 에이전트가 새 Soul에 맞게 작업 수행. 기존 학습 기록(아카이브)은 참조 가능하지만 새 부서의 맥락으로 작업. "조직 해체가 이렇게 안전하다니."

**Resolution**: 김대표는 비용 대시보드에서 삭제된 마케팅부의 과거 비용 기록($120)이 여전히 "아카이브 부서" 항목으로 표시되는 것을 확인한다. 회계 추적이 끊기지 않았다. 조직도에서 미배속 에이전트 2명이 "미배속" 영역에 표시되어 있고, 필요할 때 다른 부서로 드래그하여 재배치할 수 있다.

**Journey가 드러내는 기능 요구사항**: 부서 삭제 cascade(진행 중 작업 대기/강제 종료), 에이전트 메모리 아카이브, 비용 기록 영구 보존, 에이전트 미배속 전환, 에이전트 재배치(부서 이동), Soul 편집, 아카이브 부서 비용 표시

### Journey 5: 시스템 관리자의 일상 -- "멀티테넌시 운영" (Admin/Operations User)

**Opening Scene**: 시스템 관리자(Admin)는 CORTHEX를 3개 회사에 제공하고 있다. 각 회사의 데이터가 완전 격리되어야 하고, 회사별 비용 한도를 관리해야 한다. 오늘은 새 회사(C사)를 온보딩하고, 기존 B사의 비용 한도를 조정하는 작업이 있다.

**Rising Action**:
1. 관리자 콘솔 -> 회사 관리 -> "C사" 신규 생성 -> companyId 자동 발급
2. C사 관리자 계정 생성 -> 초기 비밀번호 + 접속 링크 발급
3. C사 관리자가 로그인 -> 자신의 관리자 콘솔에서 부서/에이전트 설계 시작 (A사/B사 데이터는 보이지 않음)
4. B사 비용 관리 -> 월 예산 한도 $500 -> $800으로 상향 조정
5. 전체 회사 비용 대시보드: A사 $320/월, B사 $480/월(한도 근접 경고), C사 $0(신규)

**Climax**: B사에서 에이전트가 일일 예산 한도에 도달. 자동 차단 알림이 B사 관리자에게 전달. Admin은 관리자 콘솔에서 B사의 LLM 사용 패턴을 확인하고, Batch API 활용률이 20%밖에 안 되는 것을 발견. B사 관리자에게 "비긴급 작업을 배치로 전환하면 비용 40% 절감 가능"이라고 안내.

**Resolution**: 3개 회사가 동시에 운영되면서도 데이터/비용/API 키가 완전히 격리되어 있다. 각 회사 관리자는 자기 회사의 조직만 보고 관리한다. Admin은 전체 현황을 모니터링하되 개별 회사의 명령 내용이나 보고서는 접근할 수 없다(데이터 격리).

**Journey가 드러내는 기능 요구사항**: 멀티테넌시(companyId 격리), 회사 CRUD, 관리자 계정 생성, 비용 한도 관리, 전체/회사별 비용 대시보드, 일일 예산 자동 차단, Batch API 활용 분석, 데이터 접근 격리

### Journey 6: 에이전트 품질 문제 대응 -- "품질 게이트가 걸러낸 환각" (Error Recovery / Quality)

**Opening Scene**: 김대표가 "TSLA 최근 실적 분석해줘"라고 명령했다. 종목분석가 에이전트가 분석을 완료하고 CIO가 종합했지만, 비서실장(편집장)의 품질 게이트에서 반려되었다.

**Rising Action**:
1. 비서실장 품질 검수 5항목 중 "근거" 항목 실패: "3분기 매출 $24.9B"이라는 수치의 출처가 불명확
2. 자동 재작업 트리거 -> 종목분석가에게 "출처 명시하여 재분석" 지시
3. 종목분석가가 도구를 통해 실제 재무 데이터를 조회하고 출처를 명시하여 재작성
4. CIO 재종합 -> 비서실장 재검수 -> 5항목 전부 PASS

**Climax**: 김대표는 통신로그 QA 탭에서 이 과정을 확인한다: "1차 제출 -> 품질 게이트 반려(근거 불충분) -> 재작업 -> 2차 제출 -> PASS". 환각이 CEO에게 도달하기 전에 걸러졌다.

**Resolution**: 김대표는 보고서에 thumbs up을 누른다. 품질 게이트 통과율이 대시보드에 반영된다. 반려 사유 "환각/사실 오류"가 집계되어 환각 탐지율 지표에 기여한다. P1 단계에서는 quality_rules.yaml 자동 규칙이 이 검수를 더 정밀하게 수행한다.

**Journey가 드러내는 기능 요구사항**: 품질 게이트 5항목(결론/근거/리스크/형식/논리), 자동 재작업, 통신로그 QA 탭, 환각 탐지, thumbs up/down 피드백, quality_rules.yaml(P1)

## Innovation & Novel Patterns

### Detected Innovation Areas

**Innovation Signal Match**: SaaS B2B 프로젝트 타입의 "Workflow automation; AI agents" 시그널에 정확히 매칭. 추가로 Fintech + AI Agent Platform의 교차 도메인 혁신이 감지됨.

**1. 패러다임 전환: "AI를 도구로" -> "AI를 조직으로"**

기존 AI 제품(ChatGPT, Claude, Copilot)은 AI를 "도구"로 사용한다 -- 사용자가 질문하면 AI가 답한다. CORTHEX는 AI를 "조직"으로 운영한다 -- CEO가 명령하면 비서실장이 분류하고, 팀장이 위임하고, 전문가가 병렬 작업하고, 결과를 종합한다. 이것은 1:1 대화에서 N:N 협업으로의 근본적 전환이다.

기존 멀티에이전트 프레임워크(AutoGPT, CrewAI, LangGraph)도 다중 에이전트를 사용하지만, 코드로 정의하고 개발자만 사용할 수 있다. CORTHEX의 혁신은 **비개발자가 웹 UI에서 조직 구조 자체를 설계하고 변경할 수 있다**는 점이다.

**2. 동적 조직 관리 -- 시장에 없는 개념**

"AI 에이전트 조직을 비개발자가 자유롭게 CRUD한다"는 개념 자체가 시장에 없다. 기존 제품들은:
- 고정 에이전트 구성 (v1 CORTHEX)
- 코드로 에이전트 정의 (CrewAI, LangGraph)
- 단일 에이전트 커스텀 (Custom GPTs)

CORTHEX v2는 부서 생성/해체, 에이전트 배치/이동, 계급 변경, 도구 권한 편집, Soul(성격) 수정, cascade 안전 처리까지 코드 없이 웹 UI에서 수행한다. 이것은 "AI 에이전트 조직 관리"라는 새로운 제품 카테고리를 정의한다.

**3. 3계급 비용 최적화 -- AI 에이전트의 기업 운영 관점**

AI 에이전트에 "계급"을 부여하고 계급별로 LLM 모델을 자동 배정하는 것은 기존에 없는 접근이다. Manager(비싼 모델, 판단/종합), Specialist(중간 모델, 분석), Worker(경량 모델, 단순 작업)로 분리하여 품질은 유지하면서 비용을 40%+ 절감한다. 이것은 실제 기업의 직급별 인건비 구조를 AI에 적용한 것이다.

**4. Fintech + AI Agent 교차 도메인**

투자 분석/자동매매를 AI 에이전트 조직이 수행하는 것은 기존에 없다. 개별 AI 트레이딩 봇(분석 또는 실행)은 있지만, CIO(분석 총괄) + 4명 전문가(병렬 분석) + VECTOR(실행봇)가 조직적으로 협업하며, CEO가 투자 성향만 설정하면 되는 구조는 CORTHEX 고유의 혁신이다.

## SaaS B2B Specific Requirements

CSV 기반 프로젝트 타입: **saas_b2b** / required_sections: tenant_model, rbac_matrix, subscription_tiers, integration_list, compliance_reqs / skip_sections: cli_interface, mobile_first

### Project-Type Overview

CORTHEX v2는 Multi-tenant SaaS B2B 플랫폼이다. 핵심 특성:
- **다중 앱 구조**: CEO/직원용 메인 앱(packages/app) + 관리자 콘솔(packages/admin) 분리
- **실시간 통신**: WebSocket 7채널 멀티플렉싱 + SSE 스트리밍 (토론/SketchVibe)
- **AI 백엔드**: LLM 멀티 프로바이더 라우팅 + 125+ 도구 통합 + Batch API
- **금융 연동**: KIS 증권 API 자동매매 (한국/미국 시장)

### Tenant Model

| 항목 | 설계 |
|------|------|
| 격리 전략 | **Shared Database, Shared Schema, Row-Level Isolation** -- 모든 테이블에 companyId 칼럼. WHERE 절로 격리 |
| 격리 대상 | DB 레코드(companyId), 파일 저장소(companyId 디렉토리), API 키(회사별 별도 볼트 엔트리), WebSocket 채널(companyId 네임스페이스) |
| 격리 검증 | ORM 미들웨어에서 companyId 자동 주입. API 미들웨어에서 JWT companyId vs 요청 companyId 불일치 시 403 |
| 테넌트 프로비저닝 | Admin이 관리자 콘솔에서 회사 생성 -> companyId 자동 발급 -> 관리자 계정 생성 -> 초기 비밀번호 + 접속 링크 |
| 테넌트 삭제 | Soft delete. 데이터는 30일 보존 후 영구 삭제 가능. 금융 거래 기록은 영구 보존(감사 추적) |
| 크로스 테넌트 접근 | 완전 금지. Admin도 개별 회사의 명령 내용/보고서 열람 불가. 비용/사용량 집계만 접근 가능 |

### RBAC Matrix (Role-Based Access Control)

| 역할 | 앱 접근 | 사령관실 | 조직 관리 | 비용 관리 | 직원 관리 | 시스템 설정 |
|------|--------|---------|----------|----------|----------|-----------|
| **Super Admin** | Admin 콘솔 | X | 전체 회사 | 전체 회사 집계 | 전체 회사 | 시스템 전체 |
| **Company Admin** | Admin 콘솔 + 메인 앱 | O (자기 회사) | 자기 회사 전체 | 자기 회사 전체 | 자기 회사 | 자기 회사 |
| **CEO** | 메인 앱 | O (전 부서) | 열람만 (변경은 Admin) | 자기 회사 전체 | X | X |
| **Human 직원** | 메인 앱 | O (권한 부여된 부서) | X | 자기 비용만 | X | X |
| **AI 에이전트** | API만 | 명령 수신 | X | X | X | X |

**권한 상세:**
- Company Admin = 해당 회사의 모든 관리 권한. CEO가 직접 겸할 수 있음
- CEO = 사령관실 전체 접근 + 비용 열람. 조직 변경은 Admin 권한 필요
- Human 직원 = Admin이 부여한 부서만 접근. 자기 워크스페이스 내 명령/비용만 확인
- AI 에이전트 = 서버 내부에서만 동작. API 엔드포인트 직접 접근 불가. allowed_tools로 도구 제한

## Project Scoping & Phased Development

### MVP Strategy & Philosophy

**MVP Approach: Platform MVP** -- 핵심 플랫폼 기능(동적 조직 관리 + 오케스트레이션)이 동작하면 "이 제품은 가치가 있다"를 증명한다. 22개 기능 전체가 아닌, CEO가 "AI 조직이 실제로 일한다"고 느끼는 최소 기능 집합을 먼저 제공한다.

**MVP 판단 기준 -- "이것 없으면 제품이 실패하는가?":**

| 기능 | 없으면 실패? | 근거 | MVP 포함 |
|------|------------|------|---------|
| 동적 조직 관리 | Yes | v2 핵심 차별점. 이것 없으면 v1과 같은 고정 조직 | P0 |
| 사령관실 | Yes | CEO 명령 입력 수단. 이것 없으면 AI와 소통 불가 | P0 |
| 비서실장 오케스트레이션 | Yes | 자동 라우팅 + 위임 체인. 이것 없으면 "조직" 작동 불가 | P0 |
| 3계급 에이전트 | Yes | 계급별 모델 배정. 이것 없으면 비용 통제 불가 | P0 |
| 도구 시스템 (30+) | Yes | 에이전트가 실제 작업 수행하는 수단. 도구 없으면 빈 깡통 | P0 |
| LLM 멀티 프로바이더 | Yes | AI 두뇌. 이것 없으면 에이전트 작동 불가 | P0 |
| 아키텍처 원칙 (#005) | Yes | 설계 원칙. Phase 1에서 안 지키면 Phase 2에서 리팩토링 필요 | P0 |
| 작전현황 대시보드 | No (수동 확인 가능) | 편의 기능. 사령관실 통신로그로 대체 가능 | P1 |
| 통신로그 | No (개발 로그로 대체 가능) | 투명성. 없어도 동작은 하지만 신뢰 확보에 필요 | P1 |
| 비용 관리 | No (수동 계산 가능) | 비용 투명성. API 대시보드에서 수동 확인 가능 | P1 |
| 품질 게이트 (yaml) | No (P0 간이 검수로 커버) | P0에서 비서실장 5항목 검수. P1에서 자동화 강화 | P1 |
| 멀티테넌시 | No (단일 테넌트로 시작 가능) | 팀 기능. CEO 1인 사용에는 불필요 | P1 |
| 전략실/AGORA/SketchVibe/SNS 등 | No | 독립 기능 영역. 핵심 오케스트레이션 안정화 후 이식 | Phase 2 |

| 비개발자 복잡도 부담 | Medium | 온보딩 10분 이내 첫 성공 목표. 고급 기능(cascade, RBAC)은 점진적 노출 |
| Fintech 규제 변경 | Low | KIS API 래퍼 계층으로 API 변경 대응. 모의거래로 규제 리스크 최소화 |

| 기능 범위 과다 | Medium | Phase 1 MVP를 P0 7개로 엄격히 제한. P1 5개는 P0 안정화 후 순차 추가. "22개 전부"는 Phase 2 목표 |

**Absolute Minimum (리소스 극한 시):**
P0 7개 중에서도 최소 동작 단위:
1. 사령관실 (명령 입력)
2. 비서실장 오케스트레이션 (자동 라우팅 + 위임)
3. LLM 프로바이더 (최소 Claude 1개)
4. 에이전트 시스템 (고정 3명 + Soul)
5. 도구 시스템 (핵심 10개)

이 5개만으로도 "AI 조직이 일한다"는 핵심 가치를 증명할 수 있다. 동적 조직 관리는 고정 조직으로 대체 가능하고, Batch API/멀티 프로바이더는 후속 추가 가능하다.

## Functional Requirements

이 섹션은 CORTHEX v2의 **능력 계약(Capability Contract)**이다. UX 설계, 아키텍처, 에픽 분해 모두 이 목록에 있는 능력만 대상으로 한다. 여기에 없는 기능은 최종 제품에 존재하지 않는다.

### 1. 조직 관리 (Dynamic Organization Management)

- FR1: Admin은 웹 UI에서 부서를 생성/수정/삭제할 수 있다
- FR2: Admin은 AI 에이전트를 생성할 때 이름, 계급(Manager/Specialist/Worker), LLM 모델, 허용 도구(allowed_tools), Soul(성격 문서)을 지정할 수 있다
- FR3: Admin은 AI 에이전트를 수정/삭제할 수 있다
- FR4: Admin은 에이전트를 다른 부서로 이동(재배치)할 수 있다
- FR5: 시스템 에이전트(비서실장 등 오케스트레이션 필수 역할)는 삭제가 차단되며, 사용자 에이전트와 시각적으로 구분된다
- FR6: 부서 삭제 시 시스템이 cascade 영향을 분석하여 표시한다 (진행 중 작업 수, 에이전트 수, 비용 기록)
- FR7: 부서 삭제 시 CEO가 진행 중 작업에 대해 "완료 대기" 또는 "강제 종료"를 선택할 수 있다
- FR8: 부서 삭제 시 에이전트 메모리/학습 기록은 아카이브로 보존되고, 비용 기록은 영구 보존되며, 하위 에이전트는 미배속 상태로 전환된다
- FR9: Admin은 조직도를 트리 뷰로 확인할 수 있다 (부서-팀장-전문가 계층, 에이전트 상태 표시)
- FR10: Admin은 조직 템플릿("투자분석", "마케팅" 등)을 선택하여 사전 구성된 부서/에이전트를 일괄 생성할 수 있다
- FR11: Admin은 에이전트의 Soul(성격 문서)을 웹 UI에서 편집할 수 있다
- FR12: Admin은 부서별 표준 템플릿(Soul 기본값, 도구 권한 기본값)을 설정할 수 있다

### 2. 사령관실 & 명령 (Command Center)

- FR13: CEO/Human 직원은 사령관실에서 자연어 텍스트 명령을 입력할 수 있다
- FR14: CEO/Human 직원은 @멘션으로 특정 팀장(Manager)에게 명령을 직접 지정할 수 있다
- FR15: CEO/Human 직원은 슬래시 명령어 8종(/전체, /순차, /도구점검, /배치실행, /배치상태, /명령어, /토론, /심층토론)을 사용할 수 있다
- FR16: CEO/Human 직원은 자주 쓰는 명령을 프리셋으로 저장하고 실행할 수 있다
- FR17: CEO/Human 직원은 사령관실에서 위임 체인 진행 상황을 실시간으로 확인할 수 있다 (어떤 에이전트가 현재 작업 중인지)
- FR18: CEO/Human 직원은 보고서에 thumbs up/down 피드백을 남길 수 있다

### 3. 오케스트레이션 & 위임 (Orchestration)

- FR19: 비서실장은 CEO 명령을 자동 분류하여 적합한 부서/팀장에게 위임할 수 있다
- FR20: 팀장(Manager)은 위임받은 작업을 하위 전문가(Specialist/Worker)에게 병렬로 배분할 수 있다
- FR21: 팀장(Manager)은 자체 분석을 수행한 후 하위 전문가 결과와 종합할 수 있다 (처장=5번째 분석가, #007)
- FR22: 비서실장은 종합된 결과를 CEO 눈높이로 정리하여 최종 보고서를 생성할 수 있다 (비서실장=편집장, #010)
- FR23: 에이전트는 자율 딥워크(계획 -> 데이터 수집 -> 분석 -> 초안 -> 최종)를 수행할 수 있다
- FR24: /전체 명령 시 모든 팀장에게 동시 위임하고 결과를 종합할 수 있다
- FR25: /순차 명령 시 팀장들에게 순차적으로 위임하고 결과를 누적 종합할 수 있다

### 4. 도구 & LLM (Tool System & LLM Routing)

- FR26: 에이전트는 허용된 도구(allowed_tools)를 호출하여 외부 데이터 조회/작업 수행을 할 수 있다
- FR27: 시스템은 에이전트별로 허용된 도구 목록을 서버 사이드에서 강제할 수 있다 (권한 외 도구 호출 자동 차단)
- FR28: Admin은 에이전트별 허용 도구를 설정/변경할 수 있다
- FR29: 시스템은 모든 도구 호출을 로그에 기록하고, 사용자가 UI에서 확인할 수 있다
- FR30: 시스템은 에이전트의 계급에 따라 LLM 모델을 자동 배정할 수 있다 (Manager=고급, Specialist=중급, Worker=경량)
- FR31: Admin은 에이전트별 LLM 모델을 수동으로 변경할 수 있다
- FR32: 시스템은 Claude/GPT/Gemini 3개 프로바이더에 요청을 라우팅할 수 있다
- FR33: 시스템은 비긴급 작업을 Batch API로 일괄 처리하여 비용을 절감할 수 있다
- FR34: 시스템은 LLM 프로바이더 장애 시 자동으로 다른 프로바이더로 fallback할 수 있다

### 5. 모니터링 & 비용 관리 (Monitoring & Cost Management)

- FR35: CEO/Admin은 홈 대시보드(작전현황)에서 요약 카드(작업/비용/에이전트/연동)를 확인할 수 있다
- FR36: CEO/Admin은 AI 사용량 그래프(프로바이더별)를 확인할 수 있다
- FR37: CEO/Admin은 통신로그에서 4개 서브탭(활동/통신/QA/도구)으로 에이전트 활동을 확인할 수 있다
- FR38: CEO/Admin은 에이전트별/모델별/부서별 비용을 실시간 추적할 수 있다
- FR39: Admin은 일일/월 예산 한도를 설정할 수 있고, 한도 도달 시 시스템이 자동 차단하고 알림을 보낸다
- FR40: CEO/Admin은 비용 대시보드에서 차트(도넛/막대)로 비용 분포를 시각화할 수 있다
- FR41: CEO는 보고서 만족도(thumbs up/down) 통계를 대시보드에서 확인할 수 있다

### 6. 보안 & 멀티테넌시 (Security & Multi-tenancy)

- FR42: 시스템은 모든 데이터를 companyId 기반으로 격리하여, 다른 회사의 데이터가 노출되지 않도록 보장한다
- FR43: Admin은 회사를 생성하고, 해당 회사의 관리자 계정을 발급할 수 있다
- FR44: Admin은 Human 직원의 워크스페이스를 생성하고 접근 권한(부서별)을 부여할 수 있다
- FR45: Human 직원은 자기 워크스페이스 내에서만 명령/비용/이력을 확인할 수 있다
- FR46: 시스템은 API 키(LLM/KIS/외부)를 AES-256-GCM으로 암호화하여 저장한다
- FR47: 시스템은 에이전트 프롬프트에 크리덴셜을 전달하지 않으며, 도구 호출 시 서버 사이드에서 직접 주입한다
- FR48: 시스템은 JWT 기반으로 사용자를 인증하고, 역할(Super Admin/Company Admin/CEO/Human 직원)에 따라 API 접근을 제한한다
- FR49: 시스템은 금융 거래/조직 변경/권한 변경을 삭제 불가 감사 로그에 기록한다

### 7. 품질 관리 (Quality Management)

- FR50: 비서실장은 모든 에이전트 결과물을 5항목(결론/근거/리스크/형식/논리)으로 자동 검수할 수 있다
- FR51: 품질 게이트 실패 시 시스템이 해당 에이전트에게 자동 재작업을 지시할 수 있다
- FR52: CEO/Admin은 통신로그 QA 탭에서 품질 검수 결과(통과/반려/재작업)를 확인할 수 있다
- FR53: 시스템은 quality_rules.yaml 기반으로 자동 규칙 검수(완전성/정확성/형식)를 수행할 수 있다
- FR54: 시스템은 에이전트 결과물에서 환각(사실과 다른 내용)을 탐지할 수 있다
- FR55: 시스템은 시스템 프롬프트와 사용자 입력을 분리하고, 에이전트 출력에서 시스템 프롬프트/크리덴셜 패턴을 필터링한다 (프롬프트 인젝션 방어)

### 8. 투자 & 금융 (Investment & Finance) -- Phase 2

- FR56: CEO는 전략실에서 포트폴리오 대시보드(보유 종목 현황, 실시간 시세)를 확인할 수 있다
- FR57: CIO와 전문가 4명(시황/종목/기술/리스크)이 병렬로 투자 분석을 수행하고 CIO가 종합할 수 있다
- FR58: VECTOR(실행봇)가 KIS API를 통해 한국/미국 시장에 매매 주문을 전송할 수 있다
- FR59: CEO는 자동매매 실행 모드를 "자율 실행" 또는 "승인 후 실행" 중 선택할 수 있다
- FR60: CEO는 투자 성향(보수/균형/공격)을 설정하여 매매 한도/비중을 제한할 수 있다
- FR61: 시스템은 실거래와 모의거래 환경을 완전 분리하고, 실거래 전환 시 2단계 확인을 요구한다
- FR62: 시스템은 모든 매매 주문(실행/취소/거부/체결/미체결)을 영구 보존한다

### 9. 협업 & 확장 기능 (Collaboration & Extended Features) -- Phase 2/3

- FR63: CEO/Human 직원은 AGORA 토론 엔진으로 다수 팀장이 라운드 기반 토론을 벌이는 것을 요청하고 결과(합의/비합의)를 확인할 수 있다
- FR64: CEO/Human 직원은 SketchVibe 캔버스에서 다이어그램을 그리며 AI와 실시간으로 토론할 수 있다
- FR65: CEO/Human 직원은 SNS 통신국을 통해 5개 플랫폼에 콘텐츠를 발행할 수 있다 (승인 플로우 포함)
- FR66: Admin은 크론 스케줄러로 반복 명령을 자동화할 수 있다
- FR67: Admin은 ARGOS에 트리거 조건을 설정하여 조건 만족 시 자동 분석을 실행할 수 있다
- FR68: CEO/Human 직원은 텔레그램으로 AI 조직에 명령을 보내고 결과를 수신할 수 있다
- FR69: 에이전트는 정보국(RAG)에 저장된 지식을 참조하여 작업 품질을 향상시킬 수 있다
- FR70: 에이전트는 과거 작업에서 자동 학습하여 반복 작업의 품질을 점진적으로 개선할 수 있다
- FR71: CEO/Human 직원은 작전일지에서 과거 명령 이력을 검색하고 리플레이/A/B 비교할 수 있다
- FR72: CEO/Human 직원은 기밀문서에 보고서를 아카이브하고 유사 문서를 검색할 수 있다
- FR73: Admin/CEO는 전력분석(Soul Gym)에서 에이전트별 성능 개선 제안을 확인하고 Soul을 업데이트할 수 있다
- FR74: Admin은 자동화 워크플로우로 다단계 파이프라인을 구축할 수 있다
- FR75: 시스템은 사용자 명령 패턴을 분석하여 자주 사용하는 워크플로우를 자동으로 제안할 수 있다 (#004, Phase 2)
- FR76: Human 직원은 사내 메신저로 다른 Human 직원과 실시간 채팅할 수 있다 (Phase 3)
