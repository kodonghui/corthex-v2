### 1.1 User Personas

### 1.4 Architecture-Driven UX Constraints

| 아키텍처 결정 | UX 영향 | 설계 지침 |
|-------------|--------|----------|
| SSE 6개 이벤트 (E5) | 허브 UI 6개 상태 표현 | `accepted`: 로딩 스피너 + "명령 접수됨". `processing`: "{에이전트명} 분석 중..." 펄스 애니메이션. `handoff`: 트래커 노드 추가(from→to 화살표). `message`: 마크다운 스트리밍. `error`: 붉은 배너 + 에러 코드 + 한국어 메시지. `done`: 비용/토큰 뱃지 |
| 비서 유무 분기 | 허브 레이아웃 2종 | 비서 있음: 풀 위드 채팅(에이전트 선택 패널 없음). 비서 없음: 좌측 w-64 에이전트 패널 + 우측 채팅 |
| 에이전트 50+ 그룹핑 | 에이전트 선택 UI | 부서별 아코디언 그룹핑 + 부서명 badge + 에이전트 lastUsedAt 내림차순. 상단 검색 input |
| 핸드오프 트래커 | 실시간 위임 시각화 | 채팅 영역 상단 고정 바: [비서실장] → [CIO] → [종목분석가] 노드 체인. 현재 활성 노드 펄스 애니메이션. 완료 노드 ✅ 체크 |
| 에러 투명성 | 핸드오프 실패 표시 | 실패 노드 ❌ 표시 + 인라인 에러 메시지: "{에이전트명}: {한국어 사유}". 블랙박스 에러 0건 원칙. SSE error의 agentName 필드가 있으면 반드시 표시 |
| 핸드오프 깊이 제한 (maxDepth) | HANDOFF_DEPTH_EXCEEDED 에러 | 에러 메시지: "위임 깊이 제한에 도달했습니다 (최대 {maxDepth}단계). 관리자에게 설정 변경을 요청하세요." 시스템 장애가 아닌 설정 제한임을 명확히 |
| 미배속 에이전트 | NEXUS + 에이전트 목록 표시 | NEXUS: "미배속" 영역(점선 테두리 그룹 노드). 에이전트 목록: 맨 아래 "미배속" 아코디언 그룹(회색 배경). 미배속 에이전트는 드래그로 부서에 재배치 가능 |
| Pre-spawn ~2초 (결정 5) | 체감 지연 흡수 | `accepted` SSE를 50ms 이내 발행 → UI 즉시 "명령 접수됨" 표시. 사용자는 2초 spawn을 기다리는 동안 로딩 상태를 봄 |

### 1.6 Engine Refactor UX Impact Summary

v2 Engine Refactor 3대 축의 UX 영향:

**축 1: Agent Engine Rewrite (SDK 기반, ~85% 코드 삭제)**
- 사용자 체감: 응답 속도 개선(SDK 최적화), N단계 핸드오프 깊이 표시, query당 비용 정밀 추적
- UX 변경: 핸드오프 트래커가 N단계 깊이 표시(기존 3단계 고정 → 동적), 비용 표시 세분화($0.0234 → 에이전트별 breakdown)
- 투명 개선: Hook 기반 보안 → UX 변경 없이 크레덴셜 스크러빙, 도구 권한 자동 체크

**축 2: Organization Design Innovation (N단계 계급, 비서 선택, NEXUS)**
- NEXUS UI: React Flow 노드 조직도. 드래그&드롭 부서/에이전트 배치. ELK.js 계층 자동 레이아웃. 계급별 색상(tier_configs.color)
- 비서 선택제: 허브 UX 2종 분기. 비서 있음(자연어 라우팅) vs 없음(에이전트 직접 선택)
- N단계 계급: 관리자 설정에서 계급 추가/편집/삭제. NEXUS 노드 색상 자동 반영
- call_agent 시각화: NEXUS 조직도 = 실제 핸드오프 호출 구조. 조직도에서 실시간 활성 노드 하이라이트(WebSocket 이벤트)

**축 3: Knowledge Enhancement (NotebookLM, pgvector)**
- 음성 브리핑: 에이전트 도구 패널에서 NotebookLM 도구 할당. 브리핑 생성 → 텔레그램 자동 전송. Google OAuth 미등록 시 ⚠️ 경고 배지
- 의미 검색: 정보국 검색 UX 개선 — 키워드 검색 + 자연어 질문 지원. "관련 문서 N건" 결과 카드
- 마인드맵/슬라이드: Phase 4+ NEXUS 연동. 에이전트가 생성한 마인드맵을 NEXUS 캔버스에 표시

## 2. Core Experience

### 2.1 Experience Vision

**한 줄 정의**: "나만의 AI 조직을 설계하고, 명령 한 줄로 팀이 일하게 한다."

CORTHEX v2를 사용하는 경험은 **CEO가 자기 회사를 운영하는 것**과 같다. 조직도를 보며 부서를 만들고, 에이전트를 배치하고, 명령을 내리면 조직이 자율적으로 움직인다. 결과가 돌아오면 보고서를 확인하고, 비용을 관리하고, 조직을 개선한다.

**핵심 감각 4가지:**
1. **통제감**: "내가 조직을 설계한다" — NEXUS에서 부서/에이전트를 드래그&드롭. 계급/모델/Soul을 내가 결정
2. **투명성**: "무슨 일이 일어나는지 보인다" — 핸드오프 트래커가 비서실장 → CIO → 전문가 위임 체인을 실시간 표시. 비용/토큰이 에이전트별로 표시
3. **위임 안도감**: "맡기면 알아서 한다" — 명령 한 줄 → accepted(50ms) → 자동 라우팅 → 병렬 분석 → 종합 보고서. 실패하면 왜 실패했는지 명시
4. **성장 실감**: "조직이 점점 나아진다" — 통신로그에서 에이전트 성능 확인 → Soul 튜닝 → 비용 최적화 → 부서 재편. PRD Journey 6 "조직 최적화" 루프: 관찰(대시보드) → 진단(통신로그) → 개선(Soul 편집/모델 변경/계급 조정) → 효과 확인(비용/품질 비교)

```
├── 비용 관리                ← 한도 설정 + 에이전트/부서/모델별 차트
├── 크레덴셜 볼트            ← API 키 / CLI 토큰 관리
├── 스케치바이브             ← Cytoscape 캔버스 (Phase 4)
└── 시스템 설정              ← 회사 정보, 연동 설정
```

### 2.5 Core Interaction Flow States

허브(사령관실)의 **SSE 기반** 명령 응답 상태 머신. 명령 응답은 SSE 스트리밍(서버→클라이언트, E5: accepted→processing→handoff→message→error→done), 취소는 REST `DELETE /api/workspace/hub/sessions/{sessionId}`, 실시간 알림(세션 외 이벤트)은 WebSocket/EventBus(D10). 두 채널이 역할별로 공존:

```
[Idle]
  │ 사용자 명령 입력 + Enter
  ▼
[Sending] → API POST /api/workspace/hub
  │ 서버 수신 즉시 (50ms)
  ▼
[Accepted] → SSE event: accepted
  │ UI: "명령 접수됨" + 로딩 스피너
  │ SDK spawn (~2초)
  ▼
[Processing] → SSE event: processing { agentName }
  │ UI: "{비서실장} 분석 중..." + 펄스 애니메이션
  │ 핸드오프 트래커 첫 노드 표시
  ▼
[Handoff] → SSE event: handoff { from, to, depth } (0~N회 반복)
  │ UI: 트래커에 [from ✅] → [to ●] 노드 추가
  │ 현재 활성 노드에 펄스 애니메이션
  ▼
[Streaming] → SSE event: message { content } (N회 반복)
  │ UI: 마크다운 스트리밍 렌더링
  ▼
[Done] → SSE event: done { costUsd, tokensUsed }
  │ UI: 비용 뱃지 + 토큰 뱃지
  │ 핸드오프 트래커 모든 노드 ✅
  │ 피드백 버튼 (👍/👎) 표시
  ▼
[Idle] ← 다음 명령 대기

  ──── 에러 경로 ────
[Any State]
  │ SSE event: error { code, message, agentName? }
  ▼
[Error]
  │ UI: 붉은 배너 + 에러 메시지
  │ 트래커에 실패 노드 ❌ 표시 (agentName 있으면)
  │ 코드별 한국어 메시지:
  │   HANDOFF_DEPTH_EXCEEDED → "위임 깊이 제한 도달 (최대 {N}단계)"
  │   HANDOFF_CIRCULAR → "순환 위임 감지됨"
  │   AGENT_TIMEOUT → "{에이전트명} 응답 시간 초과"
  │   TOOL_PERMISSION_DENIED → "도구 접근 권한 없음"
  │   SESSION_LIMIT_EXCEEDED → "동시 세션 한도 초과"
  ▼
[Idle] ← 재시도 또는 다음 명령

  ──── 취소 경로 ────
[Processing/Handoff/Streaming]
  │ 사용자가 [■ 중지] 버튼 클릭 또는 새 명령 입력
  ▼
[Cancelling]
  │ UI: "처리 중단 중..." + 현재 트래커 노드 ⬜ (회색)
  │ REST DELETE /api/workspace/hub/sessions/{sessionId} 호출 + SSE 연결 close
  │ 새 명령이면: 이전 결과 폴드 처리, 새 Sending 상태 진입
  ▼
[Idle]

  ──── 재연결 경로 ────
[Any Active State]
  │ SSE 연결 끊김 (네트워크 불안정, 모바일 전환 등)
  ▼
[Reconnecting]
  │ UI: 상단 노란 배너 "연결 중..." + 재연결 스피너
  │ 자동 재시도: 1초 → 2초 → 4초 (지수 백오프, 최대 30초)
  │ 이미 수신한 스트리밍 콘텐츠: 로컬 상태에 보존 (절대 삭제 안 함)
  │ 재연결 성공 시: lastEventId 기준 이후 이벤트만 재수신, 기존 콘텐츠 이어붙이기
  │ 30초 이상 실패: "연결이 끊어졌습니다. [새로고침]" 버튼 + 기존 콘텐츠는 표시 유지
  ▼
[Previous State 복원] 또는 [Error]
```

**Phase 2+ Journey 참고**: AGORA 토론(라운드 기반 다자 토론 UI, /토론 슬래시 명령) 및 전략실(포트폴리오 대시보드, KIS 자동매매)은 Phase 2 기능으로 별도 UX Journey가 필요하며, Phase 2 UX 설계 시 상세화한다. v1에서 핵심 기능이므로 Phase 2 스코프에서 반드시 커버해야 함.

## 3. Emotional Response & Trust

### 3.1 감정 곡선 (Emotional Journey Map)

각 핵심 Journey에서 사용자가 경험하는 감정 흐름:

```
Journey A: "첫 조직 만들기" — 감정 곡선
                                                    ★ Aha!
                                          😊 흥미     "4명이 동시에!"
                                        /           \
                              😌 안도   /             \ 😎 자신감
                            (템플릿 적용)/               \(비용 확인)
                           /                              \
              😕 불안     /                                 ──────
            (이게 뭐지?)/
           /
  😐 중립
  (가입 시작)

감정 트리거:
  가입 → 중립: "어떤 건지 모르겠지만 해보자"
  템플릿 선택 → 불안→안도: "이렇게 쉽게?" 프로그레스 바가 생성 과정 보여줌
  NEXUS 조직도 → 흥미: "진짜 조직이 생겼네" ELK 레이아웃의 시각적 임팩트
  첫 명령 결과 → Aha!: "4명이 동시에 일한 거잖아!" 핸드오프 트래커 시각화
  비용 확인 → 자신감: "$0.02면 충분히 쓸 수 있겠다" 투명한 비용 표시
```

```
Journey B: "일상 위임" — 감정 곡선

  😎 자신감──────────────────────────────────😎 만족
  (입력)   \                              / (보고서 도착)
            \ 😌 안도                    /
             (accepted 50ms, 즉시 응답)/
              \                      /
               ──────────────────────
               😐 대기 (핸드오프 관찰)
               "누가 뭘 하는지 보인다"

감정 트리거:
  명령 입력 → 자신감: 반복 사용으로 패턴 학습 완료
  accepted 50ms → 안도: "접수됐다" 즉각 피드백
  핸드오프 트래커 → 호기심: "CIO가 4명에게 위임했네" 실시간 관찰
  보고서 도착 → 만족: 기대한 포맷으로 결과 제공
```

```
Journey C: "비서 없는 직원" — 감정 곡선

  😕 불안──────────😐 탐색──────────😌 안도──────────😊 친밀
  (누구한테 시키지?)  (에이전트 목록)   (직접 선택 후)   (1:1 대화)

감정 트리거:
  허브 진입 → 불안: "에이전트가 많은데 누구한테?" 비서가 없어서 직접 선택 필요
  에이전트 패널 탐색 → 탐색: 부서별 그룹핑 + 추천 에이전트 카드가 선택 부담 줄임
  에이전트 클릭 → 안도: "이 전문가에게 직접 물어보자" 1:1 채팅 시작
  응답 수신 → 친밀: "이 에이전트 괜찮네, 다음에도 여기로" 자주 사용 에이전트 학습
```

```
Journey D: "조직 리디자인" — 감정 곡선

  😐 분석──────😊 창조──────😌 구조화──────😎 완성──────😌 안도
  (현재 조직도)  (부서 생성)   (에이전트 배치) (저장 완료)   (변경이력 확인)

감정 트리거:
  NEXUS 진입 → 분석: "현재 조직은 이렇게 돼 있구나" ELK 레이아웃 한눈에 파악
  더블클릭 → 창조: "부서를 내 손으로 만든다!" 즉각적 노드 생성
  드래그&드롭 → 구조화: "이 전문가를 여기 배치하면..." 레고 블록 맞추는 감각
  저장 → 완성: "조직 재편 끝!" 즉시 반영 확인
  변경이력 확인 → 안도: "잘못되면 되돌릴 수 있다" 최근 10건 이력 보존
```

### 3.2 신뢰 구축 패턴 (Trust Building Patterns)

| 신뢰 요소 | 구현 | UX 디테일 |
|-----------|------|----------|
| **즉각 반응** | accepted 이벤트 50ms 이내 | 명령 전송 → 0.05초 이내 "접수됨" 표시. 사용자가 "됐나?" 의심할 틈 없음 |
| **과정 투명성** | 핸드오프 트래커 실시간 표시 | 위임 체인 각 노드에 에이전트 이름 + 상태(●진행/✅완료/❌실패) 표시. "블랙박스"가 아님을 시각적으로 증명 |
| **비용 정직성** | 응답마다 비용/토큰 표시 | `done` 이벤트의 `costUsd`, `tokensUsed` 뱃지. 소수점 4자리까지 ($0.0234). 숨기지 않고 항상 노출 |
| **실패 설명** | 에러 코드 + 한국어 메시지 + 실패 에이전트명 | `HANDOFF_DEPTH_EXCEEDED` → "위임 깊이 제한 도달 (최대 5단계). 관리자에게 설정 변경을 요청하세요." 원인+해결책 제시 |
| **예측 가능성** | 유사 명령 → 유사 라우팅 경향 | 비서실장 Soul이 일관된 라우팅 기준을 정의. "투자 분석" 유형은 CIO로 라우팅되는 경향. 컨텍스트에 따라 달라질 수 있으나 기본 패턴은 학습 가능 |
| **복구 가능성** | 취소 + 이전 결과 보존 | [■ 중지] 언제든 클릭 가능. 이전 대화 내역 보존. "실수해도 돌이킬 수 있다" |

### 3.3 통제감 설계 (Control & Empowerment)

**Admin의 통제감 계층:**

```
Level 1: 조직 구조 통제
  └─ NEXUS 드래그&드롭 → 부서 생성/이동/삭제
     "내가 조직을 만든다"

Level 2: 에이전트 능력 통제
  └─ Soul 편집 (시스템 프롬프트) + 도구 권한 체크박스 + 모델 선택
     "에이전트의 성격과 능력을 내가 결정한다"

Level 3: 비용 통제
  └─ 부서별/에이전트별 비용 한도 설정 + 초과 시 자동 차단
     "예산은 내가 정한다"

Level 4: 접근 통제
  └─ 직원별 워크스페이스 범위 + 도구 접근 권한
     "누가 무엇을 쓸 수 있는지 내가 결정한다"
```

**User(직원/CEO)의 통제감:**

```
Level 1: 대화 통제
  └─ @멘션으로 에이전트 직접 지정 / 비서실장에게 위임 선택
     "누구에게 시킬지 내가 고른다"

Level 2: 비서 선택 통제
  └─ 비서실장 활성화/비활성화 토글
     "자동 라우팅이 싫으면 끌 수 있다"

Level 3: 결과 피드백 통제
  └─ 👍/👎 + 코멘트 → 만족도 통계 → Soul 개선 근거
     "피드백이 실제로 반영된다"
```

### 3.4 불안 해소 패턴 (Anxiety Reduction)

| 불안 시나리오 | 사용자 내면 | 해소 UX |
|-------------|-----------|---------|
| "명령이 접수됐나?" | 전송 후 0.5초 무반응 = 불안 | `accepted` 이벤트 50ms 이내 → "명령 접수됨" 메시지 + 로딩 스피너 |
| "지금 뭘 하고 있는 거지?" | 처리 중 2-10초 대기 | 핸드오프 트래커 실시간 노드 추가. "{비서실장} 분석 중..." 펄스 텍스트 |
| "이거 비용 폭탄 아니야?" | 결과 도착 전 비용 불안 | Processing 중 예산 프로그레스 바 "예산 사용: 12% [$0.60/$5.00]" (Stop hook 기반, 도구 호출 단위 갱신). 색상 구간: 초록 <70%, 노랑 70-90%, 빨강 >90%. `done` 시 최종 비용 뱃지. 한도 초과 시 자동 차단 |
| "실수했는데 취소 가능해?" | 잘못된 명령 전송 | [■ 중지] 버튼 항상 표시 (Processing~Streaming 구간). 즉시 취소 가능 |
| "연결 끊겼는데 결과 날아간 거야?" | 모바일 전환, 네트워크 불안 | 로컬 상태 보존 + "연결 중..." 노란 배너 + 자동 재연결. 기존 콘텐츠 절대 삭제 안 함 |
| "이 에이전트 어떻게 쓰는 건지?" | 새 에이전트, 사용법 모름 | 에이전트 카드에 "전문 분야: {description}" + "추천 명령: {example_commands}" 표시 |
| "조직 바꿨는데 원래대로 돌릴 수 있어?" | NEXUS 편집 후 불안 | 변경 사항 실시간 저장 + "변경 이력" 패널 (최근 10건). Phase 2에서 Undo/Redo 추가 |

### 3.5 마이크로인터랙션 감정 설계

| 인터랙션 | 시각 피드백 | 감정 목표 |
|---------|-----------|----------|
| 명령 전송 | 입력 필드 → 메시지 버블 슬라이드업 (200ms ease-out) | 즉각성: "보냈다" |
| accepted | 메시지 하단 "접수됨" 뱃지 fade-in (150ms) + 미묘한 체크 아이콘 | 안도: "받았다" |
| processing | 에이전트 아바타 펄스 (opacity 0.5→1.0, 1.5초 주기) | 호기심: "일하고 있구나" |
| handoff | 트래커 노드 슬라이드-인 (300ms, spring easing) + 화살표 그리기 애니메이션 | 투명성: "다음 담당자에게 넘겼구나" |
| streaming | 텍스트 타이핑 효과 (커서 깜박) + 마크다운 실시간 렌더링 | 몰입: "답이 오고 있다" |
| done | 비용 뱃지 팝-인 (scale 0→1, 200ms bounce) + 피드백 버튼 fade-in | 만족: "끝났다, 비용도 확인" |
| error | 붉은 배너 shake (6px, 300ms) + 에러 아이콘 bounce | 주목: "뭔가 잘못됐다" (공포가 아닌 알림) |
| cancel | 트래커 노드 회색 전환 (300ms fade) + "중단됨" 텍스트 | 통제감: "내가 멈췄다" |
| **NEXUS: 노드 생성** | 더블클릭 → 노드 팝-인 (scale 0→1, 250ms bounce) + 이름 입력 포커스 | 창조: "만들었다!" |
| **NEXUS: 드래그** | 노드 그림자 확대 (shadow-lg → shadow-2xl) + opacity 0.8 | 물리감: "잡고 있다" |
| **NEXUS: 드롭 (배치)** | 타겟 부서 노드 테두리 하이라이트 (#3B82F6 2px) + 성공 시 초록 flash | 성취: "배치 완료!" |
| **NEXUS: 삭제** | 확인 다이얼로그 (2초 딜레이 삭제 버튼) + 붉은 테두리 경고 | 신중함: "정말 삭제?" |
| **NEXUS: 엣지 그리기** | 소스→타겟 점선 미리보기 + 타겟 노드 글로우 | 연결감: "관계가 생겼다" |

| **Datadog/Grafana** | 실시간 모니터링 대시보드 + 메트릭 카드 + 시계열 차트 | 작전현황 대시보드: 비용 카드, 세션 메트릭, 에이전트 성능 차트, 만족도 게이지 | Datadog은 인프라 모니터링, CORTHEX는 AI 에이전트 성능/비용 모니터링 |
| **Terminal/CLI** | 텍스트 명령 → 즉시 응답 패턴 | 사령관실 입력 패러다임: 명령 입력 → 즉시 처리 → 결과 스트리밍 | CLI는 단일 프로세스, CORTHEX는 멀티 에이전트 체인. 핸드오프 시각화 추가 |

### 4.3 반면교사 (Anti-Patterns to Avoid)

| 피해야 할 패턴 | 사례 | CORTHEX에서의 방지책 |
|---------------|------|---------------------|
| **과도한 마법사(Wizard)** | 10단계 온보딩 → 이탈률 80% | 템플릿 1클릭 → 즉시 사용. 온보딩 3단계 이내 (템플릿 선택 → 첫 명령 → 결과 확인) |
| **블랙박스 AI** | "처리 중..." 만 보이고 내부 과정 불명 | 핸드오프 트래커로 위임 체인 전체 시각화. 각 에이전트 이름 + 상태 표시 |
| **설정 지옥** | 100개 옵션 한 화면에 나열 | 역할별 분리 (Admin: NEXUS/부서/에이전트/비용, User: 프리셋/비서 토글). 기본값 제공. 각 관리 페이지 내 탭 분리 + "고급 설정" 접기(Collapsible) 패턴. 필수 설정만 초기 노출, 세부 옵션은 펼치기 |
| **모바일 우선 강요** | 복잡한 B2B 도구를 모바일에 억지로 맞춤 | 데스크톱 우선. 모바일은 읽기/간단 명령만 (PWA Phase 3). NEXUS 캔버스는 데스크톱 전용 |
| **과잉 게이미피케이션** | 뱃지, 레벨, 리워드 남발 → 프로 사용자 이탈 | 업무 도구 톤 유지. 만족도 통계는 데이터 기반, 게임 요소 아님 |
| **실시간 알림 폭격** | Slack 피로 — 모든 이벤트에 알림 | WebSocket 이벤트는 현재 화면 관련만. 텔레그램 알림은 옵트인 |

### 4.4 v1 디자인 자산 계승 (v1 Design Legacy)

v1에서 검증된 디자인 패턴 중 v2에서 유지할 것:

| v1 패턴 | 검증 결과 | v2 계승 방식 |
|---------|----------|-------------|
| 군사 메타포 (사령관실, 작전현황, 통신로그) | 사용자가 "전문적이고 진지한 느낌" 선호 | 네이밍 유지: 사령관실→허브, 작전현황→대시보드. 톤은 유지하되 UI는 모던화 |
| 다크 테마 | 장시간 사용 시 눈 피로 감소. 전문적 인상 | `bg-slate-900` 기반 다크 테마 기본. 라이트 모드는 Phase 3 |
| 슬래시 명령 체계 | 8종 슬래시 명령(분석, 토론, 보고서 등) 자연스러운 사용 패턴 형성 | `/분석`, `/토론`, `/보고서`, `/검색`, `/비교`, `/번역`, `/요약`, `/실행` 동일 체계 유지 |
| @멘션 에이전트 지정 | "비서실장 거치지 않고 직접 시키고 싶을 때" 필수 | `@에이전트명` 동일 지원. 자동완성 드롭다운 |
| 통신로그 4탭 | 활동/통신/QA/도구 분류가 디버깅에 유용 | 4탭 구조 유지. UI만 모던화 (Tailwind 기반) |

## 6. Defining Experience — CORTHEX가 유일한 이유

### 6.1 핵심 차별화 요소 (Unique Value Propositions)

| # | 차별화 포인트 | 경쟁 제품 한계 | CORTHEX v2 해결 |
|---|-------------|-------------|----------------|
| 1 | **조직도 = 위임 경로 기반** | ChatGPT/Claude: 에이전트 목록일 뿐, 위임 관계 없음 | NEXUS 조직도가 `call_agent` 위임 경로의 기반 구조 제공. 비서실장은 조직도 기반으로 라우팅하되 LLM 판단으로 최적 경로 선택 |
| 2 | **N단계 계급 시스템** | 다른 플랫폼: 에이전트가 평등 (flat). 비용/품질 차등 불가 | Admin이 계급 수/이름/색상/모델 자유 설정. 전략적 비용 최적화 (상위 계급 = 고급 모델, 하위 = 경량) |
| 3 | **비서실장 선택제** | 고정 라우터 or 직접 지정만 | Human 직원이 비서실장 활성화/비활성화 토글. 두 UX 패턴 자유 전환 |
| 4 | **실시간 위임 체인 시각화** | "처리 중..." 블랙박스 | SSE 핸드오프 트래커: 비서실장 → CIO → 전문가 위임 체인 실시간 노드 표시. 각 에이전트 진행 상태 |
| 5 | **동적 조직 관리** | 29명 고정 에이전트 (v1), 경쟁사도 고정 프리셋 | Admin이 부서/에이전트를 자유 생성/편집/삭제. 템플릿으로 빠른 시작, 이후 커스터마이징 |
| 6 | **에이전트별 비용 투명성** | 월정액 or 총 사용량만 | `done` 이벤트마다 costUsd/tokensUsed 표시. 에이전트별/부서별 비용 집계 대시보드 |

### 6.2 "Aha!" 모멘트 설계

사용자가 CORTHEX의 차별성을 체감하는 핵심 순간들:

```
Aha! #1: "조직이 진짜 움직이잖아!" (온보딩 5분차)
  트리거: 첫 명령 → 핸드오프 트래커에서 4명 병렬 작업 관찰
  감정: 불신 → 경이
  UX 요소: 트래커 노드가 하나씩 추가되며 병렬 분기 시각화
  설계 원칙: 과정을 보여줘야 신뢰가 생긴다

Aha! #2: "부서를 내 손으로 만들었어!" (NEXUS 첫 편집)
  트리거: 캔버스 더블클릭 → 부서 노드 생성 → 에이전트 드래그 배치
  감정: 탐색 → 창조적 흥분
  UX 요소: 즉각적 노드 생성 + 드래그&드롭 + ELK 자동정렬
  설계 원칙: 직접 조작이 통제감을 만든다

Aha! #3: "비용이 이렇게 투명하구나" (첫 비용 확인)
  트리거: 응답 완료 후 $0.0234 비용 뱃지 확인
  감정: 비용 불안 → 안심
  UX 요소: 소수점 4자리 정밀 비용 + 예산 프로그레스 바
  설계 원칙: 숨기지 않으면 신뢰가 생긴다

Aha! #4: "에이전트 성격을 내가 정한다고?" (Soul 편집)
  트리거: 에이전트 Soul(시스템 프롬프트) 직접 수정 → 톤 변화 확인
  감정: 호기심 → 통제감
  UX 요소: Soul 에디터 (textarea) + 즉시 반영 테스트 명령
  설계 원칙: 커스터마이징이 소유감을 만든다

Aha! #5: "잘못 보냈어도 다시 보내면 돼" (잘못된 라우팅 복구)
  트리거: 비서실장이 잘못된 부서로 라우팅 → 사용자가 @멘션으로 직접 재지정
  감정: 좌절 → 통제감 회복
  UX 요소: @멘션 자동완성 + 이전 응답 폴드 + 새 명령 즉시 처리
  설계 원칙: 실패 복구가 쉬우면 실패를 두려워하지 않는다
```

## 8. Design Directions — 미적 접근법

### 8.1 Direction A: "Command Center" (추천 ★)

**컨셉**: 군사 지휘센터의 긴장감 있는 전문성. v1의 "사령관실" 감성을 현대적으로.

```
색상: Slate-950 기반 극 다크. 액센트는 차가운 Blue-500
폰트: Pretendard (단정, 권위적) + JetBrains Mono (기술적)
감정: 통제, 정밀, 신뢰
```

| 요소 | 디테일 |
|------|--------|
| 배경 | `bg-slate-950` (#020617) — 거의 검정, 깊은 어둠 |
| 카드 | `bg-slate-800/80 backdrop-blur-sm` — 반투명 글라스 |
| 테두리 | `border-slate-700` — 미묘한 구분선 |
| 액센트 | `blue-500` (#3B82F6) — 차가운 파란, 정보 디스플레이 느낌 |
| 데이터 | `tabular-nums`, 고정폭 숫자 정렬, 밀집 테이블 |
| 트래커 | 노드 체인 + 펄스, 레이더 스크린 느낌 |
| NEXUS | 다크 캔버스 + 밝은 노드, 회로도 느낌 |
| 전체 톤 | Bloomberg Terminal + Linear + 전투지휘소 |

**장점**: v1 사용자 친숙, 전문적 인상, 정보 밀도 높음
**단점**: 신규 사용자에게 위압적일 수 있음
**적합 대상**: 파워유저, 1인 사업가, 데이터 중심 사용자

### 8.2 Direction B: "Productivity Suite"

**컨셉**: Linear/Notion 스타일의 깔끔한 업무 도구. 접근성 높고 모던.

```
색상: Slate-900 기반 (A보다 밝음). 액센트는 따뜻한 Violet-500
폰트: Inter (모던, 가독성) + Fira Code
감정: 효율, 정돈, 친근
```

| 요소 | 디테일 |
|------|--------|
| 배경 | `bg-slate-900` (#0f172a) — 다크 그레이 (완전 검정은 아님) |
| 카드 | `bg-slate-800 border border-slate-700` — 솔리드, 명확한 구분 |
| 테두리 | `border-slate-700` + `rounded-xl` (더 둥근 모서리) |
| 액센트 | `violet-500` (#8B5CF6) — 따뜻한 보라, 창의적 느낌 |
| 데이터 | 여유로운 간격, 큰 숫자, 아이콘 중심 |
| 트래커 | 심플 스텝 인디케이터, Notion 프로그레스 바 느낌 |
| NEXUS | 밝은 캔버스 + 그림자 카드, Figma 느낌 |
| 전체 톤 | Linear + Notion + Vercel Dashboard |

**장점**: 친근, 낮은 학습 곡선, 모던
**단점**: v1 "전투지휘소" 감성 상실, 정보 밀도 낮음
**적합 대상**: 팀 관리자, 신규 사용자, 디자인 감각 중시

### 8.3 Direction C: "Hybrid Minimal" (절충안)

**컨셉**: Command Center의 다크 톤 + Productivity Suite의 깔끔함. 최선의 절충.

```
색상: Slate-950/900 혼합. 액센트 Blue-500 (메인) + Emerald-500 (보조)
폰트: Pretendard + JetBrains Mono (A와 동일)
감정: 전문적이되 위압적이지 않음
```

| 요소 | 디테일 |
|------|--------|
| 배경 | `bg-slate-950` (외곽) + `bg-slate-900` (패널) — 깊이감 |
| 카드 | `bg-slate-800 rounded-lg` — 솔리드, 적당한 둥글기 |
| 테두리 | `border-slate-700` — 미묘하지만 명확 |
| 액센트 | `blue-500` (CTA, 링크) + `emerald-500` (성공, 데이터) |
| 데이터 | 밀집+여유 혼합. 대시보드=밀집, 설정=여유 |
| 트래커 | 노드 체인 (A 스타일) + 심플 에이전트 아바타 (B 스타일) |
| NEXUS | 다크 캔버스 (A 스타일) + 깔끔한 노드 카드 (B 스타일) |
| 전체 톤 | Vercel + GitHub + Linear 혼합 |

**장점**: 두 방향의 장점 결합, 유연
**단점**: 명확한 개성이 부족할 수 있음
**적합 대상**: 대부분의 사용자

### 8.4 최종 방향 선택: Direction A "Command Center" ★

**선택 근거:**
1. v1 사용자(현재 유일한 사용자 = CEO)가 "전투지휘소" 감성을 선호
2. "나만의 AI 회사" 메타포와 CEO 통제감에 가장 부합
3. 정보 밀도가 높아 파워유저에게 적합
4. Bloomberg/Datadog 참조 — 데이터 중심 도구의 표준
5. Design System (Section 5)에서 이미 이 방향으로 토큰 정의 완료

**완화 조치 (위압감 방지):**
- 온보딩 템플릿으로 초기 복잡도 감소
- 빈 상태에 친절한 안내 메시지
- 점진적 기능 노출 (처음에는 허브+NEXUS만, 나머지는 사이드바 접기)
- Phase 3에서 라이트 모드 추가

**Admin ↔ User 앱 시각 구분:**
- User 앱 (/app): 순수 slate 톤, 액센트 `blue-500`
- Admin 콘솔 (/admin): 헤더에 `bg-indigo-950/30` 미묘한 인디고 틴트 + "관리자" 뱃지. 즉각적 컨텍스트 전환 인지
- `backdrop-blur-sm`: 프로그레시브 인핸스먼트. `@supports(backdrop-filter: blur())` 미지원 시 `bg-slate-800` 폴백

## 10. Component Strategy — Atomic Design 기반 컴포넌트 체계

### 10.1 Atomic Design 계층

```
Atoms → Molecules → Organisms → Templates → Pages
─────   ─────────   ─────────   ─────────   ─────
Button  SearchBar   AgentCard   HubLayout   /app/hub
Badge   InputGroup  HandoffNode DashLayout  /app/dashboard
Avatar  TabBar      NexusNode   AdminLayout /admin/nexus
Icon    Dropdown    ChatBubble  CanvasLayout/admin/agents
Spinner NavItem     CostBadge   ListDetail  /admin/depts
Toggle  FormField   SummaryCard FormLayout  /admin/creds
```

### 10.2 Atoms (원자 컴포넌트)

| 컴포넌트 | Props | Variants | 위치 |
|---------|-------|----------|------|
| `Button` | `variant`, `size`, `loading`, `icon`, `disabled` | primary, secondary, ghost, danger, icon, loading | `packages/ui/` |
| `Badge` | `variant`, `label`, `dot` | success, error, warning, info, tier, cost, model | `packages/ui/` |
| `Avatar` | `name`, `src`, `size`, `status` | sm(24), md(32), lg(40). 상태: online/offline/busy | `packages/ui/` |
| `Icon` | `name`, `size`, `className` | Lucide 아이콘. sm(16), md(20), lg(24), xl(32) | `packages/ui/` |
| `Spinner` | `size`, `className` | sm, md, lg. animate-spin | `packages/ui/` |
| `Toggle` | `checked`, `onChange`, `disabled`, `label` | — | `packages/ui/` |
| `Input` | `type`, `placeholder`, `error`, `disabled` | text, password, search | `packages/ui/` |
| `Textarea` | `rows`, `placeholder`, `maxLength` | — | `packages/ui/` |

### 10.3 Molecules (분자 컴포넌트)

| 컴포넌트 | 구성 Atoms | Props | 사용처 |
|---------|-----------|-------|--------|
| `SearchBar` | Input + Icon(Search) + Button(clear) | `value`, `onChange`, `placeholder` | 에이전트/부서/통신로그 검색 |
| `InputGroup` | Input + Label + ErrorMsg | `label`, `error`, `required` | 모든 폼 필드 |
| `TabBar` | Button[] (ghost variant) | `tabs`, `activeTab`, `onChange` | 통신로그 4탭, 설정 탭 |
| `Dropdown` | Button + 팝오버 리스트 | `options`, `value`, `onChange` | 모델/계급/부서 선택 |
| `NavItem` | Icon + Label + Badge(선택) | `icon`, `label`, `active`, `badge` | 사이드바 네비게이션 |
| `FormField` | Label + Input/Textarea/Dropdown + Help | `label`, `help`, `error`, `children` | Admin 폼 전체 |
| `CostDisplay` | Badge + Text | `costLabel`, `tokenLabel` | 응답 하단 비용 표시. 포맷팅은 앱 레벨 `useCostFormat()` 훅에서 처리 → 순수 UI 컴포넌트 |

```
│ ── Phase 2 ──   │  text-slate-500 + 🔒 아이콘
│ 🔒 전략실        │  hover: tooltip "Phase 2에서 활성화"
│ 🔒 AGORA        │
```

### 11.5 실시간 UX 패턴

**SSE 스트리밍 (허브 응답):**
```
수신: EventSource + onmessage 핸들러
상태: useReducer (Idle→Sending→Accepted→Processing→Handoff→Streaming→Done)
렌더링: 마크다운 청크 → React state append → 실시간 렌더링
최적화: requestAnimationFrame 배치, 60fps 유지
메모리: 메시지 100개 초과 시 오래된 메시지 virtualize (react-window)
```

**WebSocket (실시간 알림):**
```
연결: /ws/events (7채널 멀티플렉싱)
채널: agent-status, session-update, cost-alert, nexus-change,
      dept-change, system-notification, heartbeat
재연결: 지수 백오프 1→2→4→8→16→30초
하트비트: 30초 간격 ping/pong, 2회 실패 시 재연결

UI 반영:
  agent-status → AgentCard 상태 뱃지 업데이트 (online/offline)
  cost-alert → Toast 경고 "예산 90% 도달"
  nexus-change → User의 /app/nexus 읽기전용 뷰 갱신 (Admin 편집 반영)
```

### 11.6 로딩 & 빈 상태 패턴

**로딩 상태:**
```
페이지 로딩: 중앙 Spinner (lg, 32px) + "로딩 중..." text-slate-400
테이블 로딩: 스켈레톤 행 3줄 (bg-slate-700/50, animate-pulse, h-10)
```

### 12.1 반응형 브레이크포인트

| 이름 | 범위 | Tailwind | 레이아웃 변경 |

### 12.3 접근성 (WCAG 2.1 AA)

**색상 대비:**

| 조합 | 전경 | 배경 | 대비 비율 | WCAG AA |
|------|------|------|----------|---------|
| 본문 텍스트 | `#f1f5f9` (slate-100) | `#0f172a` (slate-900) | 15.1:1 | ✅ Pass |
| 보조 텍스트 | `#94a3b8` (slate-400) | `#0f172a` (slate-900) | 5.5:1 | ✅ Pass |
| 비활성 텍스트 | `#64748b` (slate-500) | `#0f172a` (slate-900) | 3.7:1 | ⚠️ AA Large only |
| 액센트 링크 | `#3B82F6` (blue-500) | `#0f172a` (slate-900) | 5.2:1 | ✅ Pass |
| 에러 텍스트 | `#EF4444` (red-500) | `#1e293b` (slate-800) | 4.6:1 | ✅ Pass |
| 성공 텍스트 | `#10B981` (emerald-500) | `#1e293b` (slate-800) | 4.8:1 | ✅ Pass |

> 비활성 텍스트(`slate-500`)는 Large text (18px+ or 14px bold)에서만 AA 충족. 일반 텍스트로 사용 시 `slate-400`으로 대체.

**키보드 네비게이션:**

| 키 | 동작 |
|----|------|
| `Tab` | 다음 인터랙티브 요소로 이동 |
| `Shift+Tab` | 이전 요소로 이동 |
| `Enter` | 버튼 클릭, 링크 활성화 |
| `Space` | 체크박스/토글/라디오 전환 |
| `Escape` | 모달/드롭다운/팝오버 닫기 |
| `↑↓` | 드롭다운/메뉴 항목 탐색 |
| `↑↓` + 슬래시 메뉴 | 슬래시 명령 선택 |
| `Home/End` | 리스트 처음/끝으로 이동 |

**포커스 관리:**
```
모든 인터랙티브 요소: focus-visible:ring-2 focus-visible:ring-blue-500/50
포커스 트랩: 모달 열림 시 모달 내부에만 Tab 순환
포커스 복원: 모달 닫힘 시 트리거 버튼으로 포커스 복원
스킵 링크: 페이지 최상단 "메인 콘텐츠로 건너뛰기" (sr-only, focus:not-sr-only)
```

**ARIA 속성:**

| 컴포넌트 | ARIA |
|---------|------|
| 사이드바 | `<nav aria-label="메인 네비게이션">` |
| 탭바 | `role="tablist"`, 탭 `role="tab" aria-selected`, 패널 `role="tabpanel"` |
| 모달 | `role="dialog" aria-modal="true" aria-labelledby="modal-title"` |
| Toast | `role="alert" aria-live="polite"` |
| 드롭다운 | `role="listbox"`, 옵션 `role="option" aria-selected` |
| 핸드오프 트래커 | `aria-label="위임 진행 상황"`, 각 노드 `aria-label="{에이전트명}: {상태}"` |
| 로딩 스피너 | `role="status" aria-label="로딩 중"` |
| 차트 | `aria-label="{차트 제목}"` + 대체 데이터 테이블 (`sr-only`) |
| SSE 스트리밍 | `aria-live="polite"` + 스트리밍 중 `aria-busy="true"` (완료 시 false) |

**모션 접근성:** `prefers-reduced-motion` 미디어 쿼리 지원 (Section 5.6 참조). 모든 애니메이션 0.01ms로 단축. 핸드오프 트래커 펄스, 스트리밍 타이핑 효과 등 비활성화.

### 12.5 성능 지침

| 지표 | 목표 | 구현 |
|------|------|------|
| FCP (First Contentful Paint) | < 1.5초 | 코드 분할 (React.lazy), 폰트 preload |
| LCP (Largest Contentful Paint) | < 2.5초 | 이미지 lazy load, 스켈레톤 UI 즉시 표시 |
| CLS (Cumulative Layout Shift) | < 0.1 | 이미지/카드 고정 높이, 폰트 display:swap |
| TTI (Time to Interactive) | < 3.5초 | 번들 분석 + tree-shaking + 청크 분할 |
| SSE 응답 시작 | < 100ms | accepted 이벤트 즉시 전송 (서버) |
| 메시지 virtualization | 100개+ | react-window, 뷰포트 외 메시지 언마운트 |
| NEXUS 렌더링 | 100 노드 < 500ms | React Flow 가상화, ELK 웹워커 |
