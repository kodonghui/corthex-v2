## 기술적 결정

| 결정 | 이유 |
|------|------|
| DFS 순환 감지 | 인접 리스트 + DFS가 위임 규칙 수(최대 50개)에서 O(V+E) 충분히 빠름 |
| Promise.allSettled (병렬 위임) | 하나가 실패해도 나머지 결과를 보존 |
| delegation-chain 이벤트 분리 | 기존 delegation-start/end와 독립적으로 체인 경로 전체를 한 번에 전송 |
| 응답 크기 10,000자 제한 | LLM max_tokens 2048이지만 한글 특성상 안전장치 필요 |
| 역할 체크 위치: 핸들러 최상단 | 불필요한 DB 쿼리를 방지하기 위해 가장 먼저 체크 |
