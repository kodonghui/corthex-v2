## 잘 된 점
- TeamCreate 오케스트레이터 프로토콜로 5개 스토리 전부 BMAD 5단계 완수 — 컨텍스트 피로 없음
- 14-3/14-4 병렬 투입 성공 — 동시에 다른 파일 수정했지만 충돌 없이 한번에 커밋
- sns-schedule-checker 워커 패턴이 야간작업(Epic 11) 워커와 동일하여 빠르게 구현
- 부분 실패 허용 설계 (14-2): 이미지 생성 실패해도 텍스트 콘텐츠는 저장 — 실용적 UX
- 코드 리뷰에서 테넌트 격리 누락 조기 발견 (14-4 HIGH: PUT/DELETE WHERE에 companyId 추가)

## 교훈
- **병렬 팀원 동일 파일 수정 주의**: 14-3과 14-4가 sns.tsx, schema.ts 동시 수정 — 이번엔 운좋게 충돌 없었으나, 같은 파일 수정하는 스토리는 직렬 처리가 안전
- **워커 동시성**: sns-schedule-checker에서 SELECT 후 UPDATE 사이 중복 발행 가능 — CR에서 상태 먼저 UPDATE 후 발행하도록 수정
- **타입체크 필수**: Epic 12 소급 작업에서 type-check 안 돌려서 CI 빨간불 — 이후 커밋 전 `turbo build type-check` 필수화
- **스키마 없는 컬럼 참조 방지**: agents.level, departments.parentDepartmentId 등 실제 DB에 없는 컬럼을 코드에 쓰면 런타임은 되지만 type-check에서 걸림

## 아키텍처 노트
- SNS 예약 발행: 60초 폴링 워커 (sns-schedule-checker) → scheduledAt <= now인 콘텐츠 자동 발행
- AI 이미지: credential-vault에서 OpenAI API 키 조회 → DALL-E 3 호출 → URL 반환
- 멀티 계정: snsAccounts 테이블 (companyId + platform + credentials) → snsContents.snsAccountId FK
- A/B 테스트: variantOf 자기참조 FK → 원본-변형 관계, AI 자동 변형 생성 (톤/CTA/해시태그 변경)
- 마이그레이션: 0021 (scheduled), 0022 (multi-account), 0023 (ab-test)
