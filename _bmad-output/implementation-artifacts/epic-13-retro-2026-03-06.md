## 잘 된 점
- files 테이블이 이미 스키마에 존재하여 마이그레이션 최소화
- api.upload 메서드를 13-2에서 추가하여 13-3에서 즉시 재사용
- JSON 컬럼(attachmentIds) 방식이 조인 테이블보다 간단하게 작동
- TEA 테스트가 getMimeIcon 매핑 순서 버그를 조기 발견

## 교훈
- **Office XML MIME 타입 매칭 순서 주의**: `openxmlformats-officedocument`에 "document" 포함 → sheet/presentation을 word보다 먼저 검사해야 함
- **FormData Content-Type**: fetch API에서 FormData 전송 시 Content-Type 헤더를 수동 설정하면 boundary가 누락됨 → isFormData 분기 필요
- **path traversal 방어**: getFilePath에 startsWith 검증 추가 — 방어적 코딩 습관화
- **drizzle-kit generate 대화형 프롬프트**: 자동화 환경에서 수동 마이그레이션 SQL 직접 작성이 더 안정적

## 아키텍처 노트
- 파일 저장: `uploads/{companyId}/{yyyy-mm}/{uuid}.{ext}` — 테넌트 격리 + 월별 정리
- 소프트 삭제: `isActive=false` — 데이터 복구 가능
- 파일 크기 제한: 10MB — 서버 메모리 보호
