## 잘 된 점

### 1. 점진적 인프라 재사용 (Incremental Reuse)
- 10-2: ChatArea/useChatStream/ws-store 인프라를 기존 에픽에서 그대로 재사용 (코드 중복 0)
- 10-3: kis-auth.ts + credential-vault를 Epic 9 도구 핸들러에서 재사용
- 10-5: 리포트 다운로드 패턴(blob + Content-Disposition)을 reports.ts에서 재활용
- 10-6: **서버 API 변경 없이** 기존 `/prices?codes=` 엔드포인트로 멀티 종목 지원

### 2. URL 기반 상태 관리 일관성
- 모든 스토리에서 URL search params 활용: `?stock=`, `?compare=`, `?bt=`, `?sp=`, `?lp=`
- 이 패턴 덕에 10-8(공유 URL)이 거의 무료로 구현됨 — `window.location.href` 복사만으로 충분
- 딥링크 + 북마크 + 새로고침 지원이 자연스럽게 확보됨

### 3. 체계적 에러 코드 및 API 패턴
- 에러 코드 체계적 네임스페이스: STRATEGY_010~052
- 모든 엔드포인트: `authMiddleware → zValidator → Drizzle ORM` 일관된 패턴
- UUID params 항상 `z.string().uuid()` 검증 (10-1에서 학습 후 전 스토리 적용)

### 4. 마이그레이션 규율
- 0011~0015 순차 번호 관리
- 10-4에서 저널 태그 ↔ 파일명 일치 규칙 학습 → 이후 모든 스토리에서 인용 및 적용

### 5. 컴포넌트 분리 아키텍처
- `strategy/` 디렉토리 ~10개 컴포넌트가 깔끔한 책임 분리로 성장
- 백테스트 엔진(10-7)은 순수 함수 모듈 — React 의존성 없이 테스트 가능

### 6. 코드 리뷰 학습 사항 전파
- 각 스토리의 Dev Notes에 이전 스토리 학습 사항을 명시적으로 인용
- 예: "10-1: UUID params에 zValidator 필수, 중첩 button 금지" → 이후 모든 스토리에서 적용

## 어려웠던 점 / 개선 필요

### 1. Stale URL State (10-8)
- `new URL(window.location.href)`로 공유 URL 생성 시, 백테스트 비활성화 상태에서도 bt/sp/lp 파라미터가 잔류
- 코드 리뷰에서 발견 → else 브랜치로 param 삭제 추가하여 수정

### 2. eslint-disable 사용 (10-8)
- useEffect 의존성 배열 문제를 eslint-disable로 우회한 초기 구현
- 코드 리뷰에서 `useRef` 패턴으로 교체 → 올바른 해결

### 3. 유틸 함수 중복 (10-6)
- `formatPrice()`, `formatVolume()`, `isMarketOpen()`이 chart-panel과 comparison-panel에 복사-붙여넣기
- 공유 utils 파일로 추출해야 하는 기술 부채

### 4. useEffect 의존성 관리 (10-2, 10-8)
- 여러 스토리에서 dependency array 관리에 주의가 필요했음
- autoRun 패턴(10-8)에서 `useRef` 가드가 필수적임을 학습

### 5. WebSocket 브로드캐스트 순서 (10-9)
- DELETE 요청에서 소유권 검증 전에 브로드캐스트가 호출되는 버그
- 코드 리뷰에서 발견 → 삭제 성공 후로 이동하여 수정

### 6. 자기 자신 공유 표시 (10-9)
- 공유 대상 목록에 현재 사용자가 표시되는 UX 문제
- `useAuthStore`에서 currentUserId를 가져와 필터링하여 수정

## 핵심 기술 결정

1. **프론트엔드 백테스트 연산** (10-7): 서버 부하 없이 브라우저에서 캔들 데이터로 직접 연산. 서버는 결과 저장만 담당
2. **Last-write-wins 협업** (10-9): CRDT/OT 없이 WebSocket 알림 + React Query 무효화만으로 실시간 동기화
3. **lightweight-charts 선택** (10-3): ~45KB gzip, React 래퍼 불필요, 다크 모드 기본 지원
4. **chatSessions.metadata JSONB** (10-2): 별도 테이블 없이 기존 세션에 유연한 JSONB 컬럼으로 종목 컨텍스트 저장
5. **URL 기반 비교 모드** (10-6): `?compare=` vs `?stock=`로 상태 분리, 조건부 패널 렌더링
