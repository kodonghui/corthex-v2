## 교훈

### 1. SDK 소스 분석의 중요성
- **문제**: Claude Agent SDK `query()`가 `ContentBlock[]` systemPrompt를 지원하는지 문서만으로는 확인 불가
- **발견**: SDK 소스(`sdk.mjs`) 직접 분석 → `if (typeof Y === 'string') H = Y` → ContentBlock[] 전달 시 빈 문자열 처리됨
- **결과**: Path B(messages.create() 직접 호출)로 전환. agent-loop.ts 전면 재작성 필요했지만, 결정 트리가 미리 준비되어 있어 혼란 없이 진행
- **교훈**: 외부 SDK에 의존하는 기능은 반드시 PoC 선행. 문서가 아닌 소스 코드 기준으로 판단

### 2. agent-loop.ts 전면 재작성의 파급 효과
- **문제**: Path B 선택 → query() → messages.create() 전환 시 Hook 파이프라인(PreToolUse, PostToolUse, Stop) 자동 발화 없음
- **대응**: Hook 호출을 agent-loop.ts 내부에서 수동으로 구현
- **교훈**: 핵심 인프라 파일(agent-loop.ts) 변경은 상류/하류 의존성을 사전에 완전히 매핑해야 함. 15-1 story에서 Hook 재구현이 명시적으로 계획되어 있었기 때문에 무사히 완료

### 3. 21SP 스토리의 적정성
- **문제**: Story 15-3이 21SP — DB 마이그레이션 + 엔진 통합 + Admin UI + ARGOS cron + CI 경계 검사까지 단일 스토리
- **결과**: 완료는 했으나, 스토리 범위가 넓어 리뷰와 테스트가 복잡해짐
- **교훈**: 13SP 이상 스토리는 분할 검토. 예: DB+엔진(8SP) + Admin UX(5SP) + 인프라(8SP)로 3분할 가능했음

### 4. E8 경계 설계의 효과
- **성공**: Tool Cache(lib/) vs Semantic Cache(engine/) 배치 결정(D18, D19)이 E8 경계를 자연스럽게 준수
- **CI 경계 검사**: engine-boundary-check.sh에 semantic-cache 가드 추가 → 외부 직접 import 차단
- **교훈**: 아키텍처 결정(D-number)을 story 작성 시점에 확정하면 구현 시 경계 위반 방지가 자연스러움

## 아키텍처 노트

### Layer 1: Prompt Cache (Anthropic Server-side)
- `cache_control: { type: 'ephemeral' }` → system prompt에 적용
- 5분 TTL (Anthropic 서버 관리), 캐시 읽기 0.1× 비용
- agent-loop.ts: `anthropic.messages.create()` 직접 호출 (SDK query() 미사용)
- cost-tracker: `cacheReadInputTokens`, `cacheCreationInputTokens` 로깅

### Layer 2: Tool Result Cache (In-Memory)
- `lib/tool-cache.ts`: CacheStore interface + InMemoryMap (LRU, 10K max)
- `lib/tool-cache-config.ts`: 7개 도구별 TTL (1분~24시간)
- `lib/tool-executor.ts`: withCache() 단일 통합 지점
- 1분 cleanup + heapUsed 100MB 경고

### Layer 3: Semantic Cache (pgvector)
- `0051_semantic-cache.sql`: semantic_cache 테이블 + VECTOR(768) + HNSW index
- `engine/semantic-cache.ts`: checkSemanticCache() (cosine >= 0.95) + saveToSemanticCache()
- Per-agent `enableSemanticCache` toggle (Admin UI, default: OFF)
- ARGOS daily cleanup (`lib/semantic-cache-cleanup.ts`)
- `lib/credential-patterns.ts`: scrubber + cache 공유 패턴

### 마이그레이션
- `0051_semantic-cache.sql`: semantic_cache 테이블 + agents.enable_semantic_cache 컬럼
