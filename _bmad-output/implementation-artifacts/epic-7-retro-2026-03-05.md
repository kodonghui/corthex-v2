## 잘 된 점

1. **기존 백엔드 코드 최대 활용** — reports.ts에 9개 엔드포인트가 이미 구현되어 있었고, 알림/활동 로그/다운로드만 추가. 바퀴 재발명 없이 확장
2. **Epic 6 알림 패턴 즉시 재활용** — `createNotification` fire-and-forget 패턴을 보고서 제출/검토/코멘트 3곳에 일관 적용
3. **cursor 페이지네이션 패턴 재적용** — Epic 4 채팅, Epic 6 작전일지에 이어 코멘트에도 `limit + before` 패턴 재사용. DESC → reverse 기법 동일
4. **공유 컴포넌트 추출 (7-4)** — reports.tsx의 287자 인라인 마크다운 CSS를 MarkdownRenderer로 추출. soul-editor.tsx도 통합하여 코드 중복 제거
5. **ConfirmDialog 교체 완성** — window.confirm → ConfirmDialog 교체 완료 (CEO 보고 + 삭제). Epic 4-6에서 만든 컴포넌트 활용
6. **채팅 메시지 링크 렌더링** — renderTextWithLinks 헬퍼로 마크다운 링크를 SPA navigate로 처리. 에이전트가 생성한 보고서로 즉시 이동 가능

## 어려웠던 점 / 개선 필요

1. **reports.tsx 전면 리라이트 크기** — 474줄 → 전면 리라이트. 기존 코드를 완전 교체하는 접근은 리스크가 높음. 점진적 리팩토링이 바람직
2. **JSX 중첩 구조 오류** — 7-2에서 에이전트 연결 버튼 추가 시 `</div>`/`)}` 중첩 오류 발생. JSX 구조가 복잡해질수록 수동 삽입 시 실수 확률 증가
3. **테스트 부재 지속** — 프론트엔드/백엔드 모두 단위 테스트 없이 빌드 성공만으로 검증. 런타임 오류 감지 불가
4. **react-markdown 의존성 위치** — MarkdownRenderer를 @corthex/ui에 넣지 못하고 app 내부에 구현. react-markdown이 app 전용 의존성이라 공유 패키지에 추가하려면 의존성 구조 변경 필요
