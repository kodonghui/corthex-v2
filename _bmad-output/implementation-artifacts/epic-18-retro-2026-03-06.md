## 잘 된 점

### 1. Stub-then-Real 패턴의 성공적 적용
- 18-1에서 연결 테스트/도구 목록을 stub으로 구현하고, 18-2에서 실제 MCP JSON-RPC로 교체
- 프론트엔드는 18-1에서 이미 완성되어 18-2에서 **변경 0건** — 백엔드만 교체하면 자동 동작
- 이 패턴 덕분에 스토리 간 의존성 관리가 깔끔했고, 각 스토리가 독립적으로 완결

### 2. 기존 인프라 재활용률 극대화
- **도구 실행 엔진(ai.ts)**: 기존 tool_use 루프에 MCP 핸들러만 추가 — 전체 구조 변경 없음
- **WebSocket 멀티플렉싱**: tool-progress 이벤트를 기존 스트리밍 채널로 자연스럽게 전달
- **activity-logger.ts**: 감사 로그를 위한 별도 모듈 불필요 — 기존 logActivity 재활용
- **authMiddleware + adminOnly**: 기존 미들웨어 조합으로 권한 분리 깔끔히 해결
- **settings.tsx 탭 구조**: MCP 탭을 기존 패턴에 자연스럽게 추가

### 3. 보안 레이어의 체계적 구축
- 18-1에서 기본 SSRF 방지 → 18-4에서 production/dev 분기 + IPv6 + 클라우드 메타데이터까지 강화
- Rate limit(분당 20회) + 감사 로그 + 응답 크기 제한(100KB) + adminOnly가 한 스토리에서 일관 적용
- 보안 테스트 187건으로 에픽 내 가장 많은 테스트 — 보안 스토리에 가장 많은 커버리지 할당은 올바른 전략

### 4. MCP 프로토콜의 깔끔한 추상화
- `mcp-client.ts` 하나의 모듈에 JSON-RPC 요청, SSE 스트리밍, SSRF 방지, 응답 크기 제한 캡슐화
- `mcpListTools`, `mcpCallTool`, `mcpCallToolStream` 세 함수로 전체 MCP 프로토콜 커버
- Content-Type 자동 분기(JSON/SSE)로 서버 호환성 극대화

### 5. BMAD 5단계 파이프라인 안정성
- 4개 스토리 모두 create-story → dev → TEA → QA → code-review 일관 실행
- Code review에서 발견된 이슈(18-1: SSRF, 이중 fetch, isActive 체크 등) 자동 수정
- TEA 리스크 기반 분석이 보안 스토리(18-4)에 더 많은 테스트를 자연스럽게 할당

## 교훈

### 1. 도구 이름 충돌 방지 전략의 한계
- 현재 "내장 도구 우선, 동명 MCP 도구 건너뛰기" 전략은 단순하지만 제한적
- 사용자가 내장 도구와 동일한 이름의 MCP 도구를 의도적으로 사용하고 싶은 경우 불가능
- 향후 네임스페이스 또는 우선순위 설정이 필요할 수 있음

### 2. 인메모리 Rate Limit의 서버 재시작 취약점
- `mcp-rate-limit.ts`의 슬라이딩 윈도우가 인메모리 Map — 서버 재시작 시 초기화
- 현재 단일 서버 구성에서는 문제 없으나, 스케일아웃 시 Redis 등 외부 저장소 필요
- 5분마다 만료 엔트리 정리로 메모리 누수는 방지됨

### 3. MCP 서버 실제 연동 테스트의 부재
- 모든 테스트가 mock/stub 기반 — 실제 MCP 서버와의 통합 테스트는 미실시
- JSON-RPC 프로토콜 구현의 정확성은 실제 서버 연결로만 완전히 검증 가능
- 이는 프로젝트 전체에 걸친 "통합 테스트 부재" 기술 부채의 연장선

### 4. SSE 스트리밍의 edge case
- SSE keep-alive 리셋 로직이 있지만, 매우 느린 MCP 서버(60초+)에 대한 처리가 제한적
- 부분 결과 반환 로직이 있으나, 부분 결과의 유효성 보장 메커니즘 없음
- 네트워크 불안정 환경에서의 재연결/재시도 로직 미구현

### 5. 워크플로우 실행 엔진은 여전히 Stub
- Epic 17에서 flagged된 워크플로우 실행 stub이 Epic 18에서도 그대로 유지
- MCP 도구 실행은 채팅 기반으로만 가능 — NEXUS 워크플로우 노드에서의 MCP 도구 호출은 미연결
- 이 연결은 향후 고도화 과제로 남음

## 아키텍처 노트

### 신규 컴포넌트
- **mcp-client.ts**: JSON-RPC 2.0 클라이언트 (mcpListTools, mcpCallTool, mcpCallToolStream)
- **mcp-rate-limit.ts**: 인메모리 슬라이딩 윈도우 속도 제한 (분당 20회)
- **settings-mcp.ts**: MCP CRUD + 테스트 + 실행 API (adminOnly 분리)
- **settings-mcp.tsx**: MCP 설정 UI (카드 목록 + 추가 폼 + 아코디언 도구 + 권한 기반)
- **tool-progress StreamEvent**: WebSocket 스트리밍에 tool-progress 이벤트 추가
- **tool-call-card.tsx**: 실행 중 progressText 표시 UI

### 파일 구조

### 보안
- adminOnly: 서버 등록/삭제/연결 테스트는 관리자 전용
- SSRF 방지: isPrivateUrl (production localhost 차단, IPv6, 클라우드 메타데이터)
- Rate limit: 분당 20회 (REST API + 채팅 내 도구 호출 양쪽)
- 감사 로그: 등록/삭제/실행 모두 activity_log 기록
- 응답 크기: 100KB 제한 + 잘림 메시지
- HTTP 경고: 비암호화 연결 경고 배너

- **프론트**: React + Vite SPA + TanStack Query + Tailwind
- **실시간**: WebSocket 8채널 멀티플렉싱
- **보안**: JWT 인증 + AES-256-GCM 자격증명 금고 + 테넌트 격리
