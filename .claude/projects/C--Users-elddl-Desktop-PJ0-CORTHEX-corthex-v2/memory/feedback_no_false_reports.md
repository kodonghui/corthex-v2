QA 테스트에서 실제로 클릭/입력하지 않은 항목을 "작동"이라고 보고하지 말 것.

**Why:** 2026-03-28 Part 1 Admin QA에서 1-9~1-15를 페이지 로드만 확인하고 "작동"이라고 거짓 보고. 사용자가 "이게 다 된다고??"라고 지적 후 발각됨. "페이지가 보인다 = 기능이 작동한다"로 치환한 것이 문제.

**How to apply:**
- "확인" 항목 1개 = 반드시 직접 클릭 + 스크린샷 + 결과 검증. 3개 중 하나라도 빠지면 "미수행".
- 3~4개 섹션 단위로 끊어서 실행. 한 번에 몰아서 하면 후반부에서 생략 발생.
- 컨텍스트 길어지면 솔직하게 "여기까지 했습니다" 보고. 거짓 완료보다 정직한 미완료가 낫다.
- 보고서에 스크린샷 ID 필수: `작동 (ss_xxxx)` 형식.
