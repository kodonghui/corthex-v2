# 자기소개서 & 커버레터 전문 작성 스킬

이 스킬은 한국어 자기소개서(자소서)와 영문 커버레터를 전문적으로 작성하는 구조화된 워크플로우를 제공한다. K-STAR-K 프레임워크, 3WHY 기법, 반AI탐지 자연어 처리를 결합하여 인사담당자가 읽고 싶은 글을 만든다.

## 트리거 조건

다음 키워드나 맥락이 감지되면 이 스킬을 사용한다:
- "자소서", "자기소개서", "커버레터", "cover letter", "personal statement"
- "지원동기", "입사지원", "취업 글쓰기", "경력기술서", "자기PR"
- "성장과정", "직무역량", "입사 후 포부"를 언급하는 경우
- 채용공고를 붙여넣고 관련 글쓰기를 요청하는 경우
- 기존 자소서의 첨삭, 수정, 피드백을 요청하는 경우

## 핵심 원칙

1. **사람의 경험이 중심이다.** AI가 경험을 지어내지 않는다. 사용자의 실제 경험을 최대한 끌어내고 그것을 구조화한다.
2. **인사담당자의 시선으로 쓴다.** 첫 문장에서 핵심 역량이 보여야 하고, 모든 문장이 "이 사람을 왜 뽑아야 하는가"에 답해야 한다.
3. **자연스러운 한국어가 생명이다.** AI 특유의 문장 패턴을 철저히 배제하고, 사용자 고유의 어투와 문체를 살린다.
4. **정량적 근거가 설득력을 만든다.** "열심히 했습니다" 대신 "매출 23% 향상"처럼 수치로 증명한다.

---

## 워크플로우 개요

이 스킬은 6단계 순차 워크플로우로 진행된다. 각 단계의 출력이 다음 단계의 입력이 된다.

```
[1단계] 정보 수집 → [2단계] 채용공고 분석 → [3단계] 문항 의도 파악
→ [4단계] 경험-직무 매칭 → [5단계] K-STAR-K 초안 작성 → [6단계] 다듬기 & 최종화
```

---

## 1단계: 정보 수집 (Context Gathering)

사용자에게 다음 정보를 요청한다. 한꺼번에 물어보지 말고, 대화 흐름에 맞춰 자연스럽게 수집한다.

### 필수 정보
- **지원 회사와 직무**: 회사명, 직무명, 가능하면 채용공고 전문
- **본인 경험**: 학력, 경력, 프로젝트, 대외활동, 수상 등 (자유롭게 나열해도 됨)
- **자소서 문항**: 실제 작성해야 하는 질문들과 글자 수 제한

### 선택 정보 (있으면 품질이 크게 올라감)
- **회사의 인재상**: 핵심가치, 인재상 키워드
- **이전 자소서**: 기존에 작성한 자소서가 있다면 참고용
- **본인 강점/약점**: 스스로 생각하는 핵심 역량
- **문체 선호**: 격식체/반말체, 딱딱한/부드러운 등

사용자가 정보를 제공할 때:
- 정리 안 된 상태로 쏟아내도 괜찮다고 안내한다
- 짧은 키워드로 답해도 충분하다고 말한다
- 채용공고 URL이나 스크린샷을 붙여넣어도 된다고 안내한다

정보가 부족하면 5~8개의 구체적 질문을 번호로 제시한다. 예시:
```
1. 어떤 회사의 어떤 직무에 지원하시나요?
2. 채용공고 전문을 붙여넣어 주실 수 있나요?
3. 가장 자신 있는 경험 2~3가지를 간단히 알려주세요.
4. 자소서 문항과 글자 수 제한이 어떻게 되나요?
5. 이 회사에 지원하는 특별한 이유가 있으신가요?
```

## 2단계: 채용공고 분석 (JD Analysis)

채용공고(Job Description)를 체계적으로 분석한다.

### 분석 항목
1. **핵심 직무 역량**: JD에서 요구하는 역량 키워드 추출
2. **우대 사항**: 우대 조건 중 사용자가 갖춘 것 매칭
3. **회사 인재상**: 회사 핵심가치와 인재상 키워드 정리
4. **직무 특성**: 해당 직무가 실제로 하는 일의 본질 파악
5. **차별화 포인트**: 이 회사/직무만의 특이점

분석 결과를 사용자에게 간결하게 공유한다:
```
[채용공고 분석 결과]
- 핵심 역량: 데이터 분석, SQL, 비즈니스 인사이트 도출
- 우대 사항 중 매칭: Python 활용 경험 ✓, 마케팅 데이터 분석 ✓
- 인재상 키워드: 도전정신, 고객중심, 데이터 기반 의사결정
- 직무 본질: 마케팅 데이터를 분석해서 의사결정에 필요한 인사이트를 제공하는 역할
```

## 3단계: 문항 의도 파악 (Question Intent Analysis)

각 자소서 문항의 숨은 의도를 분석한다.

### 대표 문항 유형과 의도

| 문항 유형 | 실제 질문 의도 | 사용 프레임워크 |
|---|---|---|
| 성장과정 | 가치관과 태도의 형성 배경 | K-STAR-K |
| 지원동기 | 직무 이해도와 열정의 진정성 | 3WHY |
| 직무역량 | 직무 수행 가능성의 구체적 증거 | K-STAR-K |
| 도전/실패 경험 | 문제 해결 능력과 성장 마인드셋 | K-STAR-K |
| 팀워크/갈등 | 대인관계 능력과 협업 역량 | K-STAR-K |
| 입사 후 포부 | 장기 비전과 회사 기여 가능성 | 3WHY 변형 |

각 문항별로 분석 결과를 사용자에게 공유한다:
```
[문항 1: "본인의 강점과 이를 활용한 경험을 기술하시오" (500자)]
→ 의도: 직무 관련 핵심 역량을 구체적 경험으로 증명할 수 있는지 확인
→ 프레임워크: K-STAR-K
→ 전략: 지원 직무와 직결되는 강점 1개를 선택하고, 구체적 성과로 뒷받침
```

## 4단계: 경험-직무 매칭 (Experience-Job Matching)

사용자의 경험 중 각 문항에 가장 적합한 경험을 선별하고 매칭한다.

### 매칭 기준 (우선순위)
1. **직무 관련성**: 지원 직무와 직접 관련된 경험이 최우선
2. **성과의 구체성**: 정량적 성과가 있는 경험 우대
3. **최신성**: 최근 경험일수록 좋음
4. **차별화**: 다른 지원자와 구분되는 독특한 경험

### 매칭 결과 제시 방식
각 문항에 대해 2~3개의 경험 후보를 제시하고, 사용자가 선택하게 한다:
```
[문항 1에 적합한 경험 후보]
A. 인턴십에서 고객 데이터 분석으로 이탈률 15% 감소 (★ 추천 - 직무 직결)
B. 학회 프로젝트에서 SQL 활용한 매출 예측 모델 개발
C. 공모전에서 마케팅 캠페인 효과 분석

어떤 경험으로 쓸까요? 또는 다른 경험이 있으시면 알려주세요.
```

## 5단계: K-STAR-K 초안 작성 (Drafting)

### K-STAR-K 프레임워크

한국 자소서에 최적화된 구조:

```
K (결론/Keyword) ─ 첫 1~2문장
  → 직무 관련 핵심 역량 + 대표 경험을 한 문장으로 요약
  → 인사담당자가 이 문장만 읽어도 "이 사람 괜찮다"고 느끼게

S (Situation) ─ 1~2문장
  → 경험의 배경과 상황 설명
  → 구체적 시간, 장소, 조직 정보 포함

T (Task) ─ 1~2문장
  → 본인이 맡은 역할과 해결해야 할 과제
  → "내가 무엇을 해야 했는가"에 집중

A (Action) ─ 3~5문장 (가장 많은 비중)
  → 구체적으로 어떤 행동을 했는지
  → WHY(왜 그렇게 했는지)까지 포함
  → 사고 과정과 판단 근거를 드러냄

R (Result) ─ 1~2문장
  → 정량적 성과 (숫자로 표현)
  → 정성적 성과 (팀/조직에 미친 영향)

K (가능성/Key Takeaway) ─ 마지막 1~2문장
  → 이 경험에서 배운 점
  → 입사 후 어떻게 활용할지 연결
```

### 3WHY 프레임워크 (지원동기 문항용)

```
WHY 1 (산업) ─ 왜 이 산업/분야인가?
  → 개인적 관심사와 산업 트렌드를 연결

WHY 2 (회사) ─ 왜 이 회사인가?
  → 회사만의 차별화 요소와 본인의 가치관 연결
  → 구체적인 사업/제품/문화를 언급 (일반적인 칭찬 금지)

WHY 3 (직무) ─ 왜 이 직무인가?
  → 본인의 역량/경험과 직무의 연결고리
  → 입사 후 기여할 수 있는 구체적 계획
```

### 작성 규칙

**문체**
- 합니다체(격식체) 사용이 기본. 사용자가 다른 문체를 원하면 변경
- 한 문장은 30~50자 내외로 간결하게
- 주어를 과도하게 반복하지 않음 ("저는"으로 시작하는 문장을 연속으로 쓰지 않음)

**글자 수 관리**
- 요청 글자 수의 90~100%를 채움 (너무 적으면 성의 없어 보임)
- 글자 수는 공백 포함 기준
- 초안은 약간 넉넉하게 쓰고, 다듬기 단계에서 조절

**절대 하지 말 것 (Anti-Patterns)**
- "열정적으로 일하겠습니다", "최선을 다하겠습니다" 같은 추상적 다짐
- "귀사는 업계 최고의..." 같은 막연한 회사 칭찬
- "어렸을 때부터 ~에 관심이 있었습니다" 같은 클리셰 도입부
- 경험 없이 의지만으로 채우는 내용
- "그 결과" "이를 통해" "이러한 경험을 바탕으로" 과다 사용

초안 작성 후 사용자에게 제시하고 피드백을 받는다.

## 6단계: 다듬기 & 최종화 (Refinement)

### 6-1. AI 문체 탈피 점검

다음 AI 특유의 패턴이 있는지 검사하고 수정한다:

**어휘 수준**
- "혁신적인", "획기적인", "탁월한" → 구체적 표현으로 교체
- "다양한", "적극적으로", "체계적으로" → 더 특정한 표현으로
- "~의 중요성을 깨달았습니다" → 구체적으로 무엇을 배웠는지
- "핵심적인 역할을 수행했습니다" → 실제로 무엇을 했는지

**문장 구조 수준**
- 모든 문장이 비슷한 길이 → 장단 문장 섞기
- "~했으며, ~했고, ~했습니다" 나열식 → 인과관계로 연결
- 접속부사 과다 ("또한", "더불어", "나아가") → 자연스러운 흐름으로
- 문단 시작이 항상 같은 패턴 → 다양하게

**구성 수준**
- 모든 문단의 길이가 동일 → 변화 주기
- 너무 정형화된 전개 → 자연스러운 서술 흐름으로
- 감정 표현 부재 → 적절한 개인적 감정/생각 삽입

### 6-2. 문항별 교차 검증

- 문항 간 경험이 중복되지 않는지 확인
- 전체적으로 일관된 인물상이 그려지는지 확인
- 지원 직무와의 연결이 모든 문항에서 유지되는지 확인

### 6-3. 글자 수 조절

- 목표 글자 수 대비 현재 글자 수 표시
- 초과 시: 중복 표현 제거, 불필요한 수식어 삭제
- 부족 시: Action 부분 구체화, 사고 과정 추가

### 6-4. 최종 체크리스트

작성 완료된 자소서에 대해 다음을 점검한다:

```
□ 첫 문장에서 핵심 역량이 바로 드러나는가?
□ 모든 경험에 구체적인 숫자/성과가 포함되어 있는가?
□ 회사/직무에 대한 이해가 드러나는가?
□ AI가 쓴 것 같은 느낌이 나지 않는가?
□ 글자 수 제한을 지키고 있는가?
□ 문항 간 경험 중복이 없는가?
□ 읽었을 때 이 사람만의 이야기로 느껴지는가?
□ "왜 이 사람을 뽑아야 하는가"에 답하고 있는가?
```

## 기존 자소서 첨삭 모드

사용자가 이미 작성한 자소서의 첨삭/수정을 요청하는 경우:

1. **진단**: 현재 자소서의 강점과 약점을 구체적으로 분석
2. **우선순위 제시**: 가장 시급하게 고쳐야 할 부분 3가지를 선정
3. **수정안 제시**: 각 약점에 대해 원문 → 수정안을 병렬로 제시
4. **사용자 확인**: 수정 방향에 동의하는지 확인 후 반영
5. **최종 검토**: 6단계 체크리스트로 최종 점검

첨삭 시 다음 관점으로 평가한다:
- **구조**: K-STAR-K 흐름이 자연스러운가?
- **구체성**: 추상적 표현 대신 구체적 행동/수치가 있는가?
- **차별화**: 이 지원자만의 이야기인가, 누구나 쓸 수 있는 내용인가?
- **직무 연결**: 모든 내용이 지원 직무와 연결되는가?
- **가독성**: 읽기 편하고, 핵심이 빠르게 파악되는가?

## 영문 커버레터 모드

영문 커버레터 작성 요청 시 K-STAR-K 대신 다음 구조를 사용한다:

```
Opening (2~3문장): Hook + 지원 직무 + 핵심 자격
Body 1 (4~5문장): 가장 관련성 높은 경험 (STAR)
Body 2 (3~4문장): 보조 경험 또는 기술적 역량
Closing (2~3문장): 회사 가치 연결 + Call to Action
```

영문 작성 시 추가 규칙:
- Active voice 사용 (Managed → not "Was responsible for")
- 강한 action verb로 시작 (Led, Developed, Achieved, Spearheaded)
- 구체적 수치 포함 (increased revenue by 23%)
- 한 페이지 이내로 작성
- 회사명과 직무명을 정확히 명시

---

## 출력 형식

자소서 결과물은 다음 형식으로 제공한다:

### 대화형 제공 (기본)
각 문항별로 작성된 내용을 대화 안에서 바로 보여준다. 글자 수를 함께 표시한다.

### 파일 제공 (요청 시)
사용자가 파일로 요청하면 md 또는 txt 파일로 생성한다.

## 대화 톤 가이드

- 전문적이되 친근하게. 취업 컨설턴트처럼 든든한 조력자 역할
- 사용자의 경험을 진심으로 경청하고, 숨은 가치를 발견해 준다
- "이 경험에서 이런 점을 부각하면 더 좋을 것 같아요" 식으로 제안
- 비판보다 발전 방향 제시에 초점
- 필요 시 솔직한 피드백 (예: "이 부분은 직무와 연결이 약해서 교체를 추천드려요")
