## 1. 왜 마케팅팀에 여러 에이전트가 필요한가

### 단일 에이전트의 한계

하나의 에이전트가 "기획→제작→게시→분석"을 전부 처리하면:
- 컨텍스트 윈도우가 빠르게 소진됨 (이미지/영상 생성 + 텍스트 작성 + API 호출)
- Soul 템플릿이 너무 방대해져 전문성 저하
- 에러 발생 시 전체 파이프라인 중단

### 팀 구성: 4명의 에이전트

```
┌──────────────────────────────────────────────────────┐
│                   마케팅 부서장                        │
│                (Marketing Manager)                     │
│     전체 캠페인 전략 수립, 작업 할당, 성과 평가         │
└───────────┬──────────────┬──────────────┬─────────────┘
            │              │              │
     ┌──────▼──────┐ ┌────▼─────┐ ┌──────▼──────┐
     │  콘텐츠     │ │ 크리에   │ │  퍼블리셔   │
     │  기획자     │ │ 이터     │ │             │
     │             │ │          │ │             │
     │ 트렌드 분석 │ │ 글 작성  │ │ 플랫폼별    │
     │ 주제 선정   │ │ 이미지   │ │ 게시        │
     │ 캘린더 관리 │ │ 영상     │ │ 성과 추적   │
     │ 타겟 분석   │ │ 카드뉴스 │ │ 보고서 작성 │
     └─────────────┘ └──────────┘ └─────────────┘
```

## 2. 에이전트별 상세 설계

### 2.1 마케팅 부서장 (Marketing Manager)

| 항목 | 내용 |
|------|------|
| **Tier** | 매니저급 |
| **역할** | 전체 캠페인 전략 수립, 팀원에게 작업 할당, KPI 관리 |
| **주요 도구** | `call_agent`, `content_calendar`, `search_web`, `search_news` |
| **Soul 핵심** | "LeetMaster의 마케팅 전략가. 트렌드를 파악하고 적시에 콘텐츠를 기획한다. 팀원에게 구체적 지시를 내리고 결과를 검증한다." |

**자율 판단 시나리오:**
- 법학 시험 시즌 감지 → 자동으로 "시험 대비 콘텐츠 캠페인" 기획
- 경쟁사 신기능 출시 감지 → "LeetMaster 차별점 강조 콘텐츠" 기획
- 게시물 성과 저조 감지 → 전략 수정 + 새 콘텐츠 방향 지시

### 2.2 콘텐츠 기획자 (Content Planner)

| 항목 | 내용 |
|------|------|
| **Tier** | 실무자급 |
| **역할** | 트렌드 분석, 키워드 리서치, 콘텐츠 주제/구조 설계 |
| **주요 도구** | `search_web`, `search_news`, `hashtag_generator`, `sentiment_analyzer`, `content_calendar` |
| **MCP** | Brave Search, Naver Search, Firecrawl |
| **Soul 핵심** | "법학/코딩테스트/취업 트렌드를 분석하는 리서처. 각 플랫폼의 특성을 이해하고 최적의 콘텐츠 구조를 설계한다." |

**출력물 예시:**
```markdown
## 콘텐츠 기획서: LeetMaster AI 면접 코칭 런칭

### 타겟
- 로스쿨 3학년 + 변시 준비생
- 코딩테스트 준비 중인 취준생

### 플랫폼별 전략
| 플랫폼 | 콘텐츠 형식 | 핵심 메시지 | 해시태그 |
|--------|-----------|-----------|---------|
| Instagram | 카드뉴스 5장 | "AI가 면접관이 되어준다" | #면접준비 #AI코칭 |
| X | 스레드 5트윗 | "3분만에 AI 면접 체험하기" | #LeetMaster #면접AI |
| YouTube | 데모 영상 90초 | "실제 AI 면접 시뮬레이션" | — |
| Tistory | 블로그 포스트 | 상세 기능 소개 + 사용법 | — |

### 크리에이터 지시사항
- 카드뉴스: 슬라이드 1=훅, 2-3=문제 제시, 4=해결책, 5=CTA
- 영상: 화면 녹화 + AI 음성 나레이션, 자막 필수
- 톤: 전문적이되 친근한, "~해보세요" 어투
```

### 2.3 크리에이터 (Content Creator)

| 항목 | 내용 |
|------|------|
| **Tier** | 실무자급 |
| **역할** | 글 작성, 이미지/카드뉴스 생성, 영상 제작 |
| **주요 도구** | `generate_ai_image`, `generate_card_news`, `generate_video`, `text_to_speech`, `translate_text` |
| **MCP** | Replicate Flux, DALL-E MCP |
| **Soul 핵심** | "시각적으로 매력적인 콘텐츠를 만드는 크리에이터. 카드뉴스, 릴스, 블로그 글을 각 플랫폼 규격에 맞게 제작한다." |

**생산 파이프라인:**

```
기획서 수신 (call_agent로)
    ↓
[텍스트 콘텐츠 생성]
  - 블로그 글 작성 (Tistory용 HTML)
  - X 스레드 텍스트 작성 (280자씩)
  - Instagram 캡션 작성 (해시태그 포함)
    ↓
[이미지 콘텐츠 생성]
  - AI 이미지: generate_ai_image (Flux 2 Pro → $0.04/장)
  - 카드뉴스: generate_card_news (Satori 템플릿 → PNG)
  - 썸네일: generate_ai_image + Sharp 리사이즈
    ↓
[영상 콘텐츠 생성]
  - 스크립트 → TTS (ElevenLabs, 한국어)
  - 이미지 시퀀스 + TTS → 영상 (Remotion 또는 FFmpeg)
  - OR AI 영상: Replicate Kling 3.0 ($0.029/초)
    ↓
[플랫폼별 리사이즈]
  - Instagram: 1080x1080 (피드), 1080x1920 (릴스)
  - X: 1200x675
  - YouTube: 1280x720 (썸네일), 1080x1920 (Shorts)
    ↓
call_agent → 퍼블리셔
```

### 2.4 퍼블리셔 (Publisher & Analyst)

| 항목 | 내용 |
|------|------|
| **Tier** | 실무자급 |
| **역할** | 플랫폼별 게시 + 성과 추적 + 보고서 생성 |
| **주요 도구** | `publish_instagram`, `publish_x`, `publish_youtube`, `publish_tistory`, `publish_daum_cafe`, `get_instagram_insights`, `engagement_analyzer`, `generate_pdf` |
| **MCP** | Playwright (Tistory/다음카페), Brave Search |
| **Soul 핵심** | "각 플랫폼의 최적 게시 시간과 포맷을 아는 퍼블리싱 전문가. 게시 후 성과를 추적하고 보고서를 작성한다." |

**게시 최적 시간:**

| 플랫폼 | 한국 기준 최적 시간 | 근거 |
|--------|-------------------|------|
| Instagram | 오전 11시, 오후 1시, 저녁 7시 | 점심/퇴근 시간 |
| X | 오전 9시, 오후 12시, 저녁 6시 | 출근/점심/퇴근 |
| YouTube | 오후 2시-4시 (업로드), 저녁 7시-9시 (시청) | 알고리즘 인덱싱 시간 확보 |
| Tistory | 오전 10시 | 검색 유입 최적화 |
| 다음카페 | 오후 8시-10시 | 커뮤니티 활동 피크 |

## 3. 콘텐츠 유형별 워크플로우

### 3.1 카드뉴스 (Instagram 캐러셀)

```
[기획자] 주제 + 슬라이드 구조 설계 (5~10장)
    ↓
[크리에이터]
  1. 슬라이드별 텍스트 작성
  2. 배경 이미지 생성 (AI 이미지 or 템플릿)
  3. Satori로 카드뉴스 PNG 생성 (1080x1080)
  4. 최종 검수
    ↓
[퍼블리셔]
  1. Instagram 캐러셀로 게시 (최대 10장)
  2. 동시에 X에 첫 장 이미지 + 링크 트윗
  3. 24시간 후 인사이트 수집
```

**예상 소요**: AI ~2분 (이미지 생성 포함)

### 3.2 블로그 포스트 (Tistory + 다음카페)

```
[기획자] SEO 키워드 + 구조 설계
    ↓
[크리에이터]
  1. 긴 글 작성 (2,000~3,000자)
  2. 본문 이미지 3~5개 생성
  3. HTML 변환 (Tistory 에디터 호환)
    ↓
[퍼블리셔]
  1. Tistory 게시 (Playwright 자동화)
  2. 다음카페 요약본 게시 (Playwright)
  3. 검색 노출 확인 (Naver Search MCP)
```

### 3.3 쇼츠/릴스 (YouTube Shorts + Instagram Reels)

```
[기획자] 60초 이내 스크립트 구조
    ↓
[크리에이터]
  1. 스크립트 작성 (60초 분량 ~150단어)
  2. TTS 음성 생성 (ElevenLabs)
  3. 방법 A: 이미지 시퀀스 + TTS → Remotion 영상
     방법 B: AI 영상 생성 (Replicate Kling)
  4. 자막 추가
  5. 세로 포맷 확인 (1080x1920, 9:16)
    ↓
[퍼블리셔]
  1. YouTube Shorts 업로드 (제목에 #Shorts)
  2. Instagram Reels 업로드
  3. 48시간 후 조회수/도달 추적
```

### 3.4 긴 영상 (YouTube)

```
[기획자] 5~15분 영상 기획서
    ↓
[크리에이터]
  1. 대본 작성 (분당 ~250단어)
  2. 섹션별 시각 자료 생성
  3. TTS 나레이션 (ElevenLabs)
  4. Remotion으로 영상 조립:
     - 인트로 (5초)
     - 본문 섹션들 (이미지 + 나레이션 + 자막)
     - 아웃트로 (CTA, 10초)
  5. 썸네일 생성 (1280x720)
    ↓
[퍼블리셔]
  1. YouTube 업로드 + 썸네일 설정
  2. 설명란에 타임스탬프 + 링크
  3. Tistory에 요약 글 + 임베드
  4. X에 홍보 트윗
```

### 3.5 X 스레드

```
[기획자] 스레드 주제 + 5~10개 트윗 구조
    ↓
[크리에이터]
  1. 각 트윗 텍스트 작성 (280자 이내)
  2. 첫 트윗용 이미지 1개 생성
  3. 필요 시 중간 트윗용 이미지 추가
    ↓
[퍼블리셔]
  1. 스레드 게시 (twitter-api-v2 tweetThread)
  2. 24시간 후 조회/리트윗/좋아요 추적
```

## 4. 자율 실행 시나리오

마케팅팀이 **스스로 판단해서 행동**하는 시나리오들:

### 시나리오 A: 정기 콘텐츠 발행
```
[ARGOS 스케줄러 → 매주 월/수/금 오전 9시]
    ↓
[마케팅 부서장]
  → content_calendar 확인 → 오늘 예정된 콘텐츠 확인
  → call_agent → 기획자 → 크리에이터 → 퍼블리셔
  → 결과 보고서 생성 → 허브에 보고
```

### 시나리오 B: 트렌드 감지 & 대응
```
[ARGOS 스케줄러 → 매일 오전 8시]
    ↓
[기획자]
  → search_news + Naver Search MCP → "법학/코딩테스트" 키워드 트렌드 분석
  → 급상승 키워드 발견 시 → 마케팅 부서장에게 보고
    ↓
[마케팅 부서장]
  → 트렌드 판단 → 긴급 콘텐츠 기획 지시
  → 크리에이터 → 퍼블리셔 자동 실행
```

### 시나리오 C: 성과 기반 자동 최적화
```
[ARGOS 스케줄러 → 매주 일요일]
    ↓
[퍼블리셔]
  → 주간 성과 수집 (각 플랫폼 인사이트)
  → 보고서 생성 → PDF → 이메일/Notion
  → 성과 저조 콘텐츠 분석 → 마케팅 부서장에게 보고
    ↓
[마케팅 부서장]
  → 다음 주 전략 조정
  → "카드뉴스 성과 좋음 → 비중 증가"
  → "블로그 SEO 트래픽 증가 → Tistory 빈도 증가"
```

## 6. 구현 우선순위

1. `publish_x` (X API v2 연동)

### Phase 3c: 마케팅팀 에이전트 생성
9. 4명의 에이전트 Soul 템플릿 작성
10. 도구 할당 (agent_tools)
11. ARGOS 스케줄 설정
12. 성과 대시보드 (admin 패널)
