## 2. 에러 및 버그 리포트 (상세)

### [BUG-001] 에이전트 "온라인 활성화" UI 컴포넌트 완전 누락
- **심각도**: Blocker (Critical)
- **페이지 URL**: `https://corthex-hq.com/admin/agents`
- **재현 단계**:
  1. 관리자로 로그인 후 'AI 에이전트' 메뉴 진입
  2. 오프라인 상태인 임의의 에이전트 좌측/우측 `EDIT` 버튼 클릭
  3. 우측에 뜨는 상세 속성 패널의 'Config', 'Soul', 'Memory' 등 전체 구조 탐색
- **기대 결과**: 꺼져있는 에이전트를 `온라인(Active/Online)` 상태로 즉각 켤 수 있는 스위치나 "활성화" 버튼이 노출되어야 함.
- **실제 결과**: `Deactivate Agent`(선제 비활성화)와 `Permanently Delete`(영구 삭제) 버튼만이 존재하며, 정작 가장 중요한 **단절된 에이전트를 다시 가동하는 스위치**가 화면 프론트엔드 컴포넌트 단에서 아예 누락됨. (이로 인해 영구적인 오프라인 상태 고착화)
- **재현율**: 항상

### [BUG-002] 오프라인 상태 시 Hub 메인 접속 무한 '재연결 중...' 에러
- **심각도**: Major
- **페이지 URL**: `https://corthex-hq.com/hub` (또는 CEO App 메인)
- **재현 단계**: 에이전트가 오프라인인 상태에서 Hub 접속 및 상단/하단 터미널 상태 확인
- **기대 결과**: 서버가 오프라인임을 알리는 명확한 안내창이 뜨거나 기능이 안전하게 락(Lock) 되어야 함.
- **실제 결과**: 화면 우측 상단에 계속해서 주황색 글씨로 "Reconnecting..." 에러가 떠있으며, 하단 터미널에서 채팅 전송 자체는 되나 아무런 응답 과정 없이 무한 로딩에 빠지는 프론트엔드 미처리 상태.
- **재현율**: 항상

### [BUG-003] 오프라인 상태에서 프리셋 버튼 강제 클릭 시 에러 토스트
- **심각도**: Major
- **페이지 URL**: `https://corthex-hq.com/hub`
- **재현 단계**: Hub 초기 화면에 떠 있는 '오늘 뉴스 브리핑해줘' 등 4개의 프리셋 버튼 중 하나 무작위 클릭
- **기대 결과**: 클릭 동작 자체가 UI 단에서 막히거나, 안전한 커스텀 모달 노출.
- **실제 결과**: 누르는 즉시 UI 하단에 `“에이전트 실행에 실패했습니다. 잠시 후 다시 시도해주세요”`라는 Back-end 서버 통신 실패 메시지가 무자비하게 반환됨. 방어 로직 부족.

## 3. 건너뛴 항목 (Skipped) 사유 종합

위의 치명적 블로킹 이슈([BUG-001])로 인해, 서버와의 명령 처리 파이프라인이 물리적으로 통제되었기 때문에 부득이하게 아래 항목들의 테스트를 패스했습니다.

- **명령 센터 및 채팅 체인 (2-2, 2-3, 2-13)**: 자연어 입력, 타 에이전트 직접 맨션(@), 슬래시(/) 명령어 실행 불가. 비서실장이 마케팅/법무팀으로 작업을 분배하고 트래킹(Handoff Tracker)하는 다중 위임 시나리오 진입 불가.
- **기타 UI 기반 기능 (2-4 ~ 2-12, 2-14 ~ 2-18)**: 부서 관리, 지식 베이스(폴더/문서), 비용 통계 결산, 트레이딩 차트, Big Five 등 에이전트의 가동 여부와 직결되지 않는 "단순 UI CRUD 컴포넌트" 들은 당장이라도 테스트가 가능하지만, 현재 맥락상 Part 2 진행 전체가 중단되었으므로 일단 일괄 Skip 리스트에 올렸습니다.

## 5. 임시 해결책 (워크어라운드: 에이전트 수동 활성화 방법)

백엔드 API(`PATCH /api/admin/agents/:id`)는 `status: 'online'` 업데이트를 지원하고 있으므로 다음 두 가지 방법 중 하나로 에이전트를 `online`으로 강제 전환할 수 있습니다.

### 방법 1. 프론트엔드 임시 버튼 추가 (추천)
에이전트 관리 페이지 코드인 `packages/admin/src/pages/agents.tsx` 파일을 열고, 약 656번째 줄에 위치한 `Save Changes` 버튼 아래에 임시로 다음 코드를 추가한 후 브라우저에서 버튼을 한 번 클릭하세요.
```tsx
<button
  onClick={() => {
    if (selectedAgent) updateMutation.mutate({ id: selectedAgent.id, status: 'online' as any })
  }}
  className="w-full py-2 font-mono text-xs uppercase font-bold text-green-500 border border-green-500 hover:bg-green-500/10 mt-2"
>
  Force Online
</button>
```

### 방법 2. Drizzle DB Studio를 통한 직접 수정
터미널에서 백엔드 폴더로 이동하여 DB 스튜디오를 실행합니다.
```bash
cd packages/server
npx drizzle-kit studio
```
스튜디오 웹 화면이 열리면 `agents` 테이블에서 `status` 컬럼의 값을 `offline`에서 `online`으로 변경하고 저장하시면 됩니다.
