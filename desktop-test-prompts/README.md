# CORTHEX Desktop App 테스트 프롬프트

5개 파트로 나뉘어 있습니다. **순서대로 하나씩** Desktop App에 붙여넣으세요.

**DB 상태**: 기본 시드 적용됨 (1회사 "CORTHEX HQ" + admin/ceo 유저 + 4부서 + 4에이전트). 온보딩에서 회사명/부서/에이전트를 수정하는 것부터 시작.

| 파트 | 파일 | 소요 시간 | 내용 |
|------|------|-----------|------|
| 0 | part0-setup.md | ~3분 | 환경 준비, 로그인 확인, 버그 보고 양식 |
| 1 | part1-admin.md | ~25분 | Admin 패널 전체 (로그인, 온보딩, CRUD, 에이전트, 권한 등) |
| 2 | part2-app.md | ~30분 | CEO App 전체 (Hub, 채팅, 슬래시 명령어, 협업, Big Five 등) |
| 3 | part3-mobile.md | ~10분 | 모바일 반응형 테스트 (iPhone + iPad) |
| 4 | part4-evaluation.md | ~5분 | 종합 평가표 + 버그 리스트 |

## Desktop App에 처음 줄 메시지

아래를 **첫 번째 메시지**로 보내세요:

```
CORTHEX v2 AI 플랫폼 전체 기능 테스트를 합니다.
총 5파트(Part 0~4)로 나눠서 진행할 거예요. 제가 파트별로 지시를 줄게요.
Chrome 브라우저를 열어서 실제로 모든 기능을 하나하나 사용해보세요.

핵심 규칙:
1. 페이지를 "보기만" 하지 마세요. 버튼이 있으면 누르고, 폼이 있으면 입력하고, 드롭다운이 있으면 열어보세요.
2. 각 단계마다 "성공 기준"이 적혀있으니 그걸 확인해주세요.
3. 에이전트 채팅 응답은 30초~2분 걸립니다. 반드시 끝까지 기다리세요.
4. 에러나 버그를 발견하면 Part 0에 있는 양식으로 상세하게 보고해주세요.
   - 단순히 "안됨"이 아니라: 어떤 페이지에서, 무엇을 클릭했는데, 뭐가 나와야 하는데, 뭐가 나왔는지를 적어주세요.
5. 에러 때문에 뒤 단계가 불가능한 경우에만 → 막힌 지점을 상세 보고 후 다음으로 진행. 그 외에는 **모든 항목을 빠짐없이 전부** 테스트하세요. 절대 건너뛰지 마세요.

기본 정보:
- Admin 패널: https://corthex-hq.com/admin (admin / admin1234)
- CEO App: https://corthex-hq.com (admin / admin1234)
- Part 0부터 시작합니다.

준비되면 "준비됐어"라고 말해주세요. 그러면 Part 0을 보내드리겠습니다.
```

그 다음 "준비됐어" 응답이 오면 part0-setup.md 내용을 붙여넣으세요.
Part 0 끝나면 part1-admin.md, 그 다음 part2, part3, part4 순서로.

## 중요: 파트 간 의존성

- **Part 0** (환경 준비)가 먼저. 로그인이 안 되면 나머지 전부 불가.
- **Part 1** (Admin)이 기반. 여기서 회사/부서/에이전트를 만들어야 Part 2 (App)에서 쓸 수 있음.
- Part 1에서 막히면 → Part 2로 넘어가지 말고 → 막힌 지점을 버그 리스트에 기록.
- **Part 3** (모바일)은 독립적. Part 1-2가 막혀도 진행 가능.
