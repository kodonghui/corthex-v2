## 발견된 버그
### BUG-001: 액센트 컬러 변경 불가
- 페이지: https://corthex-hq.com/settings?tab=display
- 재현 단계:
  1. 설정 → 테마 탭 이동
  2. 액센트 컬러 섹션에서 Olive 외 다른 색상 (Orange, Green, Purple, Gray) 클릭
  3. 체크마크가 여전히 Olive에 유지됨
- 기대 결과: 클릭한 색상으로 체크마크 이동 + 액센트 컬러 변경
- 실제 결과: 클릭해도 반응 없음. 체크마크 Olive 고정, 라벨 "Olive (기본값)" 유지. 콘솔 에러 없음.
- 심각도: Minor (기능은 존재하나 동작하지 않음)

### BUG-002: 비밀번호 빈칸 제출 시 유효성 검증 메시지 없음
- 페이지: https://corthex-hq.com/settings (일반 탭)
- 재현 단계:
  1. 일반 탭에서 비밀번호 변경 섹션으로 이동
  2. 새 비밀번호/비밀번호 확인 필드를 비운 상태로 "비밀번호 변경" 버튼 클릭
- 기대 결과: "비밀번호를 입력해주세요" 등 유효성 검증 오류 메시지 표시
- 실제 결과: 아무 반응 없음 (silent fail). 오류 메시지, 토스트, 필드 하이라이트 모두 없음
- 비고: 비밀번호 불일치 시에는 "비밀번호가 일치하지 않습니다" 빨간 메시지가 정상 출력됨
- 심각도: Minor (사용자 혼란 유발 가능)

### BUG-003: 라이트 모드 전환 시 UI 미적용 의심
- 페이지: https://corthex-hq.com/settings?tab=display
- 재현 단계:
  1. 테마 탭에서 모드를 "라이트" 클릭
  2. 토스트 "테마가 변경되었습니다" 확인
  3. 그러나 UI가 여전히 다크 모드로 표시
- 기대 결과: 라이트 모드로 전체 UI 변경
- 실제 결과: 토스트는 뜨지만 UI가 다크 유지 (OS가 다크 모드인 경우 "시스템" 모드와 동일하게 보일 수 있으나, 명시적으로 "라이트" 선택 시에는 라이트가 적용되어야 함)
- 심각도: Major (테마 설정의 핵심 기능)

## UX 탐색 발견사항
- **알림 필터 입력** → 텍스트 입력 정상 동작, XSS 입력(`<script>alert(1)</script>`) 안전하게 텍스트로 처리됨
- **알림 설정 톱니바퀴 아이콘** → 클릭 시 Notification Settings 페이지로 전환, "← BACK TO LIST" 링크로 목록 복귀 가능
- **알림 설정 페이지 하단** → "알림은 30일간 보관됩니다" 안내 메시지 존재 (좋은 UX)
- **사이드바 숨겨진 메뉴** → 스크롤 시 추가 링크 확인: memories, files, costs, performance, activity-log, ops-log, classified, notifications, settings
- **비밀번호 불일치 검증** → "비밀번호가 일치하지 않습니다" 빨간 메시지 정상 출력 (정상 동작)
- **테마 모드 전환 토스트** → 변경 즉시 "✓ 테마가 변경되었습니다" 토스트 표시 (좋은 피드백)
- **사용자명/이메일 필드** → 읽기 전용으로 보임 (수정 불가), 이름 필드만 수정 가능
- **소울 편집 탭** → 에이전트 선택 드롭다운에 "내 에이전트 2개" 표시
- **MCP 연동 탭** → 빈 상태에서 친절한 안내 메시지 제공 ("서버를 추가하면 에이전트가 외부 도구를 사용할 수 있습니다")
