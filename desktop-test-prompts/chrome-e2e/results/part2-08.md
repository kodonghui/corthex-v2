## 발견된 버그

### BUG-001: 사이드바에 "보고서" 메뉴 항목 없음
- 페이지: 사이드바 전체
- 재현 단계: 1. 사이드바의 모든 메뉴를 스크롤하여 확인
- 기대 결과: "보고서" 메뉴가 사이드바에 표시되어야 함
- 실제 결과: 사이드바에 "보고서" 항목이 없음. /reports URL 직접 접근으로만 가능
- 심각도: Major — 사용자가 보고서 페이지를 발견하기 어려움

### BUG-002: STATISTICS 패널 Total이 검색 필터와 동기화되지 않음
- 페이지: https://corthex-hq.com/reports
- 재현 단계: 1. 보고서 목록 페이지 열기 2. 검색 필드에 매칭되지 않는 검색어 입력 (예: "zzzznotexist") 3. STATISTICS 패널 확인
- 기대 결과: Total이 필터링 결과에 따라 0으로 변경
- 실제 결과: Active Reports 배지는 0으로 변하지만, STATISTICS의 Total은 여전히 2 표시
- 심각도: Cosmetic — 통계가 필터링과 동기화되지 않아 혼동 가능

## UX 탐색 발견사항
- **필터 체크박스 (Drafts/Submitted/Reviewed)** → 정상 동작. Drafts 체크 시 DRAFT 보고서만 표시, Submitted 체크 시 "보고서가 없습니다" 정상 표시
- **검색 기능** → 실시간 필터링 정상 동작. "QA" 입력 시 두 보고서 모두 표시 (둘 다 QA 포함), 매칭 안되는 검색어 시 0건으로 빈 상태 표시
- **Request Edit 버튼** → 클릭 시 인라인 편집 모드로 전환. 제목과 내용이 수정 가능한 입력 필드로 변경, 저장/취소 버튼 표시. 취소 시 원래 보기 모드로 복귀
- **Complete Review 버튼** → 클릭 시 확인 다이얼로그 표시: "CEO에게 보고 — 이 보고서를 CEO에게 보고하시겠습니까? 보고 후 본문 수정이 제한됩니다." 보고하기/취소 버튼 제공. 좋은 UX
- **삭제(휴지통) 아이콘** → 호버 시 하이라이트 효과 있음. 툴팁 없음 (Minor — 삭제 아이콘에 툴팁 추가 권장)
- **테이블 헤더 (TITLE/STATUS/AUTHOR/DATE)** → 클릭해도 정렬 기능 없음 (커서 포인터도 변하지 않아 정렬 불가를 암시)
- **보고서 상세 보기 (기존 보고서)** → AI Recommendation 섹션이 표시되어 부가 가치 제공
- **콘솔 에러** → 없음
