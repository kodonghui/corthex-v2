## 발견된 버그
### BUG-001: 워크플로우 목록 로딩 에러
- 페이지: https://corthex-hq.com/n8n-workflows
- 재현 단계: 1. 사이드바 → 워크플로우 클릭 2. 페이지 로딩 대기
- 기대 결과: 워크플로우 목록이 정상 로딩되거나, n8n 미연결 시 안내 메시지 표시
- 실제 결과: 에러 배너 "워크플로우 목록을 불러올 수 없습니다" 표시 (빨간 배경). 에러 원인이나 해결 방법 안내 없음
- 심각도: Minor (n8n 백엔드 미연결 시 발생하는 것으로 추정, 하지만 사용자에게 원인/해결 안내 부재)

### BUG-002: "+ NEW WORKFLOW" 버튼 클릭 시 무반응
- 페이지: https://corthex-hq.com/n8n-workflows
- 재현 단계: 1. 워크플로우 페이지 진입 2. "+ NEW WORKFLOW" 버튼 클릭
- 기대 결과: 워크플로우 생성 다이얼로그/페이지 표시 또는 에러 피드백
- 실제 결과: 클릭 시 아무런 반응 없음 (다이얼로그 없음, 네비게이션 없음, 에러 메시지 없음)
- 심각도: Minor (n8n 미연결 상태에서도 최소한 피드백 필요)

### BUG-003: 작업 행 Actions 메뉴(⋮) 클릭 시 무반응
- 페이지: https://corthex-hq.com/jobs
- 재현 단계: 1. Jobs 페이지 진입 2. 작업 행 우측 끝 "⋮" 아이콘 클릭

## UX 탐색 발견사항
- **Jobs 탭 전환** → 야간 작업(1), 반복 스케줄, ARGOS 트리거 탭 모두 정상 전환. 각 탭에 맞는 빈 상태 메시지 표시 ("등록된 반복 스케줄이 없습니다", "등록된 이벤트 트리거가 없습니다")
- **작업 행 클릭** → 행이 확장되어 AI 에이전트 응답 결과 표시. 마크다운 형식의 결과물이 잘 렌더링됨
- **"결과 보기 →" 링크** → 해당 작업의 채팅 세션(/chat?session=...)으로 이동. 에이전트와의 대화 내역 확인 가능
- **"보고서 보기 →" 링크** → 존재 확인 (클릭 미테스트)
- **작업 생성 폼 - 에이전트 선택** → 콤보박스 방식. 선택 가능한 에이전트: 경영보좌관, 보안감사봇 (2개)
- **작업 생성 폼 - 체인 단계 추가** → "+ 체인 단계 추가 (순차 실행)" 링크 존재. 복수 에이전트 순차 실행 지원
- **워크플로우 필터 입력** → "Filter active workflows..." 필드에 텍스트 입력 가능. 정상 동작
- **워크플로우 통계** → TOTAL THROUGHPUT: 2.4M (+14.2% THIS WEEK), ACTIVE INSTANCES: 882, SYSTEM HEALTH: 99.9% (하드코딩된 데모 데이터로 추정)
- **LIVE EXECUTION STREAM 패널** → 확장/축소 아이콘 존재하나 클릭 시 반응 불분명. "실행 기록이 없습니다" 표시
- **워크플로우 "Send command to kernel..." 입력** → 페이지 하단에 커맨드 입력 필드 존재 (미테스트)
- **사이드바에서 Jobs 메뉴 찾기** → 사이드바에 "Jobs" 또는 "작업" 라벨이 명시적으로 표시되지 않아 찾기 어려움. URL 직접 입력(/jobs)으로 접근 필요. UX 개선 검토 필요
