# Part 1-2: 온보딩 테스트 (4분)

## 성공 기준
5단계 위저드를 끝까지 완료하고 대시보드로 복귀

## 테스트 단계

사이드바 → "온보딩" 클릭 (또는 /admin/onboarding)

### Step 1 — 회사 설정
1. 현재 회사 목록 확인 (기본: "CORTHEX HQ")
2. "CORTHEX HQ" 옆 "Edit" 클릭
3. 회사명을 `크롬QA공사`로 변경
4. "Save" 클릭
5. **확인**: 토스트 메시지 표시
6. **확인**: 회사명 변경됨
7. 스크린샷: `screenshots/part1-02-step1.png`

### Step 2 — 부서/템플릿 설정
8. 조직 템플릿 목록 확인 (투자분석, 마케팅, 올인원 등)
9. "올인원" 템플릿의 "Apply" 클릭
10. **확인**: 에이전트 자동 생성됨
11. 커스텀 부서 추가: `보안점검팀` 입력 → "Add" 클릭
12. **확인**: 목록에 "보안점검팀" 추가됨
13. 빈 이름으로 "Add" 클릭 → **확인**: 방어됨 (아무 일 안 일어남)
14. 스크린샷: `screenshots/part1-02-step2.png`

### Step 3 — API 키
15. Anthropic API 키 입력란 확인
16. `sk-chrome-qa-test-0329` 입력 → "등록" 클릭
17. **확인**: 등록된 키 개수 표시
18. "Continue" 클릭
19. 스크린샷: `screenshots/part1-02-step3.png`

### Step 4 — 팀원 초대
20. 아이디: `cqa01`
21. 이름: `크롬검수원`
22. 이메일: `cqa01@chrometest.dev`
23. "초대하기" 클릭
24. **확인**: 임시 비밀번호 모달 팝업 (다크 테마 오버레이)
25. **확인**: 모달 안에 비밀번호가 빈칸이 아닌 실제 문자열
26. **확인**: "직원이 첫 로그인하면 이 비밀번호는 사라집니다" 안내
27. "Copy" 클릭 → **확인**: 버튼 텍스트 "Copied!" 변경
28. **비밀번호 메모** (part1-08에서 사용)
29. "Confirm" 클릭 → 모달 닫힘
30. 스크린샷: `screenshots/part1-02-step4.png`

### Step 5 — 완료
31. 요약 정보 확인
32. "CORTHEX 사용 시작하기" 클릭
33. **확인**: 대시보드로 이동
34. 스크린샷: `screenshots/part1-02-step5.png`
