## PART 0: 테스트 환경 준비 (약 3분)

이 문서는 나머지 모든 파트의 사전 조건입니다. **반드시 Part 0을 먼저 완료한 뒤 Part 1로 넘어가세요.**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### 0-1. Chrome 브라우저 열기
1. Chrome 브라우저를 실행
2. DevTools(F12)를 열어두면 에러 발생 시 Console 탭에서 확인 가능 (디버깅용)
3. Console에 기존 빨간색 에러가 있으면 무시 (서드파티 에러일 수 있음)

### 0-2. Admin 로그인 확인
1. `https://corthex-hq.com/admin/login` 으로 이동
2. "아이디" 필드에 `admin` 입력
3. "비밀번호" 필드에 `admin1234` 입력
4. "세션 시작" 버튼 클릭
5. **확인**: 대시보드 페이지가 로드되고, 상단에 숫자가 있는 카드 3개가 보임
6. **실패 시**: 에러 메시지를 정확히 적고, Console 탭의 빨간 에러를 캡처한 뒤 아래 버그 보고 템플릿으로 보고

### 0-3. CEO App 로그인 확인
1. 새 탭을 열고 `https://corthex-hq.com/login` 접속
2. "아이디" 필드에 `admin` 입력
3. "비밀번호" 필드에 `admin1234` 입력
4. "INITIALIZE COMMAND" 버튼 클릭
5. **확인**: Hub 페이지가 로드되고, 하단에 터미널 입력창이 보임
6. **실패 시**: 버그 보고

### 0-4. DB Seed 데이터 확인
1. Admin 탭으로 전환 (0-2에서 로그인한 탭)
2. 좌측 사이드바에서 "회사 관리" 클릭
3. **확인**: 회사 카드가 최소 1개 있는지 확인 (예: "CORTHEX HQ")
4. 좌측 사이드바에서 "에이전트" 클릭
5. **확인**: 에이전트 행이 최소 1개 있는지 확인
6. **만약 회사가 0개이면**: "시드 데이터가 없습니다. 서버에서 seed를 실행해야 합니다." 라고 보고하고 **모든 테스트를 중단**하세요. 회사가 없으면 나머지 기능이 전부 동작하지 않습니다.

### 0-5. 버그 보고 양식

**이후 모든 테스트에서 에러/버그를 발견하면 반드시 아래 양식으로 보고하세요.**
"안됨"이나 "에러" 한 마디로 끝내지 마세요. 아래 항목을 빠짐없이 채워주세요.

```
### BUG-XXX: [한줄 요약 -- 무엇이 안 되는지]

- **심각도**: Critical(기능 완전 불가) / Major(주요 기능 일부 불가) / Minor(사소한 문제) / Cosmetic(외관만)
- **페이지 URL**: [에러가 발생한 전체 URL, 예: https://corthex-hq.com/admin/employees]
- **재현 단계**:
  1. [첫 번째 행동 -- 예: "Add Employee" 버튼 클릭]
  2. [두 번째 행동 -- 예: 아이디 "test01" 입력]
  3. [세 번째 행동 -- 예: "초대" 버튼 클릭]
  4. [에러 발생 시점 -- 예: 모달이 닫히면서 빈 화면]
- **기대 결과**: [정상적으로 어떻게 되어야 하는지 -- 예: 임시 비밀번호 모달이 뜨고 비밀번호가 표시되어야 함]
- **실제 결과**: [실제로 어떻게 됐는지 -- 예: 모달은 떴지만 비밀번호 자리가 빈칸]
- **에러 메시지**: [화면에 표시된 에러 텍스트 정확히 -- 없으면 "없음"]
- **콘솔 에러**: [DevTools Console 탭에 빨간색으로 뜬 에러 메시지 -- 없으면 "없음"]
- **스크린샷**: [해당 화면 캡처]
- **재현율**: 항상(매번 발생) / 가끔(2~3회 중 1회) / 1회(한 번만 발생)
- **영향 범위**: [이 버그 때문에 뒤에 못 한 단계가 있으면 기재 -- 예: "1-5 직원 생성 이후 전체 불가"]
```

### 0-6. 에러 발생 시 기본 대처법
- **페이지가 흰 화면이면**: 새로고침 → 그래도 안 되면 Console 에러 확인
- **버튼 클릭이 안 되면**: 1초 대기 후 재시도 → 그래도 안 되면 페이지 새로고침
- **"401 Unauthorized" 에러면**: Admin/App 로그인 페이지로 돌아가서 재로그인
- **모달이 안 닫히면**: 모달 바깥 영역 클릭 또는 ESC 키
- **어떤 에러든 3회 이상 반복되면**: 버그 보고 후 해당 단계를 건너뛰고 다음으로 진행
